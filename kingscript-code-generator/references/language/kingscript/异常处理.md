## 异常处理

在 KingScript 中，异常处理是一种用于捕获和处理程序运行过程中发生的错误或异常情况的机制。

### 1.1 捕获异常

使用 `try...catch` 结构可以捕获并处理异常：

```kingscript
try {
    const invalidJson = "{name: 'John'}"; // 缺少双引号，不是有效的 JSON
    const data = JSON.parse(invalidJson);
    console.log(data.name);
} catch (error) {
    console.log(error); // 输出 Expected property name or '}' in JSON at position 1 (line 1 column 2)
}
```

在上述示例中，会抛出一个 `Error` 异常，并被捕获处理。

### 1.2 finally 块

使用 `finally` 块可以指定无论是否捕获到异常都执行的代码块：

```kingscript
try {
    const invalidJson = "{\"name\": \"John\"}"; 
    const data = JSON.parse(invalidJson);
    console.log(data.name);
} catch (error : any) {
    console.error("发生错误:", error.message);
} finally {
    console.log("无论如何都会执行");
}
```

### 1.3 抛出异常

使用 `throw` 关键字可以抛出一个异常：

```kingscript
function divide(a: number, b: number): number {
    if (b === 0) {
        throw new Error('除数不能为0');
    }
    return a / b;
}

try {
    const result = divide(10, 0);
    console.log('结果:', result);
} catch (error : any) {
    console.log('捕获到异常:', error.message);
}
```

### 1.4 异常处理的处理建议

虽然我们演示了如何拦截异常并打印日志的功能，但我们强烈建议：

> 不要拦截异常并打印日志，然后继续执行代码或return null，这会以为产品很稳定，但其实很多BUG。

建议的方法是：
- 不拦截异常，让外界统一处理；
- 或者拦截异常，重新抛出更友好的异常，例如异常文本中包含单据编号方便排查。

另外，注意不要在异常信息中包含敏感、隐私或机密等信息。
