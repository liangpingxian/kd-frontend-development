### 1. 文件命名
`KingScript` 脚本文件命名规范与 `Java` 类似，支持：字母、数字、下划线组合（数字不能出现在开头），接口以“I”开头。
`KingScript` 脚本名称不能超过 **50** 个字符。
#### 正确的命名
```kingscript
Person.ts
IAnimal.ts
Dog1.ts
Cat_1.ts
_MyPlugin.ts
```
#### 错误的命名
```kingscript
Person 1.ts //不能出现空格
1Cat.ts //不能以数字开头
```
#### 推荐命名
`KingScript` 可以用于插件、操作、业务扩展点等多种业务/功能场景，命名可以结合单据、功能等能表达明确的业务意义，好的命名能直观看出脚本的功能与作用。
```kingscript
SaveAfterOperator.ts //保存后操作插件
ChangeOrderStatusBillPlugin.ts //改变订单状态的单据插件
```
### 2. 接口命名
接口命名推荐使用 `I` 开头并使用大驼峰（首字母大写）
```kingscript
interface IAnimal {
  name: string
}
```
### 3. 类命名
类命名推荐使用大驼峰（首字母大写）
```kingscript
class Dog implements IAnimal{
    constructor(){
        this.name = "dog";
    }
}
```
### 4. 方法命名
方法命名使用小驼峰（首字母小写）
```kingscript
class Dog implements IAnimal{
    constructor(){
        this.name = "dog";
    }
    getName():string{
        return this.name;
    }
}
```
### 5. 变量命名
变量命名使用小驼峰（首字母小写）
```kingscript
class Dog implements IAnimal{
    //类属性定义
    color:string
    constructor(name:string,color:string){
        this.color = color;
    }
    getColor():string{
        return this.color;
    }
    toString():string{
        //过程变量定义
        let colorPrefix:string = "Dog color is:";
        return colorPrefix + this.color;
    }
}
```
