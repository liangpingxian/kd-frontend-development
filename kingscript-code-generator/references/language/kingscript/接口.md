## 接口

在 KingScript 中，接口是一种用于定义对象结构的抽象类型。它描述了对象应该具有的属性和方法，但不提供具体的实现。

### 1.1 接口声明

使用 `interface` 关键字可以声明一个接口：

```kingscript
interface Person {
    name: string;
    age: number;
}
```

### 1.2 实现接口

类可以通过 implements 关键字来实现接口，确保其满足接口的要求：

```kingscript
class Employee implements Person {
    name: string;
    age: number;

    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }
}
```

### 1.3 可选属性

接口中的属性可以是可选的，使用 `?` 符号表示：

```kingscript
interface Person {
    name: string;
    age?: number; // 可选属性
}

let person1: Person = { name: "Alice" };
let person2: Person = { name: "Bob", age: 30 };
```

### 1.4 函数类型接口

接口也可以用于定义函数的签名，这样你就可以像函数一样调用了：

```kingscript
interface GreetFunction {
    (name: string): void;
}

function greet(fn: GreetFunction) {
    fn("Hello, World!");
}
```

### 1.5 接口继承

接口可以继承其他接口，以扩展其功能：

```kingscript
interface Animal {
    name: string;
}

interface Dog extends Animal {
    bark(): void;
}

let dog: Dog = { name: "Buddy", bark() { console.log("Woof!"); } };
```

### 1.6 接口与类的组合

接口可以用于描述类的结构，而类则提供具体的实现：

```kingscript
interface Shape {
    area(): number;
}

class Circle implements Shape {
    radius: number;

    constructor(radius: number) {
        this.radius = radius;
    }

    area(): number {
        return Math.PI * this.radius ** 2;
    }
}
```

### 1.7 接口与类型别名的比较

接口和类型别名都可以用于定义对象结构，但它们在使用场景上有区别：

- 接口主要用于描述类的结构。
- 类型别名可以用于定义任意类型的别名。

```kingscript
interface Person {
    name: string;
}

type Point = { x: number; y: number };
```

### 1.8 Hello, Interface!

我们可以使用接口来定义一个简单的 `Person` 接口，并创建一个实现该接口的类：

```kingscript
interface Person {
    name: string;
    age?: number;
}

class Employee implements Person {
    name: string;
    age?: number;

    constructor(name: string, age?: number) {
        this.name = name;
        this.age = age;
    }
}

let employee: Employee = new Employee("Alice", 30);
console.log(employee); // 输出 { name: "Alice", age: 30 }
```

输出结果：

```json
{
  "name": "Alice",
  "age": 30
}
```
