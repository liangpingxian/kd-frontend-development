# Kingscript Examples

这个目录存放“这段代码怎么写”的共享示例，面向事件写法、场景代码和完整插件示例。

## 使用建议

- 先按插件类型找最接近的示例。
- 如果已经知道事件名，优先进入事件拆分目录，而不是直接读超长总览文档。
- 示例负责回答“怎么写”，SDK 知识卡负责回答“是什么、何时触发、有哪些边界”。

## 外部资料的使用边界

如果维护者本地还有更完整的示例资料或反编译案例库，可以把它们作为 ideas 来源，但不建议直接整包搬进 `references/examples/`。

更推荐的做法是：

- 从外部资料里提炼高频场景
- 归并成可检索的事件拆分目录或场景拆分目录
- 保持一个场景只保留一份主版本，避免仓库内出现多份互相冲突的示例

适合优先沉淀进仓库的内容：

- 用户会直接问“怎么写”的事件
- 需要完整代码链路才能解释清楚的场景
- 容易踩运行时坑的联动写法

## 维护原则

- 不为长尾事件批量生成空壳示例，优先补高频、高风险、用户会直接问“怎么写”的事件。
- 同一事件如果已经在总览文档中存在完整写法，优先拆分复用，不重新发明第二份版本。
- 示例命名统一使用事件名的小写形式，和现有 `表单插件-事件拆分/` 保持一致。
