## 使用建议

- 已知事件名时，优先进入事件拆分目录，而不是直接阅读超长总览。
- 已知业务对象但还不知道具体事件时，优先看场景拆分目录，例如表单、字段控件、控件、列表、操作插件、单据体、基础资料、弹性域。
- 报表、转换、后台任务、移动端这几类场景现在也优先走场景拆分目录，再按需要回到 overview。
- 先找最接近的业务场景，再根据示例中的接口名回跳到 SDK 卡片确认边界。
- 一个问题如果同时涉及“事件写法”和“参数语义”，先看 examples，再看 `references/sdk/` 下对应知识卡。
