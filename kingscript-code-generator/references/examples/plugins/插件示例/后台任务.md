## setTaskId - 设置任务标识

**所属类**：`AbstractTask`
**签名**：`setTaskId(id: string): void`
**触发时机**：调度框架创建任务实例后、执行 `execute` 前，由框架自动调用，注入当前任务的唯一标识。
**典型用途**：在任务执行过程中通过任务 ID 记录日志、写入执行记录表、用于防重复执行判断。

### 业务场景：库存盘点任务记录执行日志

每晚定时执行库存盘点，通过 `setTaskId` 注入的任务 ID 写入执行日志表，便于运维追踪任务执行状态和排查问题。

```typescript
import { AbstractTask } from "@cosmic/bos-core/kd/bos/schedule";
import { BusinessDataServiceHelper, QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { SaveServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper/operation";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";

class InventoryCheckTask extends AbstractTask {

  setTaskId(id: string): void {
    super.setTaskId(id);

    // 任务启动时，写入执行日志（状态：执行中）
    let logObj = BusinessDataServiceHelper.newDynamicObject("kdec_task_log");
    logObj.set("kdec_task_id", id);
    logObj.set("kdec_task_name", "库存盘点任务");
    logObj.set("kdec_start_time", new Date());
    logObj.set("kdec_status", "running");
    SaveServiceHelper.saveOperate("kdec_task_log", [logObj], OperateOption.create());
  }

  execute(requestContext: any, params: any): void {
    let taskId = this.getTaskId();
    let processedCount = 0;

    // 查询所有启用仓库
    let warehouses = QueryServiceHelper.query(
      "bd_warehouse",
      "id,number,name",
      [new QFilter("enable", QCP.equals, "1")]
    );

    for (let warehouse of warehouses) {
      let warehouseId = warehouse.getLong("id");
      let warehouseName = warehouse.getString("name");

      // 查询库存余额与实际盘点数据比对（简化逻辑）
      let stockItems = QueryServiceHelper.query(
        "im_inventory_balance",
        "material.id,material.name,qty",
        [new QFilter("warehouse.id", QCP.equals, warehouseId)]
      );

      for (let item of stockItems) {
        processedCount++;
      }
    }

    // 任务完成后，更新日志状态
    let logObj = QueryServiceHelper.queryOne(
      "kdec_task_log",
      "id",
      [new QFilter("kdec_task_id", QCP.equals, taskId)]
    );

    if (logObj != null) {
      let log = BusinessDataServiceHelper.loadSingle(logObj.get("id"), "kdec_task_log");
      log.set("kdec_end_time", new Date());
      log.set("kdec_status", "completed");
      log.set("kdec_result", "盘点完成，共处理" + processedCount + "条记录");
      SaveServiceHelper.saveOperate("kdec_task_log", [log], OperateOption.create());
    }
  }
}

let plugin = new InventoryCheckTask();
export { plugin };
```

**要点说明**：

1. `setTaskId` 由调度框架在 `execute` 之前自动调用，无需手动触发
2. 重写时必须调用 `super.setTaskId(id)` 保证父类逻辑正常
3. 任务 ID 是本次调度执行的唯一标识，同一调度计划每次执行 ID 不同
4. 可利用任务 ID 实现防重复执行：在 `execute` 开头查询是否存在同 ID 的运行中记录

## setMessageHandle - 设置消息处理器

**所属类**：`AbstractTask`
**签名**：`setMessageHandle(messageHandler: MessageHandler): void`
**触发时机**：调度框架创建任务实例后、执行 `execute` 前，由框架自动调用，注入消息处理器实例。
**典型用途**：重写此方法可在消息处理器注入时做初始化准备，如配置消息格式、预设进度上报策略。

### 业务场景：数据同步任务初始化进度上报

ERP 与外部系统的数据同步任务，在消息处理器注入时预设进度上报模板和错误通知策略，后续执行过程中通过处理器实时反馈同步进度。

```typescript
import { AbstractTask } from "@cosmic/bos-core/kd/bos/schedule";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { SaveServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper/operation";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";

class DataSyncTask extends AbstractTask {

  setMessageHandle(messageHandler: any): void {
    super.setMessageHandle(messageHandler);

    // 消息处理器注入时，立即上报任务启动信息
    if (messageHandler != null) {
      messageHandler.sendMessage("数据同步任务初始化完成，准备开始同步...");
    }
  }

  execute(requestContext: any, params: any): void {
    let handler = this.getMessageHandler();

    // ---- 1. 同步供应商数据 ----
    if (handler != null) {
      handler.sendMessage("正在同步供应商数据...");
    }

    let suppliers = QueryServiceHelper.query(
      "bd_supplier",
      "id,number,name,status",
      [new QFilter("modifytime", QCP.large_than, this.getLastSyncTime())]
    );

    let supplierCount = 0;
    for (let supplier of suppliers) {
      // 同步逻辑（简化）
      supplierCount++;
    }

    if (handler != null) {
      handler.sendMessage("供应商同步完成，共" + supplierCount + "条");
    }

    // ---- 2. 同步物料数据 ----
    if (handler != null) {
      handler.sendMessage("正在同步物料数据...");
    }

    let materials = QueryServiceHelper.query(
      "bd_material",
      "id,number,name,materialtype",
      [new QFilter("modifytime", QCP.large_than, this.getLastSyncTime())]
    );

    let materialCount = 0;
    for (let material of materials) {
      materialCount++;
    }

    if (handler != null) {
      handler.sendMessage("物料同步完成，共" + materialCount + "条。任务全部完成。");
    }
  }

  // 辅助方法：获取上次同步时间（简化）
  getLastSyncTime(): Date {
    let result = QueryServiceHelper.queryOne(
      "kdec_sync_config",
      "kdec_last_sync_time",
      [new QFilter("kdec_task_type", QCP.equals, "data_sync")]
    );
    if (result != null) {
      return result.get("kdec_last_sync_time") as Date;
    }
    // 默认返回7天前
    let d = new Date();
    d.setDate(d.getDate() - 7);
    return d;
  }
}

let plugin = new DataSyncTask();
export { plugin };
```

**要点说明**：

1. `setMessageHandle` 由调度框架自动调用，重写时必须调用 `super.setMessageHandle(messageHandler)`
2. 注入的 `MessageHandler` 用于向调度平台上报任务执行进度和状态
3. 使用前应做非空判断，某些调度场景下可能不注入消息处理器
4. 消息上报频率不宜过高，建议按阶段汇报而非逐条记录汇报

## getMessageHandler - 获取消息处理器

**所属类**：`AbstractTask`
**签名**：`getMessageHandler(): MessageHandler`
**触发时机**：任务执行过程中随时可调用，获取框架注入的消息处理器实例。
**典型用途**：在 `execute` 方法中获取消息处理器，实时上报任务进度百分比、阶段性结果、错误信息。

### 业务场景：批量生成对账单并上报进度

月末批量生成供应商对账单，按供应商逐个处理，通过 `getMessageHandler` 获取消息处理器实时上报处理进度和异常信息。

```typescript
import { AbstractTask } from "@cosmic/bos-core/kd/bos/schedule";
import { BusinessDataServiceHelper, QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { SaveServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper/operation";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";

class ReconciliationTask extends AbstractTask {

  execute(requestContext: any, params: any): void {
    let handler = this.getMessageHandler();

    // 查询需要对账的供应商
    let suppliers = QueryServiceHelper.query(
      "bd_supplier",
      "id,number,name",
      [new QFilter("enable", QCP.equals, "1")]
    );

    let total = suppliers.size();
    let successCount = 0;
    let failCount = 0;

    // 上报任务开始
    if (handler != null) {
      handler.sendMessage("对账单生成任务启动，共" + total + "个供应商待处理");
    }

    for (let i = 0; i < total; i++) {
      let supplier = suppliers.get(i);
      let supplierName = supplier.getString("name");
      let supplierId = supplier.getLong("id");

      try {
        // 查询该供应商本月的采购入库数据
        let inbillData = QueryServiceHelper.query(
          "im_purinstorebill",
          "billentry.material.name,billentry.qty,billentry.amount",
          [
            new QFilter("supplier.id", QCP.equals, supplierId),
            new QFilter("bizdate", QCP.large_equals, this.getMonthStart()),
            new QFilter("bizdate", QCP.less_equals, this.getMonthEnd()),
            new QFilter("billstatus", QCP.equals, "C")
          ]
        );

        // 生成对账单
        let reconciliation = BusinessDataServiceHelper.newDynamicObject("kdec_reconciliation");
        reconciliation.set("kdec_supplier", supplierId);
        reconciliation.set("kdec_period", this.getCurrentPeriod());
        reconciliation.set("kdec_bill_count", inbillData.size());

        let totalAmount = BigDecimal.ZERO;
        for (let row of inbillData) {
          let amt = row.get("billentry.amount") as BigDecimal;
          if (amt != null) {
            totalAmount = totalAmount.add(amt);
          }
        }
        reconciliation.set("kdec_total_amount", totalAmount);

        SaveServiceHelper.saveOperate("kdec_reconciliation", [reconciliation], OperateOption.create());
        successCount++;

      } catch (e: any) {
        failCount++;
        // 上报单个供应商处理异常
        if (handler != null) {
          handler.sendMessage("[异常] 供应商" + supplierName + "对账单生成失败：" + e.getMessage());
        }
      }

      // 每处理10个上报一次进度
      if ((i + 1) % 10 === 0 || i === total - 1) {
        if (handler != null) {
          let progress = Math.round(((i + 1) / total) * 100);
          handler.sendMessage("进度：" + progress + "% (" + (i + 1) + "/" + total + ")，成功" + successCount + "，失败" + failCount);
        }
      }
    }

    // 上报最终结果
    if (handler != null) {
      handler.sendMessage("任务完成：共" + total + "个供应商，成功" + successCount + "，失败" + failCount);
    }
  }

  getMonthStart(): Date {
    let d = new Date();
    return new Date(d.getFullYear(), d.getMonth(), 1);
  }

  getMonthEnd(): Date {
    let d = new Date();
    return new Date(d.getFullYear(), d.getMonth() + 1, 0);
  }

  getCurrentPeriod(): string {
    let d = new Date();
    let month = d.getMonth() + 1;
    return d.getFullYear() + "-" + (month < 10 ? "0" + month : "" + month);
  }
}

let plugin = new ReconciliationTask();
export { plugin };
```

**要点说明**：

1. `getMessageHandler()` 返回值可能为 `null`，每次使用前必须做非空判断
2. 进度上报建议按批次汇报（如每 10 条），避免过于频繁影响性能
3. 异常信息通过消息处理器上报后，运维人员可在调度监控界面查看
4. 消息内容应简洁明了，包含关键数据（进度百分比、成功/失败计数）

## stop - 停止任务

**所属类**：`AbstractTask`
**签名**：`stop(): void`（throws KDException）
**触发时机**：运维人员在调度监控界面手动停止任务，或任务超时被框架强制终止时调用。
**典型用途**：释放资源、记录中断状态、回滚未完成的数据、发送中断通知。

### 业务场景：大批量价格更新任务优雅停止

批量更新物料价格的调度任务，处理数据量大、耗时长。被停止时需要记录已处理的进度断点，下次启动时可从断点续跑，并通知相关负责人任务被中断。

```typescript
import { AbstractTask } from "@cosmic/bos-core/kd/bos/schedule";
import { BusinessDataServiceHelper, QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { SaveServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper/operation";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";
import { MessageInfo } from "@cosmic/bos-core/kd/bos/workflow/engine/msg/info";
import { MessageCenterServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper/workflow";
import { ArrayList } from "@cosmic/bos-script/java/util";
import { LocaleString } from "@cosmic/bos-core/kd/bos/dataentity/entity";

class BatchPriceUpdateTask extends AbstractTask {

  stop(): void {
    let handler = this.getMessageHandler();
    let taskId = this.getTaskId();

    // ---- 1. 记录中断断点 ----
    // 查询当前任务的进度记录
    let progressObj = QueryServiceHelper.queryOne(
      "kdec_task_progress",
      "id,kdec_processed_count,kdec_total_count",
      [new QFilter("kdec_task_id", QCP.equals, taskId)]
    );

    if (progressObj != null) {
      let progress = BusinessDataServiceHelper.loadSingle(progressObj.get("id"), "kdec_task_progress");
      progress.set("kdec_status", "interrupted");
      progress.set("kdec_stop_time", new Date());
      SaveServiceHelper.saveOperate("kdec_task_progress", [progress], OperateOption.create());

      let processed = progressObj.getInt("kdec_processed_count");
      let total = progressObj.getInt("kdec_total_count");

      if (handler != null) {
        handler.sendMessage("任务被中断，已处理" + processed + "/" + total + "，断点已保存");
      }
    }

    // ---- 2. 发送中断通知 ----
    let messageInfo = new MessageInfo();
    messageInfo.setMessageTitle(new LocaleString("价格更新任务中断通知"));
    messageInfo.setMessageContent(new LocaleString(
      "批量价格更新任务(ID:" + taskId + ")已被中断停止，请在调度监控中查看详情并决定是否重新执行。"
    ));
    messageInfo.setMessageType(MessageInfo.TYPE_MESSAGE);

    // 通知系统管理员
    let adminIds = new ArrayList();
    let admins = QueryServiceHelper.query(
      "bos_user",
      "id",
      [new QFilter("usertype", QCP.equals, "admin")]
    );
    for (let admin of admins) {
      adminIds.add(admin.getLong("id"));
    }
    messageInfo.setUserIds(adminIds);

    if (adminIds.size() > 0) {
      MessageCenterServiceHelper.sendMessage(messageInfo);
    }

    // ---- 3. 调用父类 stop 释放框架资源 ----
    super.stop();
  }

  execute(requestContext: any, params: any): void {
    let taskId = this.getTaskId();
    let handler = this.getMessageHandler();

    // 查询是否有上次中断的断点
    let lastProgress = QueryServiceHelper.queryOne(
      "kdec_task_progress",
      "kdec_processed_count",
      [
        new QFilter("kdec_task_name", QCP.equals, "batch_price_update"),
        new QFilter("kdec_status", QCP.equals, "interrupted")
      ]
    );

    let startIndex = 0;
    if (lastProgress != null) {
      startIndex = lastProgress.getInt("kdec_processed_count");
      if (handler != null) {
        handler.sendMessage("检测到上次中断断点，从第" + startIndex + "条继续执行");
      }
    }

    // 查询待更新的物料价格
    let materials = QueryServiceHelper.query(
      "bd_material",
      "id,number,name",
      [new QFilter("enable", QCP.equals, "1")],
      "number"
    );

    let total = materials.size();

    // 创建进度记录
    let progressRecord = BusinessDataServiceHelper.newDynamicObject("kdec_task_progress");
    progressRecord.set("kdec_task_id", taskId);
    progressRecord.set("kdec_task_name", "batch_price_update");
    progressRecord.set("kdec_total_count", total);
    progressRecord.set("kdec_processed_count", startIndex);
    progressRecord.set("kdec_status", "running");
    progressRecord.set("kdec_start_time", new Date());
    SaveServiceHelper.saveOperate("kdec_task_progress", [progressRecord], OperateOption.create());

    // 从断点开始批量处理
    for (let i = startIndex; i < total; i++) {
      let material = materials.get(i);
      let materialId = material.getLong("id");

      // 价格更新逻辑（简化）
      let priceObj = QueryServiceHelper.queryOne(
        "kdec_price_adjust",
        "kdec_new_price",
        [
          new QFilter("kdec_material.id", QCP.equals, materialId),
          new QFilter("kdec_effective_date", QCP.less_equals, new Date())
        ],
        "kdec_effective_date desc"
      );

      if (priceObj != null) {
        let materialData = BusinessDataServiceHelper.loadSingle(materialId, "bd_material");
        materialData.set("kdec_standard_price", priceObj.get("kdec_new_price"));
        SaveServiceHelper.saveOperate("bd_material", [materialData], OperateOption.create());
      }

      // 每100条更新一次进度断点
      if ((i + 1) % 100 === 0) {
        let prog = BusinessDataServiceHelper.loadSingle(progressRecord.get("id"), "kdec_task_progress");
        prog.set("kdec_processed_count", i + 1);
        SaveServiceHelper.saveOperate("kdec_task_progress", [prog], OperateOption.create());
      }
    }

    // 任务完成，更新状态
    let finalProg = BusinessDataServiceHelper.loadSingle(progressRecord.get("id"), "kdec_task_progress");
    finalProg.set("kdec_processed_count", total);
    finalProg.set("kdec_status", "completed");
    finalProg.set("kdec_end_time", new Date());
    SaveServiceHelper.saveOperate("kdec_task_progress", [finalProg], OperateOption.create());

    if (handler != null) {
      handler.sendMessage("价格更新任务完成，共处理" + total + "条物料");
    }
  }
}

let plugin = new BatchPriceUpdateTask();
export { plugin };
```

**要点说明**：

1. `stop` 由调度框架在外部触发停止时调用，重写时必须调用 `super.stop()` 释放框架资源
2. `stop` 可能抛出 `KDException`，框架会捕获并记录
3. 长耗时任务应设计断点续跑机制：定期保存已处理进度，`stop` 时记录中断位置，下次 `execute` 时读取断点
4. `stop` 中不应做耗时操作，应快速完成资源释放和状态记录
5. `stop` 被调用后 `execute` 中的循环不会自动中断，如需配合可通过共享标志位实现（但注意无状态原则）
