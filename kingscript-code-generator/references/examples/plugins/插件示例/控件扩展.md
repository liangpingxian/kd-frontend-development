## addClickListener - 注册点击监听器

**所属接口**：`ISuportClick`
**签名**：`addClickListener(listener: ClickListener): void`
**触发控件**：Button、自定义控件
**典型用途**：为按钮或自定义控件注册 `ClickListener`，使插件能响应 `beforeClick` 和 `click` 事件。

### 业务场景：自定义"一键生成入库单"按钮

采购订单表单上放置一个自定义按钮"一键生成入库单"，通过 `addClickListener` 注册点击监听，点击后校验单据状态，然后调用下推生成入库单。

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { Button } from "@cosmic/bos-core/kd/bos/form/control";
import { DynamicObject } from "@cosmic/bos-core/kd/bos/dataentity/entity";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { OperationServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper/operation";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";
import { BillShowParameter, ShowType } from "@cosmic/bos-core/kd/bos/form";

class PurOrderFormPlugin extends AbstractBillPlugIn {

  registerListener(e: $.java.util.EventObject): void {
    // 通过 addClickListener 为按钮注册点击监听
    let btn = this.getView().getControl("kdec_btn_gen_inbill") as Button;
    btn.addClickListener(this);
  }

  beforeClick(evt: any): void {
    if (evt.getSource() instanceof Button && evt.getSource().getKey() === "kdec_btn_gen_inbill") {
      // 点击前校验：单据必须已审核
      let billStatus = this.getModel().getValue("billstatus") as string;
      if (billStatus !== "C") {
        this.getView().showErrorNotification("单据未审核，不允许生成入库单");
        evt.setCancel(true);
        return;
      }

      // 校验是否已生成过入库单
      let billNo = this.getModel().getValue("billno") as string;
      let exists = QueryServiceHelper.exists(
        "im_purinstorebill",
        [new QFilter("kdec_source_billno", QCP.equals, billNo)]
      );
      if (exists) {
        this.getView().showErrorNotification("该订单已生成入库单，请勿重复操作");
        evt.setCancel(true);
      }
    }
  }

  click(evt: any): void {
    if (evt.getSource() instanceof Button) {
      let btn = evt.getSource() as Button;
      if (btn.getKey() === "kdec_btn_gen_inbill") {
        // 执行下推生成入库单
        let billId = this.getModel().getDataEntity(false).get("id");
        let result = OperationServiceHelper.executeOperate(
          "push", "pm_purorderbill", [billId], OperateOption.create()
        );

        if (result.isSuccess()) {
          this.getView().showSuccessNotification("入库单生成成功");
        } else {
          let msg = result.getMessage() || "生成失败";
          this.getView().showErrorNotification(msg);
        }
      }
    }
  }
}

let plugin = new PurOrderFormPlugin();
export { plugin };
```

**要点说明**：

1. `addClickListener(this)` 将插件自身注册为监听器，插件需实现 `beforeClick` 和 `click` 方法
2. 即使 `beforeClick` 方法体为空，也必须声明，否则可能报异常
3. 在 `beforeClick` 中通过 `evt.setCancel(true)` 阻止后续 `click` 执行
4. 通过 `evt.getSource().getKey()` 区分是哪个按钮触发的事件

## addItemClickListener - 注册菜单点击监听器

**所属接口**：`ISuportClick`
**签名**：`addItemClickListener(listener: ItemClickListener): void`
**触发控件**：Toolbar、自定义控件
**典型用途**：为工具栏或自定义控件注册 `ItemClickListener`，使插件能响应 `beforeItemClick` 和 `itemClick` 事件。

### 业务场景：工具栏自定义"批量审核"和"导出报表"菜单

付款单表单工具栏上有"批量审核"和"导出报表"两个自定义菜单，通过 `addItemClickListener` 注册监听，分别处理批量审核逻辑和报表导出逻辑。

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { Toolbar } from "@cosmic/bos-core/kd/bos/form/control";
import { DynamicObject } from "@cosmic/bos-core/kd/bos/dataentity/entity";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";

class PayBillFormPlugin extends AbstractBillPlugIn {

  registerListener(e: $.java.util.EventObject): void {
    // 通过 addItemClickListener 为工具栏注册菜单点击监听
    let toolbar = this.getView().getControl("tbmain") as Toolbar;
    toolbar.addItemClickListener(this);
  }

  beforeItemClick(evt: any): void {
    let itemKey = evt.getItemKey();

    if (itemKey === "kdec_batch_audit") {
      // 批量审核前校验：付款金额不能为零
      let payAmount = this.getModel().getValue("kdec_pay_amount") as BigDecimal;
      if (payAmount == null || payAmount.compareTo(BigDecimal.ZERO) <= 0) {
        this.getView().showErrorNotification("付款金额必须大于零");
        evt.setCancel(true);
        return;
      }

      // 校验关联的采购订单是否都已审核
      let billId = this.getModel().getDataEntity(false).get("id");
      let unauditedCount = QueryServiceHelper.queryOne(
        "kdec_pay_order_link",
        "count(id) as cnt",
        [
          new QFilter("kdec_paybill.id", QCP.equals, billId),
          new QFilter("kdec_order_status", QCP.not_equals, "C")
        ]
      );

      if (unauditedCount != null && unauditedCount.getInt("cnt") > 0) {
        this.getView().showErrorNotification("存在未审核的关联采购订单，请先完成订单审核");
        evt.setCancel(true);
      }
    }

    if (itemKey === "kdec_export_report") {
      // 导出前校验：单据必须已保存
      let billNo = this.getModel().getValue("billno") as string;
      if (billNo == null || billNo === "") {
        this.getView().showTipNotification("请先保存单据再导出");
        evt.setCancel(true);
      }
    }
  }

  itemClick(evt: any): void {
    let itemKey = evt.getItemKey();

    if (itemKey === "kdec_batch_audit") {
      // 执行审核操作
      let auditOp = OperateOption.create();
      auditOp.setVariableValue(OperateOptionConst.ISSHOWMESSAGE, "false");
      let result = this.getView().invokeOperation("audit", auditOp);

      if (result.isSuccess()) {
        this.getView().showSuccessNotification("审核通过");
      } else {
        this.getView().showErrorNotification("审核失败：" + result.getMessage());
      }
    }

    if (itemKey === "kdec_export_report") {
      // 查询付款明细用于导出
      let billId = this.getModel().getDataEntity(false).get("id");
      let details = QueryServiceHelper.query(
        "kdec_paybill",
        "billentry.supplier.name,billentry.payamount,billentry.paycurrency.name",
        [new QFilter("id", QCP.equals, billId)]
      );
      this.getView().showSuccessNotification("报表已生成，共 " + details.size() + " 条明细");
    }
  }
}

let plugin = new PayBillFormPlugin();
export { plugin };
```

**要点说明**：

1. `addItemClickListener(this)` 将插件自身注册为菜单点击监听器
2. `beforeItemClick` 在 `itemClick` 之前执行，用于校验和拦截
3. 通过 `evt.getItemKey()` 区分不同菜单项，**注意 ItemKey 严格区分大小写**，写错会导致不进入预期逻辑
4. 同一个 Toolbar 上多个菜单共享一个监听器，在事件方法中用 `itemKey` 分支处理
5. 多数表单控件不应使用 `addItemClickListener`，该方法主要用于 Toolbar，误用在其他控件上是常见错误
