# 大任务插件（Task Plugin）开发案例集

---

## 1. 类定义 - AbstractTaskClick

### 基本信息

| 属性 | 说明 |
|------|------|
| 类定义 | `public class AbstractTaskClick implements TaskClick` |
| 导入路径 | `@cosmic/bos-core/kd/bos/form/control/plugin` |
| 实现接口 | TaskClick |
| 用途 | 大任务插件的基类，用于处理大任务面板中任务节点的点击交互和生命周期管理 |

### 说明

`AbstractTaskClick` 是大任务插件的基类，实现了 `TaskClick` 接口。大任务（Job）是苍穹平台中用于引导用户按步骤完成复杂业务流程的功能，每个步骤称为一个「任务节点」。通过继承此基类，可以自定义任务节点被点击时的行为，如打开指定单据、执行校验、跳转页面等。

### 业务场景

创建一个采购业务大任务插件，引导用户按步骤完成采购全流程：选择供应商 → 创建采购订单 → 确认入库 → 生成应付单。每个任务节点点击时打开对应的业务单据，并根据上下文自动携带参数。

### 完整示例代码

```typescript
import { AbstractTaskClick } from "@cosmic/bos-core/kd/bos/form/control/plugin";
import { ClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { ItemClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { ShowFormHelper } from "@cosmic/bos-core/kd/bos/form";
import { ShowType } from "@cosmic/bos-core/kd/bos/form";
import { BillShowParameter } from "@cosmic/bos-core/kd/bos/form";

/**
 * 采购全流程大任务插件
 *
 * 大任务插件生命周期：
 * 1. setContext - 初始化上下文（主界面视图、任务配置、任务ID）
 * 2. click     - 用户点击任务节点时触发
 * 3. itemClick - 用户点击任务节点内的菜单项时触发
 * 4. release   - 任务面板关闭时释放资源
 */
class PurchaseProcessTaskPlugin extends AbstractTaskClick {

  setContext(mainView: any, jobFormInfo: any, taskId: string): void {
    super.setContext(mainView, jobFormInfo, taskId);
  }

  click(e: ClickEventArgs): void {
    // 获取当前任务节点的关键信息
    const mainView = this.getMainView();
    const taskId = this.getTaskId();
    const jobFormInfo = this.getJobFormInfo();

    // 根据任务ID判断当前步骤
    if (taskId === "task_select_supplier") {
      // 步骤1：打开供应商选择列表
      const showParam = new BillShowParameter();
      showParam.setFormId("bd_supplier");
      showParam.getOpenStyle().setShowType(ShowType.Modal);
      mainView.showForm(showParam);

    } else if (taskId === "task_create_purorder") {
      // 步骤2：新建采购订单
      const showParam = new BillShowParameter();
      showParam.setFormId("pm_purorderbill");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (taskId === "task_confirm_instock") {
      // 步骤3：打开入库单列表
      const showParam = new BillShowParameter();
      showParam.setFormId("im_instock");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      mainView.showForm(showParam);

    } else if (taskId === "task_create_paybill") {
      // 步骤4：新建应付单
      const showParam = new BillShowParameter();
      showParam.setFormId("ap_paybill");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);
    }
  }

  itemClick(e: ItemClickEventArgs): void {
    // 处理任务节点内的菜单项点击
    const itemKey = e.getItemKey();

    if (itemKey === "viewhelp") {
      this.getMainView().showTipNotification("请按步骤依次完成采购流程");
    }
  }

  release(): boolean {
    // 释放资源，返回 true 表示释放成功
    return true;
  }
}

let plugin = new PurchaseProcessTaskPlugin();
export { plugin };
```

### 注意事项

- 所有大任务插件必须继承 `AbstractTaskClick`，不要直接实现 `TaskClick` 接口
- 插件类中**禁止定义类属性**，只能使用方法内局部变量
- 通过 `this.getMainView()` 获取主界面视图，进行页面跳转和消息提示
- 通过 `this.getTaskId()` 获取当前任务节点 ID，区分不同步骤的处理逻辑
- 通过 `this.getJobFormInfo()` 获取大任务表单的配置信息
- 文件末尾必须导出 plugin 单例：`let plugin = new XxxPlugin(); export { plugin };`

## 2. 接口定义 - TaskClick

### 基本信息

| 属性 | 说明 |
|------|------|
| 接口定义 | `public interface TaskClick` |
| 导入路径 | `@cosmic/bos-core/kd/bos/form/control/plugin` |
| 用途 | 定义大任务插件的事件契约，包含上下文初始化、点击处理、资源释放等方法 |

### 说明

`TaskClick` 是大任务插件的核心接口，定义了任务节点交互的完整契约。接口包含四个方法：`setContext`（上下文初始化）、`click`（节点点击）、`itemClick`（菜单项点击）、`release`（资源释放）。在实际开发中通常继承 `AbstractTaskClick` 基类，而非直接实现此接口。

### 接口方法一览

| 方法 | 签名 | 用途 |
|------|------|------|
| setContext | `setContext(mainView, jobFormInfo, taskId): void` | 初始化插件上下文 |
| click | `click(e: ClickEventArgs): void` | 处理任务节点点击 |
| itemClick | `itemClick(e: ItemClickEventArgs): void` | 处理菜单项点击 |
| release | `release(): boolean` | 释放资源 |

### 业务场景

创建一个费用报销大任务插件，完整实现 `TaskClick` 接口的所有方法：初始化时加载报销单配置，点击任务节点时打开不同类型的报销单，菜单项点击时查看审批进度，关闭时清理临时数据。

### 完整示例代码

```typescript
import { AbstractTaskClick } from "@cosmic/bos-core/kd/bos/form/control/plugin";
import { ClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { ItemClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { BillShowParameter, ShowType } from "@cosmic/bos-core/kd/bos/form";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";

/**
 * 费用报销大任务插件 - 演示 TaskClick 接口的完整实现
 *
 * TaskClick 接口定义了大任务插件的4个核心方法：
 * - setContext:  上下文初始化，获取主界面、任务配置、任务ID
 * - click:      任务节点被点击时的处理
 * - itemClick:  任务节点内菜单按钮被点击时的处理
 * - release:    任务面板关闭时释放资源
 */
class ExpenseReimbTaskPlugin extends AbstractTaskClick {

  // 接口方法1：上下文初始化
  setContext(mainView: any, jobFormInfo: any, taskId: string): void {
    super.setContext(mainView, jobFormInfo, taskId);
    // setContext 是最先触发的方法，此时可获取上下文信息
    // mainView: 主界面视图，用于页面跳转和消息提示
    // jobFormInfo: 大任务表单配置信息
    // taskId: 当前任务节点的唯一标识
  }

  // 接口方法2：任务节点点击
  click(e: ClickEventArgs): void {
    const taskId = this.getTaskId();
    const mainView = this.getMainView();
    const currUserId = RequestContext.get().getCurrUserId();

    if (taskId === "task_travel_expense") {
      // 差旅报销：打开差旅费报销单新增页
      const showParam = new BillShowParameter();
      showParam.setFormId("er_travelexpbill");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (taskId === "task_daily_expense") {
      // 日常报销：打开日常费用报销单新增页
      const showParam = new BillShowParameter();
      showParam.setFormId("er_dailyexpbill");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (taskId === "task_check_status") {
      // 查看审批状态：查询当前用户待审批的报销单数量
      const pendingCount = QueryServiceHelper.queryOne(
        "er_expensebill",
        "count(id) as cnt",
        [
          new QFilter("creator", "=", currUserId),
          new QFilter("billstatus", "=", "B")
        ]
      );

      if (pendingCount != null) {
        const cnt = pendingCount.get("cnt");
        mainView.showTipNotification("您有 " + cnt + " 张报销单正在审批中");
      }
    }
  }

  // 接口方法3：菜单项点击
  itemClick(e: ItemClickEventArgs): void {
    const itemKey = e.getItemKey();
    const mainView = this.getMainView();

    if (itemKey === "viewhistory") {
      // 查看报销历史
      const showParam = new BillShowParameter();
      showParam.setFormId("er_expensebill_list");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      mainView.showForm(showParam);

    } else if (itemKey === "viewpolicy") {
      // 查看报销制度
      mainView.showTipNotification("请参考公司报销制度文档（OA公告栏）");
    }
  }

  // 接口方法4：释放资源
  release(): boolean {
    // 任务面板关闭时调用，释放插件持有的资源
    // 返回 true 表示释放成功，false 表示释放失败
    return true;
  }
}

let plugin = new ExpenseReimbTaskPlugin();
export { plugin };
```

### 注意事项

- `TaskClick` 接口定义了大任务插件的完整契约，实际开发中继承 `AbstractTaskClick` 基类即可
- 四个方法的调用顺序：`setContext` → `click`/`itemClick`（用户交互时） → `release`（面板关闭时）
- `setContext` 是初始化入口，在任务面板加载时自动调用，不需要手动触发
- `release` 必须返回 `boolean`，返回 `true` 表示资源释放成功

## 3. setContext - 上下文初始化

### 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | TaskClick |
| 插件基类 | AbstractTaskClick |
| 触发时机 | 大任务面板加载时，插件实例化后最先调用 |
| 方法签名 | `setContext(mainView: IFormView, jobFormInfo: JobFormInfo, taskId: string): void` |

### 说明

`setContext` 是大任务插件最先被调用的方法，系统在加载任务面板时自动调用。通过此方法将主界面视图、大任务配置信息和当前任务节点 ID 传入插件，供后续 `click` 和 `itemClick` 使用。调用 `super.setContext()` 后，可通过 `this.getMainView()`、`this.getJobFormInfo()`、`this.getTaskId()` 获取这些上下文。

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| mainView | IFormView | 主界面视图对象，用于页面跳转、消息提示等 |
| jobFormInfo | JobFormInfo | 大任务表单配置信息，包含任务节点定义 |
| taskId | string | 当前任务节点的唯一标识 |

### 业务场景

在合同管理大任务中，初始化时根据任务节点 ID 和当前用户权限，预判该用户是否有权限操作当前步骤，并根据合同类型动态加载不同的任务配置参数。

### 完整示例代码

```typescript
import { AbstractTaskClick } from "@cosmic/bos-core/kd/bos/form/control/plugin";
import { ClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { ItemClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";
import { BillShowParameter, ShowType } from "@cosmic/bos-core/kd/bos/form";

/**
 * 合同管理大任务插件 - 演示 setContext 上下文初始化
 */
class ContractTaskPlugin extends AbstractTaskClick {

  setContext(mainView: any, jobFormInfo: any, taskId: string): void {
    super.setContext(mainView, jobFormInfo, taskId);

    // setContext 是最先执行的方法，适合做初始化校验
    // 此时 this.getMainView()、this.getTaskId() 已可用

    // 根据任务节点判断是否需要权限校验
    if (taskId === "task_contract_approve") {
      // 查询当前用户是否具有合同审批权限
      const currUserId = RequestContext.get().getCurrUserId();

      const hasPermission = QueryServiceHelper.queryOne(
        "bos_user_role",
        "id",
        [
          new QFilter("user", "=", currUserId),
          new QFilter("role.number", "=", "contract_approver")
        ]
      );

      if (hasPermission == null) {
        mainView.showTipNotification("您没有合同审批权限，请联系管理员");
      }
    }
  }

  click(e: ClickEventArgs): void {
    const taskId = this.getTaskId();
    const mainView = this.getMainView();

    if (taskId === "task_contract_create") {
      // 新建合同
      const showParam = new BillShowParameter();
      showParam.setFormId("conm_contractbill");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (taskId === "task_contract_approve") {
      // 打开待审批合同列表
      const showParam = new BillShowParameter();
      showParam.setFormId("conm_contractbill_list");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      mainView.showForm(showParam);
    }
  }

  itemClick(e: ItemClickEventArgs): void {
    // 无菜单项处理
  }

  release(): boolean {
    return true;
  }
}

let plugin = new ContractTaskPlugin();
export { plugin };
```

### 注意事项

- `setContext` 是插件最先执行的方法，在 `click`/`itemClick` 之前调用
- 必须调用 `super.setContext(mainView, jobFormInfo, taskId)`，否则后续 `this.getMainView()` 等方法返回 null
- 适合在此方法中做权限预检、配置预加载等初始化操作
- 不要在此方法中做耗时操作（如大量数据查询），否则会影响任务面板加载速度
- 插件类中禁止定义类属性存储上下文，`super.setContext()` 已帮你保存

## 4. click - 任务节点点击事件

### 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | TaskClick |
| 插件基类 | AbstractTaskClick |
| 触发时机 | 用户点击大任务面板中的任务节点时触发 |
| 方法签名 | `click(e: ClickEventArgs): void` |

### 说明

当用户在大任务面板上点击某个任务节点时，系统调用该节点绑定的插件的 `click` 方法。通过 `ClickEventArgs` 可获取点击事件的相关信息，结合 `this.getTaskId()` 判断当前节点，执行对应的业务逻辑（如打开单据、跳转页面、执行查询等）。

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | ClickEventArgs | 点击事件参数对象 |

### 业务场景

在生产制造大任务中，用户按流程执行：创建生产工单 → 领料出库 → 工序汇报 → 产品入库。每个节点点击时检查前置条件是否满足，满足则打开对应单据，不满足则提示用户先完成前置步骤。

### 完整示例代码

```typescript
import { AbstractTaskClick } from "@cosmic/bos-core/kd/bos/form/control/plugin";
import { ClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { ItemClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";
import { BillShowParameter, ShowType } from "@cosmic/bos-core/kd/bos/form";

/**
 * 生产制造大任务插件 - 演示 click 任务节点点击处理
 */
class ManufactureTaskPlugin extends AbstractTaskClick {

  setContext(mainView: any, jobFormInfo: any, taskId: string): void {
    super.setContext(mainView, jobFormInfo, taskId);
  }

  click(e: ClickEventArgs): void {
    const taskId = this.getTaskId();
    const mainView = this.getMainView();

    if (taskId === "task_create_workorder") {
      // 步骤1：创建生产工单（无前置条件）
      const showParam = new BillShowParameter();
      showParam.setFormId("mfg_workorder");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (taskId === "task_material_issue") {
      // 步骤2：领料出库 - 检查是否有已审核的生产工单
      const hasWorkOrder = QueryServiceHelper.queryOne(
        "mfg_workorder",
        "id",
        [
          new QFilter("billstatus", QCP.equals, "C"),
          new QFilter("closestatus", QCP.not_equals, "B")
        ]
      );

      if (hasWorkOrder == null) {
        mainView.showErrorNotification("请先创建并审核生产工单");
        return;
      }

      const showParam = new BillShowParameter();
      showParam.setFormId("im_materialissue");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (taskId === "task_process_report") {
      // 步骤3：工序汇报 - 检查是否已领料
      const hasIssued = QueryServiceHelper.queryOne(
        "im_materialissue",
        "id",
        [
          new QFilter("billstatus", QCP.equals, "C"),
          new QFilter("billtype", QCP.equals, "production")
        ]
      );

      if (hasIssued == null) {
        mainView.showErrorNotification("请先完成领料出库");
        return;
      }

      const showParam = new BillShowParameter();
      showParam.setFormId("mfg_processreport");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (taskId === "task_product_instock") {
      // 步骤4：产品入库 - 检查是否有已汇报的工序
      const hasReported = QueryServiceHelper.queryOne(
        "mfg_processreport",
        "id",
        [new QFilter("billstatus", QCP.equals, "C")]
      );

      if (hasReported == null) {
        mainView.showErrorNotification("请先完成工序汇报");
        return;
      }

      const showParam = new BillShowParameter();
      showParam.setFormId("im_productinstock");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);
    }
  }

  itemClick(e: ItemClickEventArgs): void {
    // 无菜单项处理
  }

  release(): boolean {
    return true;
  }
}

let plugin = new ManufactureTaskPlugin();
export { plugin };
```

### 注意事项

- `click` 在用户点击任务节点时触发，是大任务插件最核心的方法
- 通过 `this.getTaskId()` 区分不同任务节点，执行对应逻辑
- 建议在打开后续步骤前校验前置步骤是否完成，给用户清晰的引导
- 使用 `BillShowParameter` 配合 `mainView.showForm()` 打开单据页面
- 页面打开方式通过 `ShowType` 控制：`Modal`（弹窗）、`MainNewTabPage`（新标签页）等
- 避免在 `click` 中做大量数据处理，保持响应速度

## 5. itemClick - 菜单项点击事件

### 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | TaskClick |
| 插件基类 | AbstractTaskClick |
| 触发时机 | 用户点击任务节点中的菜单按钮/操作项时触发 |
| 方法签名 | `itemClick(e: ItemClickEventArgs): void` |

### 说明

当任务节点包含下拉菜单或操作按钮时，用户点击菜单中的某一项会触发 `itemClick` 方法。与 `click`（点击整个任务节点）不同，`itemClick` 处理的是节点内部的细分操作。通过 `e.getItemKey()` 获取被点击菜单项的标识，执行对应的业务逻辑。

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | ItemClickEventArgs | 菜单项点击事件参数 |

### 常用方法

| 方法 | 返回类型 | 说明 |
|------|---------|------|
| e.getItemKey() | string | 获取被点击菜单项的标识 |

### 业务场景

在销售业务大任务中，「客户管理」任务节点提供多个操作菜单：新建客户、查看客户列表、导入客户、查看客户信用额度。用户点击不同菜单项时执行不同的业务操作。

### 完整示例代码

```typescript
import { AbstractTaskClick } from "@cosmic/bos-core/kd/bos/form/control/plugin";
import { ClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { ItemClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { BillShowParameter, ShowType, ListShowParameter } from "@cosmic/bos-core/kd/bos/form";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";

/**
 * 销售业务大任务插件 - 演示 itemClick 菜单项点击处理
 */
class SalesCustomerTaskPlugin extends AbstractTaskClick {

  setContext(mainView: any, jobFormInfo: any, taskId: string): void {
    super.setContext(mainView, jobFormInfo, taskId);
  }

  click(e: ClickEventArgs): void {
    const mainView = this.getMainView();

    // 点击任务节点本身，默认打开客户列表
    const showParam = new ListShowParameter();
    showParam.setFormId("bd_customer_list");
    showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
    mainView.showForm(showParam);
  }

  itemClick(e: ItemClickEventArgs): void {
    const itemKey = e.getItemKey();
    const mainView = this.getMainView();

    if (itemKey === "newcustomer") {
      // 菜单项：新建客户
      const showParam = new BillShowParameter();
      showParam.setFormId("bd_customer");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (itemKey === "customerlist") {
      // 菜单项：查看客户列表
      const showParam = new ListShowParameter();
      showParam.setFormId("bd_customer_list");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      mainView.showForm(showParam);

    } else if (itemKey === "importcustomer") {
      // 菜单项：导入客户（打开引入界面）
      const showParam = new BillShowParameter();
      showParam.setFormId("bos_import");
      showParam.getOpenStyle().setShowType(ShowType.Modal);
      mainView.showForm(showParam);

    } else if (itemKey === "creditcheck") {
      // 菜单项：查看客户信用额度汇总
      const currOrgId = RequestContext.get().getOrgId();

      const creditInfo = QueryServiceHelper.queryOne(
        "bd_customer_credit",
        "sum(creditamount) as totalcredit,sum(usedamount) as totalused",
        [
          new QFilter("org", "=", currOrgId),
          new QFilter("enable", "=", "1")
        ]
      );

      if (creditInfo != null) {
        const totalCredit = creditInfo.get("totalcredit") as BigDecimal;
        const totalUsed = creditInfo.get("totalused") as BigDecimal;
        const available = totalCredit.subtract(totalUsed).setScale(2, BigDecimal.ROUND_HALF_UP);

        mainView.showTipNotification(
          "信用总额：" + totalCredit.toString() +
          "，已用：" + totalUsed.toString() +
          "，可用：" + available.toString()
        );
      } else {
        mainView.showTipNotification("暂无客户信用额度数据");
      }
    }
  }

  release(): boolean {
    return true;
  }
}

let plugin = new SalesCustomerTaskPlugin();
export { plugin };
```

### 注意事项

- `itemClick` 与 `click` 的区别：`click` 是点击任务节点本身，`itemClick` 是点击节点内的菜单项
- 通过 `e.getItemKey()` 获取菜单项标识，标识在大任务配置中定义
- 菜单项标识（itemKey）需与大任务面板配置中的菜单定义一致
- 如果任务节点没有菜单项，`itemClick` 不会被触发，但仍需实现（空方法体即可）
- 建议将复杂查询逻辑抽离为独立方法，保持 `itemClick` 方法简洁

## 6. release - 资源释放

### 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | TaskClick |
| 插件基类 | AbstractTaskClick |
| 触发时机 | 大任务面板关闭或切换时，释放插件持有的资源 |
| 方法签名 | `release(): boolean` |

### 说明

`release` 是大任务插件生命周期的最后一个方法，在任务面板关闭或任务切换时由系统调用。用于清理插件运行过程中产生的临时资源（如缓存数据、临时文件引用等）。返回 `true` 表示资源释放成功，返回 `false` 表示释放失败（系统会记录日志）。

### 返回值说明

| 返回值 | 类型 | 说明 |
|-------|------|------|
| true | boolean | 资源释放成功 |
| false | boolean | 资源释放失败，系统记录日志 |

### 业务场景

在项目管理大任务中，任务面板打开期间通过 `AppCache` 缓存了项目进度数据用于多节点间共享。当任务面板关闭时，需要清理这些缓存数据，避免占用缓存空间和数据过期问题。

### 完整示例代码

```typescript
import { AbstractTaskClick } from "@cosmic/bos-core/kd/bos/form/control/plugin";
import { ClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { ItemClickEventArgs } from "@cosmic/bos-core/kd/bos/form/control/events";
import { BillShowParameter, ShowType } from "@cosmic/bos-core/kd/bos/form";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";
import { AppCache } from "@cosmic/bos-core/kd/bos/cache";

/**
 * 项目管理大任务插件 - 演示 release 资源释放
 */
class ProjectTaskPlugin extends AbstractTaskClick {

  setContext(mainView: any, jobFormInfo: any, taskId: string): void {
    super.setContext(mainView, jobFormInfo, taskId);

    // 初始化时将项目进度数据写入应用缓存，供多节点共享
    const currUserId = RequestContext.get().getCurrUserId();
    const cacheKey = "proj_task_" + currUserId;

    const projectStats = QueryServiceHelper.queryOne(
      "proj_project",
      "count(id) as total,sum(case when status='C' then 1 else 0 end) as completed",
      [
        new QFilter("manager", "=", currUserId),
        new QFilter("closestatus", "!=", "B")
      ]
    );

    if (projectStats != null) {
      const cacheData = projectStats.get("total") + "|" + projectStats.get("completed");
      AppCache.get("proj_task").put(cacheKey, cacheData);
    }
  }

  click(e: ClickEventArgs): void {
    const taskId = this.getTaskId();
    const mainView = this.getMainView();

    if (taskId === "task_project_create") {
      // 新建项目
      const showParam = new BillShowParameter();
      showParam.setFormId("proj_project");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      showParam.setStatus("addnew");
      mainView.showForm(showParam);

    } else if (taskId === "task_project_progress") {
      // 查看项目进度 - 从缓存读取
      const currUserId = RequestContext.get().getCurrUserId();
      const cacheKey = "proj_task_" + currUserId;
      const cacheData = AppCache.get("proj_task").get(cacheKey) as string;

      if (cacheData != null) {
        const parts = cacheData.split("|");
        mainView.showTipNotification(
          "项目总数：" + parts[0] + "，已完成：" + parts[1]
        );
      }

      // 打开项目列表
      const showParam = new BillShowParameter();
      showParam.setFormId("proj_project_list");
      showParam.getOpenStyle().setShowType(ShowType.MainNewTabPage);
      mainView.showForm(showParam);
    }
  }

  itemClick(e: ItemClickEventArgs): void {
    // 无菜单项处理
  }

  release(): boolean {
    // 清理缓存数据
    try {
      const currUserId = RequestContext.get().getCurrUserId();
      const cacheKey = "proj_task_" + currUserId;

      // 移除应用缓存中的临时数据
      AppCache.get("proj_task").remove(cacheKey);

      return true;
    } catch (error: any) {
      // 释放失败，返回 false 让系统记录日志
      return false;
    }
  }
}

let plugin = new ProjectTaskPlugin();
export { plugin };
```

### 注意事项

- `release` 是插件生命周期的最后一个方法，在此之后插件实例不再可用
- 必须返回 `boolean`：`true` 表示成功，`false` 表示失败
- 适合清理的资源：`AppCache` 缓存数据、`PageCache` 页面缓存、临时标记等
- 不要在 `release` 中执行数据库写入操作（如保存单据），此时已脱离业务上下文
- 即使没有资源需要清理，也要实现此方法并返回 `true`
- 建议用 `try-catch` 包裹清理逻辑，避免异常导致系统错误
