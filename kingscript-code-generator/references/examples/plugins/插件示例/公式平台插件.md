## getFuncInfo - 获取自定义函数信息

**所属接口**：`IFuncParamInputFormPlugin`（extends `IFormulaPlugin`）
**签名**：`getFuncInfo(): FuncInfo`
**触发时机**：公式平台加载自定义函数时调用，用于向公式引擎注册函数的元信息。
**典型用途**：定义函数名称、参数列表、返回值类型、函数描述等，使自定义函数在公式编辑器中可被选择和使用。

### 业务场景：注册"计算含税金额"自定义公式函数

在公式平台注册一个 `CALC_TAX_AMOUNT` 函数，接收不含税金额和税率两个参数，返回含税金额。业务人员可在单据公式中直接调用该函数进行税额计算。

```typescript
class CalcTaxAmountFormulaPlugin implements IFuncParamInputFormPlugin {

  getFuncInfo(): FuncInfo {
    let funcInfo = new FuncInfo();

    // ---- 1. 基本信息 ----
    funcInfo.setName("CALC_TAX_AMOUNT");
    funcInfo.setCaption("计算含税金额");
    funcInfo.setDescription("根据不含税金额和税率计算含税金额。公式：不含税金额 × (1 + 税率/100)");
    funcInfo.setCategory("财务计算");

    // ---- 2. 定义参数列表 ----
    let params = new ArrayList();

    // 参数1：不含税金额
    let paramAmount = new FuncParam();
    paramAmount.setName("amount");
    paramAmount.setCaption("不含税金额");
    paramAmount.setDataType("Decimal");
    paramAmount.setRequired(true);
    paramAmount.setDescription("不含税的原始金额");
    params.add(paramAmount);

    // 参数2：税率
    let paramTaxRate = new FuncParam();
    paramTaxRate.setName("taxRate");
    paramTaxRate.setCaption("税率(%)");
    paramTaxRate.setDataType("Decimal");
    paramTaxRate.setRequired(true);
    paramTaxRate.setDescription("税率百分比，如13表示13%");
    params.add(paramTaxRate);

    funcInfo.setParams(params);

    // ---- 3. 返回值类型 ----
    funcInfo.setReturnType("Decimal");

    return funcInfo;
  }

  getDesignerParameter(): FormulaDesignerParameter {
    return null;
  }
}

let plugin = new CalcTaxAmountFormulaPlugin();
export { plugin };
```

**要点说明**：

1. `FuncInfo` 是函数的元数据描述，公式引擎通过它注册和识别自定义函数
2. `setName` 设置的函数名即为公式中调用时使用的标识符，建议大写加下划线命名
3. `setCategory` 用于在公式编辑器中分组展示，方便用户查找
4. 每个 `FuncParam` 描述一个参数的名称、类型、是否必填等信息
5. 金额相关参数的 `dataType` 应设置为 `Decimal`，确保精度

## getDesignerParameter - 获取公式设计器参数

**所属接口**：`IFuncParamInputFormPlugin`（extends `IFormulaPlugin`）
**签名**：`getDesignerParameter(): FormulaDesignerParameter`
**触发时机**：公式设计器打开自定义函数的参数录入界面时调用。
**典型用途**：配置自定义函数在公式设计器中的参数输入表单，包括指定参数录入界面的表单 ID、预设默认值、控制参数的可选范围等。

### 业务场景：配置"跨单据取数"函数的设计器参数面板

自定义一个 `CROSS_BILL_QUERY` 函数，用于在公式中跨单据查询数据。在公式设计器中，用户需要选择目标单据类型、指定查询字段和过滤条件。通过 `getDesignerParameter` 配置专用的参数录入面板。

```typescript
class CrossBillQueryFormulaPlugin implements IFuncParamInputFormPlugin {

  getFuncInfo(): FuncInfo {
    let funcInfo = new FuncInfo();
    funcInfo.setName("CROSS_BILL_QUERY");
    funcInfo.setCaption("跨单据取数");
    funcInfo.setDescription("从指定单据中查询符合条件的字段值，支持单值和汇总取数");
    funcInfo.setCategory("数据查询");

    let params = new ArrayList();

    // 参数1：目标单据标识
    let paramEntity = new FuncParam();
    paramEntity.setName("entityName");
    paramEntity.setCaption("目标单据");
    paramEntity.setDataType("String");
    paramEntity.setRequired(true);
    paramEntity.setDescription("要查询的目标单据标识，如 pm_purorderbill");
    params.add(paramEntity);

    // 参数2：查询字段
    let paramField = new FuncParam();
    paramField.setName("queryField");
    paramField.setCaption("查询字段");
    paramField.setDataType("String");
    paramField.setRequired(true);
    paramField.setDescription("要返回的字段标识，如 totalamount");
    params.add(paramField);

    // 参数3：过滤值
    let paramFilter = new FuncParam();
    paramFilter.setName("filterValue");
    paramFilter.setCaption("过滤值");
    paramFilter.setDataType("String");
    paramFilter.setRequired(false);
    paramFilter.setDescription("过滤条件的匹配值");
    params.add(paramFilter);

    funcInfo.setParams(params);
    funcInfo.setReturnType("Decimal");

    return funcInfo;
  }

  getDesignerParameter(): FormulaDesignerParameter {
    let designerParam = new FormulaDesignerParameter();

    // ---- 1. 指定参数录入界面 ----
    // 设置自定义的参数录入表单，用户在公式设计器中点击该函数时弹出此表单
    designerParam.setFormId("kdec_crossbill_param_form");

    // ---- 2. 预设默认参数值 ----
    let defaultParams = new HashMap();
    defaultParams.put("entityName", "pm_purorderbill");
    defaultParams.put("queryField", "totalamount");
    defaultParams.put("aggregateType", "SUM");
    designerParam.setDefaultValues(defaultParams);

    // ---- 3. 配置参数选项范围 ----
    // 限制可选的单据类型范围
    let entityOptions = new ArrayList();
    entityOptions.add("pm_purorderbill");   // 采购订单
    entityOptions.add("sm_salorder");       // 销售订单
    entityOptions.add("im_inbill");         // 入库单
    entityOptions.add("ap_paybill");        // 付款单
    designerParam.setCustomParam("entityOptions", entityOptions);

    // 限制可选的汇总方式
    let aggregateOptions = new ArrayList();
    aggregateOptions.add("SUM");
    aggregateOptions.add("AVG");
    aggregateOptions.add("MAX");
    aggregateOptions.add("MIN");
    aggregateOptions.add("COUNT");
    designerParam.setCustomParam("aggregateOptions", aggregateOptions);

    // ---- 4. 设计器显示配置 ----
    designerParam.setCustomParam("showPreview", true);
    designerParam.setCustomParam("helpUrl", "https://help.kingdee.com/formula/cross_bill_query");

    return designerParam;
  }
}

let plugin = new CrossBillQueryFormulaPlugin();
export { plugin };
```

**要点说明**：

1. `getDesignerParameter` 返回 `null` 时，公式设计器使用默认的参数录入方式
2. `setFormId` 可指定一个自定义动态表单作为参数录入界面，适合参数较复杂的场景
3. `setDefaultValues` 预设参数默认值，降低用户配置成本
4. `setCustomParam` 可传递自定义配置项给参数录入表单，控制下拉选项、显示行为等
5. 此方法与 `getFuncInfo` 配合使用：`getFuncInfo` 定义函数元信息，`getDesignerParameter` 定义设计器交互
