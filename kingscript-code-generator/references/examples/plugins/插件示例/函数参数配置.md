## checkSetting - 校验函数参数配置

**所属接口**：`IFuncParamEditor`
**签名**：`checkSetting(msgBuilder: StringBuilder): Boolean`
**触发时机**：用户在公式设计器中配置完函数参数后点击确认时调用，用于校验参数配置的合法性。
**典型用途**：校验用户填写的函数参数是否完整、格式是否正确、参数间逻辑是否一致，不通过时将错误写入 `msgBuilder` 并返回 `false`。

### 业务场景：校验"账龄分析"函数的区间参数配置

自定义一个账龄分析函数，用户需配置账龄区间（如 0-30天、31-60天、61-90天、90天以上）。校验区间不能重叠、不能有间断、起始值必须小于结束值。

```typescript
class AgingAnalysisFuncParamEditor implements IFuncParamEditor {

  checkSetting(msgBuilder: StringBuilder): Boolean {
    // ---- 1. 校验必填项 ----
    let entityName = this.getParamValue("entityName") as string;
    if (entityName == null || entityName === "") {
      msgBuilder.append("请选择应收/应付单据类型；");
      return false;
    }

    let dateField = this.getParamValue("dateField") as string;
    if (dateField == null || dateField === "") {
      msgBuilder.append("请选择日期字段；");
      return false;
    }

    // ---- 2. 校验账龄区间配置 ----
    let intervals = this.getParamValue("agingIntervals") as string;
    if (intervals == null || intervals === "") {
      msgBuilder.append("请至少配置一个账龄区间；");
      return false;
    }

    let isValid = true;
    let ranges = JSON.parse(intervals) as any[];

    if (ranges.length === 0) {
      msgBuilder.append("请至少配置一个账龄区间；");
      return false;
    }

    // 逐个区间校验
    for (let i = 0; i < ranges.length; i++) {
      let start = ranges[i].start as number;
      let end = ranges[i].end as number;

      // 起始值必须 >= 0
      if (start < 0) {
        msgBuilder.append("第" + (i + 1) + "个区间的起始天数不能为负数；");
        isValid = false;
      }

      // 结束值必须大于起始值（最后一个区间可为 -1 表示无上限）
      if (end !== -1 && end <= start) {
        msgBuilder.append("第" + (i + 1) + "个区间的结束天数必须大于起始天数；");
        isValid = false;
      }

      // 相邻区间连续性校验：当前起始 = 上一个结束 + 1
      if (i > 0) {
        let prevEnd = ranges[i - 1].end as number;
        if (prevEnd !== -1 && start !== prevEnd + 1) {
          msgBuilder.append("第" + i + "和第" + (i + 1) + "个区间不连续，存在间断或重叠；");
          isValid = false;
        }
      }
    }

    // 最后一个区间必须为开放区间（end = -1）
    let lastRange = ranges[ranges.length - 1];
    if (lastRange.end !== -1) {
      msgBuilder.append("最后一个区间应为开放区间（如 90天以上）；");
      isValid = false;
    }

    // ---- 3. 校验汇总字段 ----
    let amountField = this.getParamValue("amountField") as string;
    if (amountField == null || amountField === "") {
      msgBuilder.append("请选择金额汇总字段；");
      isValid = false;
    }

    return isValid;
  }

  getSetting(): string {
    return "";
  }
}

let plugin = new AgingAnalysisFuncParamEditor();
export { plugin };
```

**要点说明**：

1. 返回 `true` 表示校验通过，`false` 表示校验失败，系统会将 `msgBuilder` 中的信息展示给用户
2. 建议所有校验规则全部执行完再返回，方便用户一次性修正所有问题
3. 对于数组/JSON 格式的参数，先做非空和格式校验再做业务逻辑校验
4. 区间类参数重点校验：连续性、不重叠、边界值合法

## getSetting - 获取函数参数配置

**所属接口**：`IFuncParamEditor`
**签名**：`getSetting(): String`
**触发时机**：用户在参数编辑器中确认配置后，系统调用此方法获取序列化的参数配置字符串，用于持久化存储。
**典型用途**：将用户在参数编辑界面填写的所有配置项收集、组装为 JSON 字符串返回，供公式引擎运行时解析使用。

### 业务场景：收集"库存预警"函数的参数配置

自定义一个库存预警函数，用户在参数面板配置：目标仓库、物料范围、预警阈值（最低库存、最高库存）、通知方式。`getSetting` 将这些配置序列化为 JSON 返回。

```typescript
class StockAlertFuncParamEditor implements IFuncParamEditor {

  checkSetting(msgBuilder: StringBuilder): Boolean {
    let warehouseId = this.getParamValue("warehouseId") as string;
    if (warehouseId == null || warehouseId === "") {
      msgBuilder.append("请选择目标仓库；");
      return false;
    }

    let minQty = this.getParamValue("minQty") as string;
    let maxQty = this.getParamValue("maxQty") as string;

    if ((minQty == null || minQty === "") && (maxQty == null || maxQty === "")) {
      msgBuilder.append("最低库存和最高库存至少填写一项；");
      return false;
    }

    // 如果两个都填了，最低不能大于最高
    if (minQty != null && minQty !== "" && maxQty != null && maxQty !== "") {
      let minVal = new BigDecimal(minQty);
      let maxVal = new BigDecimal(maxQty);
      if (minVal.compareTo(maxVal) > 0) {
        msgBuilder.append("最低库存不能大于最高库存；");
        return false;
      }
    }

    return true;
  }

  getSetting(): string {
    // ---- 1. 收集基本参数 ----
    let warehouseId = this.getParamValue("warehouseId") as string;
    let warehouseName = this.getParamValue("warehouseName") as string;

    // ---- 2. 收集物料范围 ----
    let materialScope = this.getParamValue("materialScope") as string;  // all / category / specific
    let materialCategoryId = this.getParamValue("materialCategoryId") as string;
    let materialIds = this.getParamValue("materialIds") as string;

    // ---- 3. 收集阈值配置 ----
    let minQty = this.getParamValue("minQty") as string;
    let maxQty = this.getParamValue("maxQty") as string;

    // ---- 4. 收集通知方式 ----
    let notifyType = this.getParamValue("notifyType") as string;       // message / email / both
    let notifyUserIds = this.getParamValue("notifyUserIds") as string;

    // ---- 5. 组装为 JSON 配置 ----
    let setting: any = {};

    setting.warehouse = {
      id: warehouseId,
      name: warehouseName || ""
    };

    setting.materialScope = materialScope || "all";
    if (materialScope === "category" && materialCategoryId != null) {
      setting.materialCategoryId = materialCategoryId;
    } else if (materialScope === "specific" && materialIds != null) {
      setting.materialIds = materialIds;
    }

    setting.threshold = {};
    if (minQty != null && minQty !== "") {
      setting.threshold.min = minQty;
    }
    if (maxQty != null && maxQty !== "") {
      setting.threshold.max = maxQty;
    }

    setting.notify = {
      type: notifyType || "message",
      userIds: notifyUserIds || ""
    };

    return JSON.stringify(setting);
  }
}

let plugin = new StockAlertFuncParamEditor();
export { plugin };
```

**返回的 JSON 示例**：

```json
{
  "warehouse": { "id": "100001", "name": "成品仓" },
  "materialScope": "category",
  "materialCategoryId": "200001",
  "threshold": { "min": "100", "max": "5000" },
  "notify": { "type": "both", "userIds": "300001,300002" }
}
```

**要点说明**：

1. `getSetting` 返回的字符串会被平台持久化，公式运行时通过反序列化还原配置
2. 建议返回 JSON 格式，结构清晰且便于公式引擎解析
3. 数值类参数以字符串形式存入 JSON，运行时再转为 `BigDecimal`，避免精度丢失
4. `getSetting` 与 `checkSetting` 配合使用：先校验再获取，通常在同一个插件类中实现
5. 对可选参数做空值判断，不存入 JSON，减少冗余数据
