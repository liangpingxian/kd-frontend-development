## afterLoadData - 单据加载完毕

**所属接口**：`IBillPlugIn`
**基类**：`AbstractBillPlugIn`
**签名**：`afterLoadData(e: EventObject): void`
**触发时机**：编辑已有单据，数据加载完成后。仅编辑/查看模式触发，新增不触发。
**典型用途**：加载完毕后补充字段值、重算未存储的字段值、根据数据状态控制界面。

### 业务场景：采购订单加载后重算含税金额与供应商信用状态

采购订单打开时，根据分录的单价和数量重新计算含税金额（未持久化字段），同时查询供应商信用状态并在界面上给出提示和锁定控制。

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { DynamicObject } from "@cosmic/bos-core/kd/bos/dataentity/entity";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";

class PurOrderFormPlugin extends AbstractBillPlugIn {

  afterLoadData(e: $.java.util.EventObject): void {
    super.afterLoadData(e);

    // ---- 1. 重算分录含税金额（未存储字段） ----
    let totalAmount = BigDecimal.ZERO;
    let rowCount = this.getModel().getEntryRowCount("billentry");

    for (let i = 0; i < rowCount; i++) {
      let qty = this.getModel().getValue("qty", i) as BigDecimal;
      let taxPrice = this.getModel().getValue("priceandtax", i) as BigDecimal;

      if (qty != null && taxPrice != null) {
        let lineAmount = qty.multiply(taxPrice);
        this.getModel().setValue("amountandtax", lineAmount, i);
        totalAmount = totalAmount.add(lineAmount);
      }
    }

    // 回写单头汇总金额
    this.getModel().setValue("totalamount", totalAmount);

    // ---- 2. 查询供应商信用状态 ----
    let supplierObj = this.getModel().getValue("supplier") as DynamicObject;
    if (supplierObj != null) {
      let supplierId = supplierObj.getLong("id");

      let creditObj = QueryServiceHelper.queryOne(
        "bd_supplier",
        "creditstatus,creditlimit",
        [new QFilter("id", QCP.equals, supplierId)]
      );

      if (creditObj != null) {
        let creditStatus = creditObj.getString("creditstatus");

        if (creditStatus === "frozen") {
          // 供应商信用冻结：提示警告并锁定提交按钮
          this.getView().showTipNotification("该供应商信用已冻结，请联系采购经理");
          this.getView().setEnable(false, "billentry");
        }
      }
    }

    // ---- 3. 根据单据状态控制界面 ----
    let billStatus = this.getModel().getValue("billstatus") as string;
    if (billStatus === "C") {
      // 已审核单据：锁定关键字段
      this.getView().setEnable(false, "supplier");
      this.getView().setEnable(false, "currency");
    }
  }
}

let plugin = new PurOrderFormPlugin();
export { plugin };
```

**要点说明**：

1. `afterLoadData` 与 `afterCreateNewData` 互斥：新增触发后者，编辑触发前者
2. 在此事件中修改字段值会标记为脏数据，关闭页面时会提示"是否保存"
3. 如果仅用于展示的计算结果，考虑使用控件直接设值避免脏数据提示
4. 所有数据通过 `this.getModel()` 获取，禁止使用类属性存储状态

## check - 单据类型参数校验

**所属接口**：`IBillTypeParameterPlugIn`
**签名**：`check(msgBuilder: StringBuilder): boolean`
**触发时机**：单据类型参数校验时调用，用于验证单据类型配置参数的合法性。
**典型用途**：校验单据类型的自定义参数是否满足业务要求，不满足时将错误信息写入 `msgBuilder` 并返回 `false`。

### 业务场景：校验采购订单单据类型的审批额度配置

当配置采购订单的单据类型参数时，校验审批额度上限必须大于 0 且不能超过系统预设最大值，同时校验必填的审批人配置是否完整。

```typescript
import { StringBuilder } from "@cosmic/bos-script/java/lang";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter, QCP } from "@cosmic/bos-core/kd/bos/orm/query";

class PurOrderBillTypeParamPlugin implements IBillTypeParameterPlugIn {

  check(msgBuilder: StringBuilder): boolean {
    let isValid = true;

    // ---- 1. 校验审批额度配置 ----
    let approvalLimit = this.getParamValue("approvalLimit") as BigDecimal;

    if (approvalLimit == null || approvalLimit.compareTo(BigDecimal.ZERO) <= 0) {
      msgBuilder.append("审批额度上限必须大于0；");
      isValid = false;
    }

    let maxLimit = new BigDecimal("10000000");
    if (approvalLimit != null && approvalLimit.compareTo(maxLimit) > 0) {
      msgBuilder.append("审批额度上限不能超过1000万；");
      isValid = false;
    }

    // ---- 2. 校验审批人配置 ----
    let approverId = this.getParamValue("defaultApprover");

    if (approverId == null || approverId === "") {
      msgBuilder.append("默认审批人不能为空；");
      isValid = false;
    } else {
      // 验证审批人是否存在且启用
      let userObj = QueryServiceHelper.queryOne(
        "bos_user",
        "id,enable",
        [new QFilter("id", QCP.equals, approverId)]
      );

      if (userObj == null) {
        msgBuilder.append("默认审批人不存在，请重新选择；");
        isValid = false;
      } else if (!userObj.getBoolean("enable")) {
        msgBuilder.append("默认审批人已被禁用，请更换；");
        isValid = false;
      }
    }

    // ---- 3. 校验采购类型与仓库配置的匹配性 ----
    let purType = this.getParamValue("purchaseType") as string;
    let warehouseId = this.getParamValue("defaultWarehouse");

    if (purType === "direct" && (warehouseId == null || warehouseId === "")) {
      msgBuilder.append("直接采购类型必须配置默认入库仓库；");
      isValid = false;
    }

    return isValid;
  }
}

let plugin = new PurOrderBillTypeParamPlugin();
export { plugin };
```

**要点说明**：

1. `check` 返回 `true` 表示校验通过，`false` 表示校验失败
2. 校验失败时必须将错误信息写入 `msgBuilder`，系统会统一展示给用户
3. 多条校验规则建议全部执行完再返回，而不是第一条失败就返回，方便用户一次性修正
4. 金额类校验使用 `BigDecimal` 的 `compareTo` 方法，禁止使用 `>` `<` 运算符
