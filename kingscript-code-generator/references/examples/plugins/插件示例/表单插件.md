# beforePropertyChanged - 字段值改变前事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelChangeListener |
| 触发时机 | 字段值被修改之前触发，可在此拦截并取消修改操作 |
| 方法签名 | `beforePropertyChanged(e: PropertyChangedArgs): void` |

## 说明

在字段值实际写入数据模型之前触发的事件回调。通过该方法可以获取字段的新值和旧值，进行前置校验，若不满足业务规则可调用 `e.setCancel(true)` 阻止本次修改。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | PropertyChangedArgs | 事件参数对象 |
| e.getPropertyName() | string | 获取被修改的字段标识 |
| e.getNewValue() | any | 获取即将设置的新值 |
| e.getOldValue() | any | 获取修改前的旧值 |
| e.getRowIndex() | number | 获取分录行索引（单据头字段为 -1） |
| e.setCancel(boolean) | void | 设置为 true 可取消本次修改 |

## 业务场景

采购订单中，单价修改前校验是否允许修改。当单据已审核时，不允许修改单价字段，防止已审核的采购订单被随意调价。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 采购订单表单插件 - 已审核单据不允许修改单价
 */
class PmPurorderPriceCheckPlugin extends AbstractBillPlugIn {

  beforePropertyChanged(e: PropertyChangedArgs): void {
    super.beforePropertyChanged(e);

    const fieldKey = e.getPropertyName();

    // 仅拦截单价字段的修改
    if (fieldKey === "price") {
      // 获取单据头的审核状态
      const billStatus = this.getModel().getValue("billstatus") as string;

      // C 表示已审核状态
      if (billStatus === "C") {
        // 阻止修改
        e.setCancel(true);
        this.getView().showTipNotification("已审核的采购订单不允许修改单价");
        return;
      }

      // 额外校验：新单价不能为负数
      const newPrice = e.getNewValue() as BigDecimal;
      if (newPrice != null && newPrice.compareTo(BigDecimal.ZERO) < 0) {
        e.setCancel(true);
        this.getView().showTipNotification("单价不能为负数");
        return;
      }
    }
  }
}

let plugin = new PmPurorderPriceCheckPlugin();
export { plugin };
```

## 注意事项

- `e.getNewValue()` 获取即将设置的新值，`e.getOldValue()` 获取修改前的旧值
- `e.setCancel(true)` 可阻止本次字段值变更，界面上的值会恢复为修改前的状态
- 此方法在值实际写入数据模型前触发，适合做前置校验拦截
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量
- 必须调用 `super.beforePropertyChanged(e)` 以确保父类逻辑正常执行

# propertyChanged - 字段值改变事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelChangeListener |
| 触发时机 | 字段值被修改并写入数据模型之后触发 |
| 方法签名 | `propertyChanged(e: PropertyChangedArgs): void` |

## 说明

字段值变更后的回调事件，是最常用的联动计算入口。当用户在界面上修改字段值后触发，可在此实现字段间的联动计算、自动取值等逻辑。通过 `e.getChangeSet()` 获取所有变更记录。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | PropertyChangedArgs | 事件参数对象 |
| e.getChangeSet() | ChangeData[] | 获取所有变更记录集合 |
| change.getPropertyName() | string | 获取变更的字段标识 |
| change.getRowIndex() | number | 获取变更所在的分录行索引（单据头字段为 -1） |
| change.getNewValue() | any | 获取变更后的新值 |
| change.getOldValue() | any | 获取变更前的旧值 |

## 业务场景

销售订单中，当用户修改分录行的数量或单价时，自动计算该行的金额（数量 * 单价）、含税金额（金额 * (1 + 税率)）、税额（含税金额 - 金额），并汇总到单据头的合计字段。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { PropertyChangedArgs } from "@cosmic/bos-core/kd/bos/entity/datamodel/events";

/**
 * 销售订单表单插件 - 数量/单价变更时自动计算金额、含税金额、税额
 */
class SmSalorderCalcPlugin extends AbstractBillPlugIn {

  propertyChanged(e: PropertyChangedArgs): void {
    super.propertyChanged(e);

    let needRecalcTotal = false;

    for (const change of e.getChangeSet()) {
      const fieldKey = change.getPropertyName();
      const rowIndex = change.getRowIndex();

      if (fieldKey === "qty" || fieldKey === "price" || fieldKey === "taxrate") {
        this.calcRowAmounts(rowIndex);
        needRecalcTotal = true;
      }
    }

    // 有金额变化时统一汇总单据头
    if (needRecalcTotal) {
      this.recalcTotalAmounts();
    }
  }

  // 计算单行金额、含税金额、税额
  private calcRowAmounts(rowIndex: number): void {
    const qty = this.getModel().getValue("qty", rowIndex) as BigDecimal;
    const price = this.getModel().getValue("price", rowIndex) as BigDecimal;
    const taxRate = this.getModel().getValue("taxrate", rowIndex) as BigDecimal;

    if (qty == null || price == null) {
      return;
    }

    // 金额 = 数量 * 单价
    const amount = qty.multiply(price).setScale(2, BigDecimal.ROUND_HALF_UP);
    this.getModel().setValue("amount", amount, rowIndex);

    // 计算含税金额和税额
    if (taxRate != null) {
      // 税率转换为小数（如 13 -> 0.13）
      const taxRateDecimal = taxRate.divide(
        new BigDecimal("100"), 6, BigDecimal.ROUND_HALF_UP
      );

      // 含税金额 = 金额 * (1 + 税率)
      const taxIncAmount = amount.multiply(
        BigDecimal.ONE.add(taxRateDecimal)
      ).setScale(2, BigDecimal.ROUND_HALF_UP);
      this.getModel().setValue("taxincamount", taxIncAmount, rowIndex);

      // 税额 = 含税金额 - 金额
      const taxAmount = taxIncAmount.subtract(amount);
      this.getModel().setValue("taxamount", taxAmount, rowIndex);
    }
  }

  // 汇总单据头合计金额、合计含税金额、合计税额
  private recalcTotalAmounts(): void {
    let totalAmount = BigDecimal.ZERO;
    let totalTaxIncAmount = BigDecimal.ZERO;
    let totalTaxAmount = BigDecimal.ZERO;
    const rowCount = this.getModel().getEntryRowCount("billentry");

    for (let i = 0; i < rowCount; i++) {
      const amount = this.getModel().getValue("amount", i) as BigDecimal;
      const taxIncAmount = this.getModel().getValue("taxincamount", i) as BigDecimal;
      const taxAmount = this.getModel().getValue("taxamount", i) as BigDecimal;

      if (amount != null) {
        totalAmount = totalAmount.add(amount);
      }
      if (taxIncAmount != null) {
        totalTaxIncAmount = totalTaxIncAmount.add(taxIncAmount);
      }
      if (taxAmount != null) {
        totalTaxAmount = totalTaxAmount.add(taxAmount);
      }
    }

    this.getModel().setValue("totalamount", totalAmount);
    this.getModel().setValue("totaltaxincamount", totalTaxIncAmount);
    this.getModel().setValue("totaltaxamount", totalTaxAmount);
  }
}

let plugin = new SmSalorderCalcPlugin();
export { plugin };
```

## 注意事项

- `e.getChangeSet()` 返回所有变更集合，每个变更包含字段名、行索引、新旧值
- `change.getRowIndex()` 获取变更所在的分录行索引，单据头字段返回 -1
- 除法运算必须指定精度和舍入模式（如 `BigDecimal.ROUND_HALF_UP`），否则会报错
- 联动计算中应避免无限循环：例如 A 字段改 B 字段、B 字段又改 A 字段会导致死循环
- 必须调用 `super.propertyChanged(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# initPropertyChanged - 初始化时触发值更新事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelChangeListener |
| 触发时机 | 数据包初始化阶段（下推、引入、复制等）字段值被赋值时触发 |
| 方法签名 | `initPropertyChanged(e: PropertyChangedArgs): void` |

## 说明

在数据包初始化阶段（如下推、引入、复制、新建等场景）字段值发生变更时触发的回调。与 `propertyChanged` 的区别在于触发时机不同：`initPropertyChanged` 在数据初始化阶段触发，此时界面尚未完成绑定；`propertyChanged` 在用户交互修改字段时触发。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | PropertyChangedArgs | 事件参数对象 |
| e.getChangeSet() | ChangeData[] | 获取所有变更记录集合 |
| change.getPropertyName() | string | 获取变更的字段标识 |
| change.getRowIndex() | number | 获取变更所在的分录行索引 |
| change.getNewValue() | any | 获取变更后的新值 |
| change.getOldValue() | any | 获取变更前的旧值 |

## 业务场景

付款单打开时（新建或下推场景），根据单据头的供应商字段自动带出该供应商的默认银行账号和开户行信息，减少用户手工录入。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { PropertyChangedArgs } from "@cosmic/bos-core/kd/bos/entity/datamodel/events";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 付款单表单插件 - 根据供应商自动带出银行账号
 */
class ApPaybillBankPlugin extends AbstractBillPlugIn {

  initPropertyChanged(e: PropertyChangedArgs): void {
    super.initPropertyChanged(e);

    for (const change of e.getChangeSet()) {
      const fieldKey = change.getPropertyName();

      if (fieldKey === "supplier") {
        this.fillSupplierBankInfo();
      }
    }
  }

  // 用户交互阶段修改供应商时也需要带出银行信息
  propertyChanged(e: PropertyChangedArgs): void {
    super.propertyChanged(e);

    for (const change of e.getChangeSet()) {
      const fieldKey = change.getPropertyName();

      if (fieldKey === "supplier") {
        this.fillSupplierBankInfo();
      }
    }
  }

  // 根据供应商查询默认银行账号并填充
  private fillSupplierBankInfo(): void {
    const supplierId = this.getModel().getValue("supplier");
    if (supplierId == null) {
      // 供应商被清空时，同步清空银行信息
      this.getModel().setValue("bankaccount", null);
      this.getModel().setValue("bankname", null);
      return;
    }

    // 查询供应商的默认银行账号
    const bankInfo = QueryServiceHelper.queryOne(
      "bd_supplier",
      "defaultbankaccount,defaultbankname",
      [new QFilter("id", "=", supplierId)]
    );

    if (bankInfo != null) {
      const bankAccount = bankInfo.get("defaultbankaccount");
      if (bankAccount != null) {
        this.getModel().setValue("bankaccount", bankAccount);
      }

      const bankName = bankInfo.get("defaultbankname");
      if (bankName != null) {
        this.getModel().setValue("bankname", bankName);
      }
    }
  }
}

let plugin = new ApPaybillBankPlugin();
export { plugin };
```

## 注意事项

- `initPropertyChanged` 在数据包初始化阶段触发（如下推、引入、复制等），此时界面可能尚未完成绑定
- `propertyChanged` 在用户交互修改字段时触发，两者的参数结构一致
- 如果初始化和交互阶段的逻辑相同，建议将共用逻辑抽取为私有方法，在两个回调中分别调用
- 必须调用 `super.initPropertyChanged(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# isSupportBatchPropChanged - 是否支持批量触发值更新事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelChangeListener |
| 触发时机 | 框架在触发 propertyChanged 前调用，判断是否将多行变更合并为一次回调 |
| 方法签名 | `isSupportBatchPropChanged(): boolean` |

## 说明

控制 `propertyChanged` 是否支持批量模式。默认返回 `false`，每个字段变更单独触发一次 `propertyChanged`。返回 `true` 后，批量操作（如导入、下推）产生的多行变更会合并为一次 `propertyChanged` 调用，通过 `e.getChangeSet()` 可获取所有变更记录。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| 返回值 | boolean | true - 启用批量模式；false - 逐行触发（默认） |

## 业务场景

采购入库单批量导入时，大量分录行的数量和单价同时发生变化。如果逐行触发 `propertyChanged` 并分别计算金额，性能较差。开启批量值更新后，所有变更行合并为一次回调，统一计算金额后再一次性汇总，大幅提高批量导入性能。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { PropertyChangedArgs } from "@cosmic/bos-core/kd/bos/entity/datamodel/events";

/**
 * 采购入库单表单插件 - 批量导入时开启批量值更新以提高性能
 */
class ImPurinbillBatchPlugin extends AbstractBillPlugIn {

  // 启用批量字段变更支持
  isSupportBatchPropChanged(): boolean {
    return true;
  }

  propertyChanged(e: PropertyChangedArgs): void {
    super.propertyChanged(e);

    let needRecalcTotal = false;

    // 批量模式下，一次回调处理所有变更行
    for (const change of e.getChangeSet()) {
      const fieldKey = change.getPropertyName();
      const rowIndex = change.getRowIndex();

      if (fieldKey === "qty" || fieldKey === "price") {
        this.calcRowAmount(rowIndex);
        needRecalcTotal = true;
      }
    }

    // 所有行处理完后统一汇总单据头合计
    if (needRecalcTotal) {
      this.recalcTotalAmount();
    }
  }

  // 计算单行金额 = 数量 * 单价
  private calcRowAmount(rowIndex: number): void {
    const qty = this.getModel().getValue("qty", rowIndex) as BigDecimal;
    const price = this.getModel().getValue("price", rowIndex) as BigDecimal;

    if (qty != null && price != null) {
      const amount = qty.multiply(price).setScale(2, BigDecimal.ROUND_HALF_UP);
      this.getModel().setValue("amount", amount, rowIndex);
    }
  }

  // 汇总所有分录行金额到单据头
  private recalcTotalAmount(): void {
    let totalAmount = BigDecimal.ZERO;
    const rowCount = this.getModel().getEntryRowCount("billentry");

    for (let i = 0; i < rowCount; i++) {
      const amount = this.getModel().getValue("amount", i) as BigDecimal;
      if (amount != null) {
        totalAmount = totalAmount.add(amount);
      }
    }

    this.getModel().setValue("totalamount", totalAmount);
  }
}

let plugin = new ImPurinbillBatchPlugin();
export { plugin };
```

## 注意事项

- 默认返回 `false`，每个字段变更都会单独触发一次 `propertyChanged`
- 返回 `true` 后，批量操作（如导入、下推、批量填充）时多行变更会合并为一次 `propertyChanged` 调用
- 开启批量模式后，`e.getChangeSet()` 中可能包含多行多字段的变更记录，需遍历处理
- 适合批量导入、下推等场景的性能优化，避免逐行触发导致的性能瓶颈
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# beforeAddRow - 创建分录行前事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelChangeListener |
| 触发时机 | 用户新增分录行之前触发，可在此拦截并取消新增操作 |
| 方法签名 | `beforeAddRow(e: BeforeAddRowEventArgs): void` |

## 说明

在分录行实际创建之前触发的事件回调。通过该方法可以对新增行操作进行前置校验，例如限制行数、校验单据状态等。若不满足条件可调用 `e.setCancel(true)` 阻止新增。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | BeforeAddRowEventArgs | 事件参数对象 |
| e.getEntryProp().getName() | string | 获取当前操作的单据体标识 |
| e.setCancel(boolean) | void | 设置为 true 可取消新增行操作 |

## 业务场景

采购订单中，限制分录行数最多只能添加50行。当用户尝试新增第51行时，阻止操作并给出提示信息，防止单据数据量过大影响系统性能和业务处理。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 采购订单表单插件 - 限制分录行数不超过50行
 */
class PmPurorderRowLimitPlugin extends AbstractBillPlugIn {

  beforeAddRow(e: $.kd.bos.entity.datamodel.events.BeforeAddRowEventArgs): void {
    super.beforeAddRow(e);

    const entryKey = e.getEntryProp().getName();

    if (entryKey === "billentry") {
      // 获取当前分录行数
      const currentRowCount = this.getModel().getEntryRowCount("billentry");

      // 最多允许50行
      if (currentRowCount >= 50) {
        e.setCancel(true);
        this.getView().showTipNotification(
          "采购订单分录行数不能超过50行，当前已有" + currentRowCount + "行"
        );
        return;
      }
    }
  }
}

let plugin = new PmPurorderRowLimitPlugin();
export { plugin };
```

## 注意事项

- `e.getEntryProp().getName()` 获取当前操作的单据体标识，用于区分多个单据体
- `e.setCancel(true)` 阻止新增行操作，新行不会被创建
- 此方法在行实际创建前触发，适合做行数限制、权限控制等前置校验
- 如果单据有多个单据体（如物料明细、费用明细等），需要通过 `entryKey` 判断是哪个单据体在新增行
- 必须调用 `super.beforeAddRow(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# afterAddRow - 创建分录行后事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelChangeListener |
| 触发时机 | 分录行创建完成之后触发 |
| 方法签名 | `afterAddRow(e: AfterAddRowEventArgs): void` |

## 说明

在分录行已创建完成后触发的事件回调。此时新行已经存在于数据模型中，可以安全地对新行设置默认值。常用于自动填充默认仓库、税率、币别等业务默认值。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | AfterAddRowEventArgs | 事件参数对象 |
| e.getEntryProp().getName() | string | 获取当前操作的单据体标识 |
| e.getRow() | number | 获取新增行的行索引 |

## 业务场景

销售订单中，每次新增分录行后，根据单据头的客户信息自动填充默认仓库（取客户的默认发货仓库）和默认税率（取客户适用的税率），减少用户手工录入，提高开单效率。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 销售订单表单插件 - 新增分录行后自动填充默认仓库和税率
 */
class SmSalorderDefaultPlugin extends AbstractBillPlugIn {

  afterAddRow(e: $.kd.bos.entity.datamodel.events.AfterAddRowEventArgs): void {
    super.afterAddRow(e);

    const entryKey = e.getEntryProp().getName();

    if (entryKey === "billentry") {
      const rowIndex = e.getRow();

      // 获取单据头的客户
      const customerId = this.getModel().getValue("customer");
      if (customerId == null) {
        return;
      }

      // 查询客户的默认发货仓库和适用税率
      const customerInfo = QueryServiceHelper.queryOne(
        "bd_customer",
        "defaultwarehouse,defaulttaxrate",
        [new QFilter("id", "=", customerId)]
      );

      if (customerInfo != null) {
        // 填充默认仓库
        const warehouseId = customerInfo.get("defaultwarehouse");
        if (warehouseId != null) {
          this.getModel().setValue("warehouse", warehouseId, rowIndex);
        }

        // 填充默认税率
        const taxRate = customerInfo.get("defaulttaxrate") as BigDecimal;
        if (taxRate != null) {
          this.getModel().setValue("taxrate", taxRate, rowIndex);
        } else {
          // 客户未设置默认税率时，使用系统默认税率13%
          this.getModel().setValue("taxrate", new BigDecimal("13"), rowIndex);
        }
      }
    }
  }
}

export { plugin };
```

# beforeDeleteRow - 删除分录行前事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelChangeListener |
| 触发时机 | 用户删除分录行之前触发，可在此拦截并取消删除操作 |

## 说明

在分录行实际删除之前触发的事件回调。通过该方法可以对待删除的行进行校验，例如检查是否关联了下游单据、是否存在未完结的业务等。若不允许删除可调用 `e.setCancel(true)` 阻止操作。

## 业务场景

销售订单中，已关联下游出库单（已下推生成出库单）的分录行不允许删除，防止上下游单据数据不一致，确保业务链路的完整性。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 销售订单表单插件 - 已关联下游单据的分录行不允许删除
 */
class SmSalorderDeleteCheckPlugin extends AbstractBillPlugIn {

  beforeDeleteRow(e: $.kd.bos.entity.datamodel.events.BeforeDeleteRowEventArgs): void {
    super.beforeDeleteRow(e);

    const entryKey = e.getEntryProp().getName();

    if (entryKey === "billentry") {
      const rowIndex = e.getRow();

      // 获取分录行的主键ID
      const entryId = this.getModel().getEntryPkValue("billentry", rowIndex);
      if (entryId == null) {
        // 未保存的新行可以直接删除
        return;
      }

      // 查询是否存在关联的下游出库单
      const downStream = QueryServiceHelper.queryOne(
        "im_saloutbill",
        "billno",
        [new QFilter("billentry.srcbillentryid", "=", entryId)]
      );

      if (downStream != null) {
        const downBillNo = downStream.get("billno") as string;
        e.setCancel(true);
        this.getView().showErrorNotification(
          "第" + (rowIndex + 1) + "行已关联下游出库单【" + downBillNo + "】，不允许删除"
        );
        return;
      }

      // 查询是否存在关联的下游发货通知单
      const deliveryNotice = QueryServiceHelper.queryOne(
        "sm_deliverynotice",
        "billno",
        [new QFilter("billentry.srcbillentryid", "=", entryId)]
      );

      if (deliveryNotice != null) {
        const noticeBillNo = deliveryNotice.get("billno") as string;
        e.setCancel(true);
        this.getView().showErrorNotification(
          "第" + (rowIndex + 1) + "行已关联发货通知单【" + noticeBillNo + "】，不允许删除"
        );
        return;
      }
    }
  }
}

let plugin = new SmSalorderDeleteCheckPlugin();
export { plugin };
```

## 注意事项

# afterDeleteRow - 删除分录行后事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelChangeListener |
| 触发时机 | 分录行删除完成之后触发 |
| 方法签名 | `afterDeleteRow(e: AfterDeleteRowEventArgs): void` |

## 说明

在分录行已经从数据模型中删除之后触发的事件回调。此时行数已经减少，适合做汇总重算、序号重排等后续处理。常用于删除行后重新汇总单据头的合计金额。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | AfterDeleteRowEventArgs | 事件参数对象 |
| e.getEntryProp().getName() | string | 获取当前操作的单据体标识 |
| e.getRow() | number | 获取被删除行原来的行索引 |

## 业务场景

销售订单中，删除分录行后需要重新汇总单据头的合计金额、合计含税金额和合计税额，确保单据头的汇总数据与分录明细保持一致。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 销售订单表单插件 - 删除分录行后重新汇总单据头合计金额
 */
class SmSalorderSummaryPlugin extends AbstractBillPlugIn {

  afterDeleteRow(e: $.kd.bos.entity.datamodel.events.AfterDeleteRowEventArgs): void {
    super.afterDeleteRow(e);

    const entryKey = e.getEntryProp().getName();

    if (entryKey === "billentry") {
      // 重新汇总单据头合计金额
      this.recalcTotalAmounts();

      // 重新生成行序号
      this.resetRowSeqNo();
    }
  }

  // 汇总计算单据头合计金额、合计含税金额、合计税额
  private recalcTotalAmounts(): void {
    let totalAmount = BigDecimal.ZERO;
    let totalTaxIncAmount = BigDecimal.ZERO;
    let totalTaxAmount = BigDecimal.ZERO;
    const rowCount = this.getModel().getEntryRowCount("billentry");

    for (let i = 0; i < rowCount; i++) {
      const amount = this.getModel().getValue("amount", i) as BigDecimal;
      const taxIncAmount = this.getModel().getValue("taxincamount", i) as BigDecimal;
      const taxAmount = this.getModel().getValue("taxamount", i) as BigDecimal;

      if (amount != null) {
        totalAmount = totalAmount.add(amount);
      }
      if (taxIncAmount != null) {
        totalTaxIncAmount = totalTaxIncAmount.add(taxIncAmount);
      }
      if (taxAmount != null) {
        totalTaxAmount = totalTaxAmount.add(taxAmount);
      }
    }

    this.getModel().setValue("totalamount", totalAmount);
    this.getModel().setValue("totaltaxincamount", totalTaxIncAmount);
    this.getModel().setValue("totaltaxamount", totalTaxAmount);
  }

  // 重新排列分录行序号
  private resetRowSeqNo(): void {
    const rowCount = this.getModel().getEntryRowCount("billentry");
    for (let i = 0; i < rowCount; i++) {
      this.getModel().setValue("seqno", new BigDecimal(String(i + 1)), i);
    }
  }
}

let plugin = new SmSalorderSummaryPlugin();
export { plugin };
```

## 注意事项

- `afterDeleteRow` 在行已删除后触发，此时分录行数已经减少
- `e.getRow()` 获取被删除行原来的行索引，注意此时该索引对应的已是下一行（或不存在）
- 适合做汇总重算、序号重排等后续处理

- 必须调用 `super.afterDeleteRow(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

| 属性 | 说明 |

| 触发时机 | 清空整个单据体（删除所有分录行）之前触发，可在此拦截并取消操作 |
| 方法签名 | `beforeDeleteEntry(e: BeforeDeleteEntryEventArgs): void` |

## 说明

在清空整个单据体数据之前触发的事件回调。与 `beforeDeleteRow`（删除单行）不同，此方法在用户执行"删除全部行"或调用 `deleteEntryData` 时触发。通过 `e.setCancel(true)` 可以阻止整个清空操作。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|

| e.getEntryProp().getName() | string | 获取当前操作的单据体标识 |
| e.setCancel(boolean) | void | 设置为 true 可取消清空操作 |

## 业务场景

已提交的单据不允许清空分录。当单据处于"已提交"或"已审核"状态时，用户尝试清空分录应被阻止，防止破坏已进入审批流程的业务数据。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 采购订单表单插件 - 已提交的单据不允许清空分录
 */
class PmPurorderEntryProtectPlugin extends AbstractBillPlugIn {

  beforeDeleteEntry(e: $.kd.bos.entity.datamodel.events.BeforeDeleteEntryEventArgs): void {
    super.beforeDeleteEntry(e);

    const entryKey = e.getEntryProp().getName();

    if (entryKey === "billentry") {
      // 获取单据状态
      const billStatus = this.getModel().getValue("billstatus") as string;

      // A-暂存  B-已提交  C-已审核
      // 已提交或已审核状态不允许清空分录
      if (billStatus === "B" || billStatus === "C") {
        e.setCancel(true);

        let statusText = "";
        if (billStatus === "B") {
          statusText = "已提交";
        } else if (billStatus === "C") {
          statusText = "已审核";
        }

        this.getView().showErrorNotification(
          "单据当前状态为【" + statusText + "】，不允许清空分录数据"
        );
        return;
      }

      // 额外校验：分录中是否有已关联下游单据的行
      const rowCount = this.getModel().getEntryRowCount("billentry");
      for (let i = 0; i < rowCount; i++) {
        const pushStatus = this.getModel().getValue("pushstatus", i) as string;
        if (pushStatus === "B") {
          e.setCancel(true);
          this.getView().showErrorNotification(
            "第" + (i + 1) + "行已下推生成下游单据，不允许清空分录"
          );
          return;
        }
      }
    }
  }
}

let plugin = new PmPurorderEntryProtectPlugin();
export { plugin };
```

## 注意事项

- `beforeDeleteEntry` 在清空整个单据体时触发，与 `beforeDeleteRow`（删除单行）是不同的事件

- `e.setCancel(true)` 可阻止整个清空操作，所有分录行将保留
- 发现第一个不允许清空的原因后应立即 `return`，避免重复提示

| 方法签名 | `beforeSetItemValue(e: BeforeSetItemValueEventArgs): void` |

## 说明

当用户选择基础资料字段（F7字段）的值时，在值写入数据模型之前触发此事件。可通过 `e.setCancel(true)` 取消赋值操作，实现赋值前的业务校验和拦截。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | BeforeSetItemValueEventArgs | 事件参数对象 |
| e.getPropertyName() | string | 获取被赋值的字段标识 |
| e.getItemValue() | Object | 获取用户选择的基础资料对象 |
| e.setCancel(cancel: boolean) | void | 设置为 true 时取消赋值操作 |

## 业务场景

在采购订单中，当用户在分录行选择物料时，需要校验该物料是否在当前供应商的准入名单中。如果物料未准入，则阻止赋值并提示用户，确保只能采购已准入的物料。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 采购订单表单插件 - 设置物料前校验供应商准入名单
 */
class PurOrderMaterialCheckPlugin extends AbstractBillPlugIn {

  beforeSetItemValue(e: $.kd.bos.entity.datamodel.events.BeforeSetItemValueEventArgs): void {
    super.beforeSetItemValue(e);

    const fieldKey = e.getPropertyName();

    if (fieldKey === "material") {
      const selectedObj = e.getItemValue();
      if (selectedObj == null) {
        return;
      }

      const materialId = selectedObj.getPkValue();
      const materialName = selectedObj.getDataEntityType()
        ? selectedObj.get("name")
        : "";

      // 获取单据头的供应商
      const supplierId = this.getModel().getValue("supplier");
      if (supplierId == null) {
        e.setCancel(true);
        this.getView().showTipNotification("请先选择供应商，再选择物料");
        return;
      }

      // 查询供应商准入名单中是否包含该物料
      const admitRecord = QueryServiceHelper.queryOne(
        "pm_supplier_admit",
        "id,status",
        [
          new QFilter("supplier", "=", supplierId),
          new QFilter("material", "=", materialId),
          new QFilter("enable", "=", "1")
        ]
      );

      if (admitRecord == null) {
        e.setCancel(true);
        this.getView().showErrorNotification(
          "物料【" + materialName + "】未在当前供应商准入名单中，不允许采购"
        );
        return;
      }

      // 校验准入状态是否有效
      const status = admitRecord.get("status") as string;
      if (status !== "C") {
        e.setCancel(true);
        this.getView().showErrorNotification(
          "物料【" + materialName + "】的供应商准入记录未审核通过，不允许采购"
        );
        return;
      }
    }
  }
}

let plugin = new PurOrderMaterialCheckPlugin();
export { plugin };
```

- `e.getItemValue()` 获取用户选择的基础资料对象，可通过 `getPkValue()` 获取主键值
- `e.setCancel(true)` 阻止基础资料赋值，阻止后界面不会显示该值

- 插件类不能定义类属性，所有变量应在方法内部声明

## 说明

当用户在分录中执行"置底"操作后触发此事件。可用于在行置底后重新编排行号、更新排序相关的业务字段等。

在费用报销单中，报销明细分录行支持排序调整。当用户将某个费用明细行置底后，需要自动重新编排所有明细行的行号（1、2、3...），并重新汇总计算报销总金额，确保行号连续且金额正确。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 费用报销单表单插件 - 分录行置底后重新编排行号
 */
class ErExpenseReimbFormPlugin extends AbstractBillPlugIn {

  afterSetEntryBottom(e: $.kd.bos.entity.datamodel.events.AfterMoveEntryEventArgs): void {
    super.afterSetEntryBottom(e);

    const entryKey = e.getEntryProp().getName();

    if (entryKey === "expenseentry") {
      this.reNumberLineNo();
      this.recalcTotalAmount();
    }
  }

  /**
   * 重新编排费用明细行号
   */
  private reNumberLineNo(): void {
    const rowCount = this.getModel().getEntryRowCount("expenseentry");

    for (let i = 0; i < rowCount; i++) {
      this.getModel().setValue("lineno", new BigDecimal(String(i + 1)), i);
    }
  }

  /**
   * 重新汇总计算报销总金额
   */
  private recalcTotalAmount(): void {
    const rowCount = this.getModel().getEntryRowCount("expenseentry");
    let totalAmount = BigDecimal.ZERO;

    for (let i = 0; i < rowCount; i++) {
      const amount = this.getModel().getValue("expenseamount", i) as BigDecimal;
      if (amount != null) {
        totalAmount = totalAmount.add(amount);
      }
    }

    this.getModel().setValue("totalamount", totalAmount);
    this.getView().showTipNotification("行号已重新编排，报销总金额：" + totalAmount.toString());
  }
}

let plugin = new ErExpenseReimbFormPlugin();
export { plugin };
```

## 基本信息

| 所属接口 | IDataModelChangeListener |
| 触发时机 | 用户执行批量填充分录操作之前触发，值尚未写入分录行 |
| 方法签名 | `beforeBatchFillEntry(e: BeforeBatchFillEntryArgs): void` |

## 说明

当用户使用批量填充功能对分录字段统一赋值时，在实际填充之前触发此事件。可通过 `e.setCancel(true)` 取消整个批量填充操作，实现填充前的业务校验和拦截。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | BeforeBatchFillEntryArgs | 事件参数对象 |
| e.getFieldKey() | string | 获取被批量填充的字段标识 |
| e.getFillValue() | Object | 获取用户输入的填充值 |
| e.setCancel(cancel: boolean) | void | 设置为 true 时取消批量填充操作 |

## 业务场景

在销售订单中，用户使用批量填充功能填充订单数量时，需要校验填充后的订单总金额是否超出客户的信用额度。如果超出，则阻止批量填充并提示用户，防止超额下单。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 销售订单表单插件 - 批量填充时校验客户信用额度
 */
class SalOrderBatchFillPlugin extends AbstractBillPlugIn {

  beforeBatchFillEntry(e: $.kd.bos.entity.datamodel.events.BeforeBatchFillEntryArgs): void {
    super.beforeBatchFillEntry(e);

    const fieldKey = e.getFieldKey();

    if (fieldKey === "qty") {
      const fillQty = e.getFillValue() as BigDecimal;

      if (fillQty == null || fillQty.compareTo(BigDecimal.ZERO) <= 0) {
        e.setCancel(true);
        this.getView().showTipNotification("批量填充的数量必须大于0");
        return;
      }

      // 获取单据头的客户
      const customerId = this.getModel().getValue("customer");
      if (customerId == null) {
        e.setCancel(true);
        this.getView().showTipNotification("请先选择客户，再进行批量填充");
        return;
      }

      // 查询客户信用额度
      const creditInfo = QueryServiceHelper.queryOne(
        "bd_customer",
        "creditlimit,usedcredit",
        [new QFilter("id", "=", customerId)]
      );

      if (creditInfo == null) {
        return;
      }

      const creditLimit = creditInfo.get("creditlimit") as BigDecimal;
      const usedCredit = creditInfo.get("usedcredit") as BigDecimal;

      if (creditLimit == null || creditLimit.compareTo(BigDecimal.ZERO) <= 0) {
        // 未设置信用额度，不做限制
        return;
      }

      // 计算批量填充后的预估订单总金额
      let estimatedTotal = BigDecimal.ZERO;
      const rowCount = this.getModel().getEntryRowCount("billentry");

      for (let i = 0; i < rowCount; i++) {
        const price = this.getModel().getValue("price", i) as BigDecimal;
        if (price != null) {
          estimatedTotal = estimatedTotal.add(fillQty.multiply(price));
        }
      }

      // 计算可用信用额度
      const safeUsedCredit = usedCredit != null ? usedCredit : BigDecimal.ZERO;
      const availableCredit = creditLimit.subtract(safeUsedCredit);

      if (estimatedTotal.compareTo(availableCredit) > 0) {
        e.setCancel(true);
        this.getView().showErrorNotification(
          "批量填充后订单预估总金额【" + estimatedTotal.toString() +
          "】超出客户可用信用额度【" + availableCredit.toString() +
          "】，请调整数量"
        );
        return;
      }
    }
  }
}

let plugin = new SalOrderBatchFillPlugin();
export { plugin };
```

## 注意事项

- `e.getFieldKey()` 获取被批量填充的字段标识，需判断是否为目标字段
- `e.getFillValue()` 获取用户输入的填充值，批量填充是对所有行统一赋相同值
- `e.setCancel(true)` 阻止整个批量填充操作，所有行都不会被填充
- 批量填充前数据尚未写入，需要基于填充值进行预估计算
- 必须调用 `super.beforeBatchFillEntry(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明

# createNewData - 创建数据包事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 新建单据时数据包创建阶段触发，早于 afterCreateNewData |
| 方法签名 | `createNewData(e: BizDataEventArgs): void` |

## 说明

在新建单据时，数据包创建阶段触发此事件。通过 `e.getDataEntity()` 直接操作底层数据包进行初始化赋值，适合做底层数据的预处理。此方法比 `afterCreateNewData` 更早触发。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | BizDataEventArgs | 事件参数对象 |
| e.getDataEntity() | Object | 获取当前正在创建的数据包对象，通过 `set(key, value)` 赋值 |

## 业务场景

在新建付款申请单时，需要根据当前登录人所属的部门规则自动生成单据编号前缀。例如：财务部前缀为"FIN-"，采购部前缀为"PUR-"，其他部门前缀为"GEN-"，再拼接日期和序号，形成完整的单据编号。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { PropertyChangedArgs } from "@cosmic/bos-core/kd/bos/entity/datamodel/events";
import { BizDataEventArgs } from "@cosmic/bos-core/kd/bos/entity/datamodel/events";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 付款申请单表单插件 - 新建时按部门规则自动生成单据编号前缀
 */
class ApPayApplyBillNoPlugin extends AbstractBillPlugIn {

  createNewData(e: BizDataEventArgs): void {
    super.createNewData(e);

    const dataEntity = e.getDataEntity();

    // 获取当前登录用户ID
    const currUserId = RequestContext.get().getCurrUserId();

    // 查询当前用户所属部门信息
    const userInfo = QueryServiceHelper.queryOne(
      "bos_user",
      "dpt.id as deptid,dpt.number as deptnumber,dpt.name as deptname",
      [new QFilter("id", "=", currUserId)]
    );

    if (userInfo == null) {
      return;
    }

    const deptNumber = userInfo.get("deptnumber") as string;
    const deptId = userInfo.get("deptid");

    // 根据部门编码确定单据编号前缀
    let billNoPrefix = "GEN-";
    if (deptNumber != null) {
      if (deptNumber.startsWith("FIN")) {
        billNoPrefix = "FIN-";
      } else if (deptNumber.startsWith("PUR")) {
        billNoPrefix = "PUR-";
      } else if (deptNumber.startsWith("SAL")) {
        billNoPrefix = "SAL-";
      }
    }

    // 拼接日期部分（YYYYMMDD格式）
    const now = new Date();
    const year = String(now.getFullYear());
    const month = String(now.getMonth() + 1).padStart(2, "0");
    const day = String(now.getDate()).padStart(2, "0");
    const dateStr = year + month + day;

    // 生成单据编号（前缀 + 日期 + 时间戳后4位作为序号）
    const seqStr = String(now.getTime()).slice(-4);
    const billNo = billNoPrefix + dateStr + "-" + seqStr;

    // 设置单据编号
    dataEntity.set("billno", billNo);

    // 设置申请部门
    if (deptId != null) {
      dataEntity.set("applydept", deptId);
    }

    // 设置申请人
    dataEntity.set("applier", currUserId);

    // 设置申请日期
    dataEntity.set("applydate", new Date());
  }
}

let plugin = new ApPayApplyBillNoPlugin();
export { plugin };
```

## 注意事项

- `createNewData` 直接操作数据包（`e.getDataEntity()`），使用 `dataEntity.set(key, value)` 赋值，不能使用 `this.getModel().setValue()`
- 此方法比 `afterCreateNewData` 更早触发，适合做底层数据初始化
- 仅在新建单据时触发，编辑已有单据时不触发
- 在此方法中界面模型尚未完全就绪，不建议调用 `this.getView()` 相关方法
- 必须调用 `super.createNewData(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明

```typescript
class SalOrderDefaultValuePlugin extends AbstractBillPlugIn {

  afterCreateNewData(e: $.java.util.EventObject): void {
    super.afterCreateNewData(e);

    // 获取当前登录用户信息
    const currUserId = RequestContext.get().getCurrUserId();

    // 设置销售组织为当前登录组织

      // 设置销售部门

        this.getModel().setValue("saledept", deptId);
      }

      // 如果未找到业务员，尝试从用户信息获取部门

        "bos_user",
        "dpt.id as deptid",

      );

      if (userInfo != null) {
        const deptId = userInfo.get("deptid");
        if (deptId != null) {
          this.getModel().setValue("saledept", deptId);
        }
      }
    }

    // 设置订单日期为当天
    this.getModel().setValue("orderdate", new Date());

    // 根据组织查询默认币别
    const currencyInfo = QueryServiceHelper.queryOne(
      "bd_currency",
      "id",
      [
        new QFilter("number", "=", "CNY"),
        new QFilter("enable", "=", "1")
      ]
    );

    if (currencyInfo != null) {
      this.getModel().setValue("currency", currencyInfo.get("id"));
    }
  }
}

let plugin = new SalOrderDefaultValuePlugin();
export { plugin };
```

## 注意事项

- `afterCreateNewData` 仅在新建单据时触发，编辑已有单据时不触发
- 使用 `this.getModel().setValue()` 设置值，与 `createNewData` 中的 `dataEntity.set()` 方式不同
- 此方法在 `createNewData` 之后执行，界面模型已完全就绪，可以正常使用 `this.getModel()` 和 `this.getView()`
- 设置基础资料字段时传入主键 ID 即可，系统会自动加载对应的基础资料信息
- 必须调用 `super.afterCreateNewData(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明

# initImportData - 批量引入数据之前触发

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 批量引入数据之前触发，用于初始化每条引入记录的默认值 |
| 方法签名 | `initImportData(e: InitImportDataEventArgs): void` |

## 说明

在单据引入（下推、批量导入等）数据时，对每条引入记录进行初始化处理。通过 `e.getDataEntity()` 操作引入的数据包，设置默认值或补充缺失字段，减少用户手动填写的工作量。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 事件参数对象 |
| e.getDataEntity() | Object | 获取当前引入的数据包对象，通过 `set(key, value)` 赋值，`get(key)` 取值 |

## 业务场景

在采购入库单引入时（如从采购订单下推生成采购入库单），需要根据当前登录组织的仓库配置，自动为引入数据的每行分录设置默认仓库，避免用户逐行手动选择仓库。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 采购入库单表单插件 - 引入时初始化默认仓库
 */
class PurInStockInitWarehousePlugin extends AbstractBillPlugIn {

  initImportData(e: $.kd.bos.entity.datamodel.events.InitImportDataEventArgs): void {
    super.initImportData(e);

    const dataEntity = e.getDataEntity();

    // 获取当前登录组织
    const currOrgId = RequestContext.get().getOrgId();

    // 查询当前组织的默认仓库
    const warehouseInfo = QueryServiceHelper.queryOne(
      "bd_warehouse",
      "id,name",
      [
        new QFilter("org", "=", currOrgId),
        new QFilter("isdefault", "=", true),
        new QFilter("usestatus", "=", 1)
      ]
    );

    if (warehouseInfo == null) {
      return;
    }

    const defaultWarehouseId = warehouseInfo.get("id");

    // 获取分录数据
    const entryRows = dataEntity.get("billentry");
    if (entryRows == null) {
      return;
    }

    // 遍历分录行，为未设置仓库的行设置默认仓库
    for (let i = 0; i < entryRows.size(); i++) {
      const row = entryRows.get(i);
      const existWarehouse = row.get("warehouse");

      // 仅当仓库未设置时才设置默认值
      if (existWarehouse == null) {
        row.set("warehouse", defaultWarehouseId);
      }
    }

    // 设置入库日期为当天
    const importDate = dataEntity.get("instockdate");
    if (importDate == null) {
      dataEntity.set("instockdate", new Date());
    }
  }
}

let plugin = new PurInStockInitWarehousePlugin();
export { plugin };
```

- 此方法适合设置默认仓库、默认日期等通用默认值，减少用户手动填写
- 必须调用 `super.initImportData(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明

# beforeImportData - 引入数据填写之前触发

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 引入数据填写到单据之前触发，可对引入数据进行校验和拦截 |
| 方法签名 | `beforeImportData(e: BeforeImportDataEventArgs): void` |

## 说明

在引入数据即将写入单据之前触发此事件。可对引入的数据进行合规性校验，不合规的数据行通过 `e.addErrorInfo()` 标记错误信息，该行将被跳过不写入。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 事件参数对象 |
| e.getDataEntities() | Array | 获取所有待引入的数据实体数组 |
| e.addErrorInfo(entity: Object, message: string) | void | 为指定数据行标记错误信息，该行将被跳过 |

## 业务场景

在应付单引入时（如从采购入库单下推生成应付单），需要校验引入数据的币别是否与当前应付单单据头的币别一致。如果币别不一致，则标记错误信息，该条数据将被跳过，防止多币别数据混入同一张应付单造成核算错误。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 应付单表单插件 - 引入时校验币别一致性
 */
class ApPayableCurrencyCheckPlugin extends AbstractBillPlugIn {

  beforeImportData(e: $.kd.bos.entity.datamodel.events.BeforeImportDataEventArgs): void {
    super.beforeImportData(e);

    // 获取单据头的币别
    const headCurrencyId = this.getModel().getValue("currency");
    if (headCurrencyId == null) {
      return;
    }

    // 获取单据头的币别名称，用于提示
    const headCurrencyName = this.getModel().getValue("currency.name") as string;
    const displayName = headCurrencyName != null ? headCurrencyName : String(headCurrencyId);

    // 获取所有待引入的数据实体
    const dataEntities = e.getDataEntities();

    for (const entity of dataEntities) {
      // 获取引入数据的币别
      const importCurrencyId = entity.get("currency");

      if (importCurrencyId == null) {
        e.addErrorInfo(entity, "引入数据的币别为空，无法引入");
        continue;
      }

      // 校验币别是否一致
      if (String(importCurrencyId) !== String(headCurrencyId)) {
        const importCurrencyName = entity.get("currency.name");
        const importDisplayName = importCurrencyName != null
          ? String(importCurrencyName)
          : String(importCurrencyId);

        e.addErrorInfo(
          entity,
          "引入数据的币别【" + importDisplayName +
          "】与单据头币别【" + displayName +
          "】不一致，不允许引入"
        );
        continue;
      }

      // 校验金额是否大于0
      const amount = entity.get("amount") as BigDecimal;
      if (amount != null && amount.compareTo(BigDecimal.ZERO) <= 0) {
        e.addErrorInfo(entity, "引入数据的金额必须大于0");
        continue;
      }
    }
  }
}

let plugin = new ApPayableCurrencyCheckPlugin();
export { plugin };
```

## 注意事项

- `beforeImportData` 在引入数据写入单据之前触发，适合做数据合规性校验
- `e.getDataEntities()` 获取所有待引入的数据实体数组，需逐条遍历校验
- `e.addErrorInfo(entity, message)` 为指定数据行标记错误信息，该行将被跳过不写入，校验通过的数据行继续正常引入
- 此方法中可通过 `this.getModel().getValue()` 获取当前单据头的信息，用于与引入数据做比对校验
- 必须调用 `super.beforeImportData(e)` 以确保父类逻辑正常执行
- 插件类不能定义类属性，所有变量应在方法内部声明

# afterImportData - 引入数据填写完毕之后触发

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 引入数据全部写入完成后触发 |
| 方法签名 | `afterImportData(e: ImportDataEventArgs): void` |

## 说明

在单据引入（如从其他单据下推、Excel导入等）数据全部写入完成后触发。可用于对引入后的数据进行后续的业务处理，例如自动补充字段值、校验数据完整性、触发关联计算等。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 引入数据事件参数，可通过 `e.getDataEntities()` 获取成功引入的数据集合 |

## 业务场景

采购订单引入完成后，自动根据物料查找最新采购价并填入单价。当用户通过引入功能批量创建采购订单分录时，系统自动从价格清单中查询每个物料的最新采购价格，并回填到分录的单价字段，同时计算金额。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";

/**
 * 采购订单表单插件 - 引入完成后自动查找最新采购价
 */
class PmPurorderImportPricePlugin extends AbstractBillPlugIn {

  afterImportData(e: $.kd.bos.entity.datamodel.events.ImportDataEventArgs): void {
    super.afterImportData(e);

    const rowCount = this.getModel().getEntryRowCount("billentry");

    if (rowCount === 0) {
      return;
    }

    // 获取当前采购组织
    const orgId = RequestContext.get().getOrgId();
    let updatedCount = 0;

    for (let i = 0; i < rowCount; i++) {
      const materialId = this.getModel().getValue("material", i);
      const supplierId = this.getModel().getValue("supplier");

      if (materialId == null) {
        continue;
      }

      // 查询该物料在该供应商下的最新采购价格
      const priceFilters = [
        new QFilter("material", "=", materialId),
        new QFilter("supplier", "=", supplierId),
        new QFilter("org", "=", orgId),
        new QFilter("usestatus", "=", 1)
      ];

      const priceInfo = QueryServiceHelper.queryOne(
        "pm_pricelist",
        "price,currency,taxrate",
        priceFilters
      );

      if (priceInfo != null) {
        const latestPrice = priceInfo.get("price") as BigDecimal;

        // 填入单价
        this.getModel().setValue("price", latestPrice, i);

        // 计算金额 = 数量 * 单价
        const qty = this.getModel().getValue("qty", i) as BigDecimal;
        if (qty != null && latestPrice != null) {
          const amount = qty.multiply(latestPrice).setScale(2, BigDecimal.ROUND_HALF_UP);
          this.getModel().setValue("amount", amount, i);
        }

        // 同步税率
        const taxRate = priceInfo.get("taxrate") as BigDecimal;
        if (taxRate != null) {
          this.getModel().setValue("taxrate", taxRate, i);
        }

        updatedCount++;
      }
    }

    if (updatedCount > 0) {
      this.getView().showSuccessNotification(
        "已自动填入" + updatedCount + "行物料的最新采购价"
      );
    } else {
      this.getView().showTipNotification(
        "未找到匹配的采购价格，请手动填写单价"
      );
    }
  }
}

let plugin = new PmPurorderImportPricePlugin();
export { plugin };
```

## 注意事项

- `afterImportData` 在所有引入数据保存成功后触发，此时数据已写入数据模型
- 必须先调用 `super.afterImportData(e)`，确保基础逻辑已执行
- 批量引入时分录行数可能较多，建议关注查询性能，可考虑批量查询后再逐行赋值
- 该事件与 `beforeImportData` 配合使用，前者用于引入前校验，后者用于引入后处理
- 插件类中不要定义类属性，所有变量均使用局部变量

# loadData - 加载数据包事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 打开已有单据加载数据时触发（新建单据不触发） |
| 方法签名 | `loadData(e: LoadDataEventArgs): void` |

## 说明

在编辑已有单据时，数据加载阶段触发。可以在此事件中自定义或增强数据加载逻辑，例如从外部系统获取补充数据、加载关联单据信息用于界面展示等。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 数据加载事件参数 |

## 业务场景

打开应付单时，加载关联的采购订单信息用于展示。当财务人员打开应付单查看或编辑时，系统自动查询该应付单关联的采购订单编号、采购日期、采购员等信息，并填入应付单的辅助展示字段中，方便财务人员核对。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 应付单表单插件 - 加载时补充关联采购订单信息
 */
class ApPayableFormPlugin extends AbstractBillPlugIn {

  loadData(e: $.kd.bos.entity.datamodel.events.LoadDataEventArgs): void {
    super.loadData(e);

    // 获取分录行数
    const rowCount = this.getModel().getEntryRowCount("billentry");

    for (let i = 0; i < rowCount; i++) {
      // 获取分录中关联的采购订单行ID
      const srcBillId = this.getModel().getValue("srcbillid", i);

      if (srcBillId == null) {
        continue;
      }

      // 查询关联的采购订单信息
      const purOrderInfo = QueryServiceHelper.queryOne(
        "pm_purorder",
        "billno,billdate,operator.name as operatorname,billstatus",
        [new QFilter("id", "=", srcBillId)]
      );

      if (purOrderInfo != null) {
        // 设置采购订单编号（辅助展示字段）
        const purBillNo = purOrderInfo.get("billno") as string;
        this.getModel().setValue("srcbillno", purBillNo, i);

        // 设置采购日期
        const purDate = purOrderInfo.get("billdate");
        this.getModel().setValue("srcbilldate", purDate, i);

        // 设置采购员名称
        const operatorName = purOrderInfo.get("operatorname") as string;
        this.getModel().setValue("srcoperator", operatorName, i);

        // 根据采购订单状态设置标记颜色
        const purStatus = purOrderInfo.get("billstatus") as string;
        if (purStatus === "C") {
          // 已审核的采购订单标记为绿色
          this.getView().setEntryForeColor("srcbillno", i, "#28a745");
        } else if (purStatus === "A") {
          // 暂存状态标记为灰色
          this.getView().setEntryForeColor("srcbillno", i, "#999999");
        }
      }
    }

    // 计算并展示应付总金额与采购总金额的差异
    const payableTotal = this.getModel().getValue("totalamount") as BigDecimal;
    if (payableTotal != null) {
      let purTotal = BigDecimal.ZERO;

      for (let i = 0; i < rowCount; i++) {
        const srcAmount = this.getModel().getValue("srcamount", i) as BigDecimal;
        if (srcAmount != null) {
          purTotal = purTotal.add(srcAmount);
        }
      }

      const diffAmount = payableTotal.subtract(purTotal);
      this.getModel().setValue("amountdiff", diffAmount);
    }
  }
}

let plugin = new ApPayableFormPlugin();
export { plugin };
```

## 注意事项

- `loadData` 仅在编辑已有单据时触发，新建单据时不会触发此事件
- 必须先调用 `super.loadData(e)`，确保基础数据已完成加载
- 适合在加载阶段补充非持久化的辅助展示数据，这些数据不会随单据保存
- 分录行数较多时应注意查询性能，建议使用批量查询代替逐行查询
- 不要在此事件中执行耗时操作，否则会影响单据打开速度

# afterCopyData - 复制数据包完成后触发

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 复制单据完成后触发（新建、编辑不触发） |
| 方法签名 | `afterCopyData(e: $.java.util.EventObject): void` |

## 说明

在用户执行复制单据操作、数据复制完成后触发。可用于清除与原单据绑定的唯一性数据（如审批信息、关联单号等），重置单据状态，确保复制后的新单据处于正确的初始状态。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 复制数据完成事件参数 |

## 业务场景

复制销售订单时，清除原单关联信息、重置单据状态为暂存、清空审批信息。当业务员复制一张已审核的销售订单来快速创建新订单时，需要将原单的审批记录、关联的出库单信息、原单编号引用等清除，同时将单据状态重置为"暂存"，避免新单据携带原单的执行信息。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 销售订单表单插件 - 复制后清除关联信息并重置状态
 */
class SmSalorderCopyPlugin extends AbstractBillPlugIn {

  afterCopyData(e: $.java.util.EventObject): void {
    super.afterCopyData(e);

    // ===== 1. 重置单据状态为暂存 =====
    this.getModel().setValue("billstatus", "A");

    // ===== 2. 清空审批相关信息 =====
    this.getModel().setValue("approver", null);
    this.getModel().setValue("approvedate", null);
    this.getModel().setValue("approveopinion", "");

    // ===== 3. 清除原单关联信息 =====
    // 清除源单编号
    this.getModel().setValue("srcbillno", "");
    // 清除源单ID
    this.getModel().setValue("srcbillid", null);
    // 清除源单类型
    this.getModel().setValue("srcbilltype", "");

    // ===== 4. 重置单据日期为当天 =====
    this.getModel().setValue("billdate", new Date());
    this.getModel().setValue("deliverydate", null);

    // ===== 5. 清除分录中的执行信息 =====
    const rowCount = this.getModel().getEntryRowCount("billentry");
    for (let i = 0; i < rowCount; i++) {
      // 清除已出库数量
      this.getModel().setValue("outqty", BigDecimal.ZERO, i);

      // 重置分录行状态为未执行
      this.getModel().setValue("rowstatus", "A", i);

      // 清除关联的出库单号
      this.getModel().setValue("outbillno", "", i);

      // 清除关联的发票号
      this.getModel().setValue("invoiceno", "", i);

      // 清除下游单据关联ID
      this.getModel().setValue("downbillid", null, i);
    }

    // ===== 6. 清除备注中的历史操作记录 =====
    const remark = this.getModel().getValue("remark") as string;
    if (remark != null && remark.indexOf("[操作记录]") >= 0) {
      this.getModel().setValue("remark", "");
    }

    this.getView().showTipNotification(
      "已复制销售订单，单据状态已重置为暂存，请核实后保存"
    );
  }
}

let plugin = new SmSalorderCopyPlugin();
export { plugin };
```

## 注意事项

- `afterCopyData` 仅在复制单据时触发，新建和编辑均不触发
- 必须先调用 `super.afterCopyData(e)`，确保基础复制逻辑已执行
- 复制后单据尚未保存，清理的数据不会影响原单据
- 分录中的执行数据（已出库数量、已开票数量等）通常需要清零
- 审批信息、流程信息必须清除，否则可能导致新单据的审批流程异常
- 单据编号通常由系统自动生成，不需要手动清除

# getEntityType - 获取主实体对象时触发

## 基本信息

|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 表单初始化早期，获取实体类型时触发（此时界面尚未渲染） |
| 方法签名 | `getEntityType(e: GetEntityTypeEventArgs): void` |

## 说明

在表单初始化的早期阶段，系统获取当前表单绑定的主实体对象时触发。可以通过此事件动态替换当前表单绑定的实体类型，实现同一个表单页面根据不同参数加载不同实体结构的效果。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 实体类型事件参数，可通过 `e.setEntityType(entityNumber)` 设置新的实体类型 |

## 业务场景

动态切换单据主实体，根据业务类型加载不同的实体结构。在一个通用的业务申请表单中，根据传入的业务类型参数（如"采购申请"、"费用报销"、"付款申请"），动态加载对应的实体模型，实现一个表单入口支持多种业务单据的效果。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 通用业务申请表单插件 - 根据业务类型动态切换实体
 */
class CommonBizApplyFormPlugin extends AbstractBillPlugIn {

  getEntityType(e: $.kd.bos.entity.datamodel.events.GetEntityTypeEventArgs): void {
    super.getEntityType(e);

    // 获取打开表单时传入的自定义参数
    const formShowParameter = this.getView().getFormShowParameter();
    const customParams = formShowParameter.getCustomParams();

    if (customParams == null) {
      return;
    }

    // 获取业务类型参数
    const bizType = customParams.get("biztype") as string;

    if (bizType == null || bizType.trim() === "") {
      return;
    }

    // 根据业务类型切换到不同的实体
    if (bizType === "purchase_apply") {
      // 切换为采购申请实体
      e.setEntityType("pm_purapply");
    } else if (bizType === "expense_reimburse") {
      // 切换为费用报销实体
      e.setEntityType("er_expensereimburse");
    } else if (bizType === "payment_apply") {
      // 切换为付款申请实体
      e.setEntityType("ap_payapply");
    } else if (bizType === "material_apply") {
      // 切换为领料申请实体
      e.setEntityType("im_materialapply");
    }
  }
}

let plugin = new CommonBizApplyFormPlugin();
export { plugin };
```

## 注意事项

- `getEntityType` 在表单初始化的最早期触发，此时界面尚未渲染，不能操作控件
- 通过 `e.setEntityType(entityNumber)` 可以动态替换实体类型
- 实体切换后，表单的字段布局将按新实体的定义渲染
- 必须先调用 `super.getEntityType(e)`
- 适用于一个表单页面根据参数加载不同实体的通用化场景
- 确保目标实体在系统中已注册且当前用户有访问权限

# queryImportBasedata - 单据引入批量匹配基础数据事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 引入数据时匹配基础资料触发（如Excel导入匹配客户、物料等） |
| 方法签名 | `queryImportBasedata(e: QueryImportBasedataEventArgs): void` |

## 说明

在单据引入（如Excel导入）时，系统需要将用户填写的文本值（如客户名称、物料编码）匹配为系统中的基础资料。此事件允许自定义匹配逻辑，支持多策略匹配，提高引入成功率。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 基础资料匹配事件参数 |
| e.getFieldKey() | string | 当前匹配的字段标识 |
| e.getImportValue() | string | Excel中用户填写的原始值 |
| e.setBaseDataPkId(id) | void | 设置匹配到的基础资料主键ID |
| e.addErrorInfo(msg) | void | 标记匹配失败的错误信息 |

## 业务场景

Excel导入销售订单时，自定义客户匹配规则。用户在Excel中填写的客户信息可能是客户编码、客户简称或客户全称，系统默认只支持编码精确匹配。通过自定义匹配逻辑，按"编码精确匹配 > 简称模糊匹配 > 全称模糊匹配"的优先级进行匹配，大幅提高导入成功率。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";

/**
 * 销售订单引入插件 - 自定义客户基础资料匹配逻辑
 */
class SmSalorderCustomerMatchPlugin extends AbstractBillPlugIn {

  queryImportBasedata(e: $.kd.bos.entity.datamodel.events.QueryImportBasedataEventArgs): void {
    super.queryImportBasedata(e);

    const fieldKey = e.getFieldKey();

    // 仅处理客户字段的匹配
    if (fieldKey !== "customer") {
      return;
    }

    // 获取用户在Excel中填写的值
    const importValue = e.getImportValue() as string;

    if (importValue == null || importValue.trim() === "") {
      return;
    }

    const trimmedValue = importValue.trim();
    const currOrgId = RequestContext.get().getOrgId();

    // 公共过滤条件：已审核且属于当前组织
    const commonFilters = [
      new QFilter("usestatus", "=", 1),
      new QFilter("org", "=", currOrgId)
    ];

    // 策略1：按客户编码精确匹配
    let customerObj = QueryServiceHelper.queryOne(
      "bd_customer",
      "id",
      [
        new QFilter("number", "=", trimmedValue),
        ...commonFilters
      ]
    );

    // 策略2：编码未匹配到，按客户简称精确匹配
    if (customerObj == null) {
      customerObj = QueryServiceHelper.queryOne(
        "bd_customer",
        "id",
        [
          new QFilter("shortname", "=", trimmedValue),
          ...commonFilters
        ]
      );
    }

    // 策略3：简称未匹配到，按客户全称模糊匹配
    if (customerObj == null) {
      customerObj = QueryServiceHelper.queryOne(
        "bd_customer",
        "id",
        [
          new QFilter("name", "like", "%" + trimmedValue + "%"),
          ...commonFilters
        ]
      );
    }

    // 策略4：全称未匹配到，按统一社会信用代码匹配
    if (customerObj == null) {
      customerObj = QueryServiceHelper.queryOne(
        "bd_customer",
        "id",
        [
          new QFilter("socialcreditcode", "=", trimmedValue),
          ...commonFilters
        ]
      );
    }

    if (customerObj != null) {
      // 匹配成功，设置基础资料主键
      e.setBaseDataPkId(customerObj.get("id"));
    } else {
      // 所有策略均未匹配到，标记错误
      e.addErrorInfo(
        "客户【" + trimmedValue + "】在系统中未找到匹配记录，请检查客户编码或名称是否正确"
      );
    }
  }
}

let plugin = new SmSalorderCustomerMatchPlugin();
export { plugin };
```

## 注意事项

- `queryImportBasedata` 在引入数据匹配基础资料时逐字段触发，需要通过 `e.getFieldKey()` 判断当前处理的字段
- `e.getImportValue()` 获取的是Excel中用户填写的原始文本值
- `e.setBaseDataPkId(id)` 设置匹配到的基础资料主键ID，系统将自动回填
- 建议按"精确匹配 > 模糊匹配"的顺序设计匹配策略，避免误匹配
- 匹配失败时通过 `e.addErrorInfo()` 标记错误，系统会在导入结果中统一展示
- 必须先调用 `super.queryImportBasedata(e)`

# beforeImportEntry - 单据体引入前事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | IDataModelListener |
| 触发时机 | 引入分录数据写入数据模型前触发 |
| 方法签名 | `beforeImportEntry(e: BeforeImportEntryEventArgs): void` |

## 说明

在引入分录数据写入数据模型之前触发，可用于对待引入的分录行数据进行校验、合并、过滤或补充。与 `beforeImportData`（单据头级别）对应，此事件专门处理分录级别的引入前逻辑。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 引入分录前事件参数 |
| e.getEntryData() | Array | 获取待引入的分录行数据集合 |
| e.addErrorInfo(entry, msg) | void | 标记某行分录的错误信息 |

## 业务场景

分录引入前检查是否有重复物料编码，有则合并数量。当用户通过Excel导入采购订单分录时，Excel中可能存在同一物料的多行记录（例如不同批次的需求），系统在引入前自动检测重复物料，将相同物料的数量进行合并，减少分录行数，简化后续处理。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 采购订单表单插件 - 引入分录前合并重复物料
 */
class PmPurorderMergeEntryPlugin extends AbstractBillPlugIn {

  beforeImportEntry(e: $.kd.bos.entity.datamodel.events.BeforeImportEntryEventArgs): void {
    super.beforeImportEntry(e);

    const entryData = e.getEntryData();

    if (entryData == null || entryData.length === 0) {
      return;
    }

    // 用Map记录物料编码对应的首次出现的分录行索引
    const materialMap = new Map<string, number>();
    // 记录需要移除的重复行索引（从后往前移除）
    const removeIndexes: number[] = [];

    for (let i = 0; i < entryData.length; i++) {
      const entry = entryData[i];
      const materialNumber = entry.get("materialnumber") as string;

      if (materialNumber == null || materialNumber.trim() === "") {
        e.addErrorInfo(entry, "第" + (i + 1) + "行物料编码不能为空");
        continue;
      }

      const trimmedNumber = materialNumber.trim();

      if (materialMap.has(trimmedNumber)) {
        // 发现重复物料，合并数量到第一次出现的行
        const firstIndex = materialMap.get(trimmedNumber) as number;
        const firstEntry = entryData[firstIndex];

        // 合并数量
        const firstQty = firstEntry.get("qty") as BigDecimal;
        const currentQty = entry.get("qty") as BigDecimal;

        if (firstQty != null && currentQty != null) {
          const mergedQty = firstQty.add(currentQty);
          firstEntry.set("qty", mergedQty);

          // 重新计算金额
          const price = firstEntry.get("price") as BigDecimal;
          if (price != null) {
            const mergedAmount = mergedQty.multiply(price).setScale(2, BigDecimal.ROUND_HALF_UP);
            firstEntry.set("amount", mergedAmount);
          }
        }

        // 合并备注信息
        const firstRemark = (firstEntry.get("remark") as string) || "";
        const currentRemark = (entry.get("remark") as string) || "";
        if (currentRemark !== "") {
          firstEntry.set("remark", firstRemark + "；" + currentRemark);
        }

        // 标记当前行待移除
        removeIndexes.push(i);
      } else {
        // 首次出现，记录索引
        materialMap.set(trimmedNumber, i);
      }
    }

    // 从后往前移除重复行，避免索引错位
    for (let i = removeIndexes.length - 1; i >= 0; i--) {
      entryData.splice(removeIndexes[i], 1);
    }

    if (removeIndexes.length > 0) {
      this.getView().showTipNotification(
        "检测到" + removeIndexes.length + "行重复物料，已自动合并数量"
      );
    }
  }
}

let plugin = new PmPurorderMergeEntryPlugin();
export { plugin };
```

## 注意事项

- `beforeImportEntry` 在分录数据写入数据模型前触发，此时可以修改或删除待引入的数据
- `e.getEntryData()` 获取的是待引入的分录行数据集合，可直接修改
- 合并操作应注意金额的重新计算，确保数量与金额一致
- 从数组中移除元素时，必须从后往前移除，避免索引错位
- 必须先调用 `super.beforeImportEntry(e)`
- 该事件与 `afterImportData` 配合使用，前者处理引入前的数据预处理，后者处理引入后的后续逻辑

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { FormShowParameter } from "@cosmic/bos-core/kd/bos/form";
import { ShowType } from "@cosmic/bos-core/kd/bos/form";
import { CloseCallBack } from "@cosmic/bos-core/kd/bos/form";
import { ClosedCallBackEvent } from "@cosmic/bos-core/kd/bos/form/events";
import { ItemClickEvent } from "@cosmic/bos-core/kd/bos/form/control/events";

/**
 * 采购订单表单插件 - 供应商选择后自动填充联系信息
 */
class PmPurorderSupplierFillPlugin extends AbstractBillPlugIn {

  registerListener(e: $.java.util.EventObject): void {
    super.registerListener(e);

    // 注册自定义按钮点击监听
    this.addItemClickService("btn_select_supplier");
  }

  itemClick(e: ItemClickEvent): void {
    super.itemClick(e);

    const itemKey = e.getItemKey();

    if (itemKey === "btn_select_supplier") {
      // 打开供应商选择界面
      const showParameter = new FormShowParameter();
      showParameter.setFormId("bd_supplier");
      showParameter.getOpenStyle().setShowType(ShowType.Modal);

      // 设置回调标识
      showParameter.setCloseCallBack(
        new CloseCallBack(this, "select_supplier")
      );

      this.getView().showForm(showParameter);
    }
  }

  closedCallBack(e: ClosedCallBackEvent): void {
    super.closedCallBack(e);

    const actionId = e.getActionId();

    if (actionId === "select_supplier") {
      const returnData = e.getReturnData();

      if (returnData == null) {
        return;
      }

      // 获取选中的供应商ID
      const supplierId = returnData.getPkValue();

      if (supplierId == null) {
        return;
      }

      // 回填供应商
      this.getModel().setValue("supplier", supplierId);

      // 查询供应商的详细信息
      const supplierInfo = QueryServiceHelper.queryOne(
        "bd_supplier",
        "name,contactperson,phone,address,bankname,bankaccount",
        [new QFilter("id", "=", supplierId)]
      );

      if (supplierInfo != null) {
        // 填充默认联系人
        const contactPerson = supplierInfo.get("contactperson") as string;
        if (contactPerson != null) {
          this.getModel().setValue("contactperson", contactPerson);
        }

        // 填充联系电话
        const phone = supplierInfo.get("phone") as string;
        if (phone != null) {
          this.getModel().setValue("contactphone", phone);
        }

        // 填充收货地址
        const address = supplierInfo.get("address") as string;
        if (address != null) {
          this.getModel().setValue("receiveaddress", address);
        }

        // 填充开户银行
        const bankName = supplierInfo.get("bankname") as string;
        if (bankName != null) {
          this.getModel().setValue("bankname", bankName);
        }

        // 填充银行账号
        const bankAccount = supplierInfo.get("bankaccount") as string;
        if (bankAccount != null) {
          this.getModel().setValue("bankaccount", bankAccount);
        }

        this.getView().showSuccessNotification(
          "已自动填充供应商【" + supplierInfo.get("name") + "】的联系信息"
        );
      }
    }
  }
}

let plugin = new PmPurorderSupplierFillPlugin();
export { plugin };
```

## 注意事项

- 必须先调用 `super.closedCallBack(e)`
- 通过 `e.getActionId()` 区分不同子页面的回调，避免混淆
- `e.getReturnData()` 可能为 null（用户未选择直接关闭），必须做空值判断

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**

class BillDeleteConfirmPlugin extends AbstractBillPlugIn {

    }
  }

  confirmCallBack(e: MessageBoxClosedEvent): void {
    super.confirmCallBack(e);

    const callBackId = e.getCallBackId();
    const result = e.getResult();

    // 处理分录删除确认
    if (callBackId === "confirm_delete_entry") {
      if (result === MessageBoxResult.Yes) {
        // 用户确认删除
        const rowIndex = this.getModel().getEntryCurrentRowIndex("billentry");
        if (rowIndex >= 0) {
          this.getModel().deleteEntryRow("billentry", rowIndex);
          this.getView().showSuccessNotification("分录行已删除");

          // 刷新分录列表
          this.getView().updateView("billentry");
        }
      }
      // 用户点击取消，不执行任何操作
    }

    // 处理整单删除确认
    if (callBackId === "confirm_delete_bill") {
      if (result === MessageBoxResult.Yes) {
        const billId = this.getModel().getDataEntity().getPkValue();
        const entityNumber = this.getModel().getDataEntityType().getName();

        // 调用删除服务
        const deleteResult = DeleteServiceHelper.delete(
          entityNumber,
          [billId],
          OperateOption.create()
        );

        if (deleteResult.isSuccess()) {
          this.getView().showSuccessNotification("单据已删除");
          // 关闭当前页面
          this.getView().close();
        } else {
          this.getView().showErrNotification(
            "删除失败：" + deleteResult.getMessage()
          );
        }
      }
    }
  }
}

let plugin = new BillDeleteConfirmPlugin();
export { plugin };
```

## 注意事项

- 必须先调用 `super.confirmCallBack(e)`
- 通过 `e.getCallBackId()` 区分不同的确认场景，每个 `showConfirm` 调用应使用不同的回调ID
- `e.getResult()` 返回用户的选择结果，常见值包括 `MessageBoxResult.Yes`、`MessageBoxResult.No`、`MessageBoxResult.Cancel`
- `showConfirm` 是异步操作，确认框弹出后代码不会阻塞等待，用户的选择结果通过 `confirmCallBack` 回调处理
- `ConfirmCallBackListener` 的第一个参数是回调ID字符串，第二个参数是当前插件实例（this）
- 执行删除等不可逆操作前，务必进行二次确认，防止误操作

# preOpenForm - 打开表单前处理

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 表单打开前最早期触发，此时界面尚未加载 |
| 方法签名 | `preOpenForm(e: PreOpenFormEventArgs): void` |

## 说明

在表单打开之前的最早阶段触发，可用于修改表单标题、布局参数，或根据条件拦截表单的打开。此时界面控件尚未创建，不能操作控件，但可以修改 `FormShowParameter` 中的参数。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | PreOpenFormEventArgs | 表单打开前事件参数 |
| e.setCancel(boolean) | void | 设置为 true 可取消打开表单 |
| e.setCancelMessage(string) | void | 设置取消打开时的提示信息 |

## 业务场景

根据用户权限控制是否允许打开某些敏感单据。在大额付款单表单中，当付款金额超过一定阈值（如100万）时，只有拥有特殊权限的用户才能打开查看。普通用户尝试打开时，系统拦截并提示无权限。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { PreOpenFormEventArgs } from "@cosmic/bos-core/kd/bos/form/events";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";

/**
 * 付款单表单插件 - 大额付款单打开权限控制
 */
class ApPaybillAccessPlugin extends AbstractBillPlugIn {

  preOpenForm(e: PreOpenFormEventArgs): void {
    super.preOpenForm(e);

    // 获取当前用户信息
    const currentUserId = RequestContext.get().getCurrUserId();

    // 获取打开参数，判断是否为编辑模式
    const formShowParameter = this.getView().getFormShowParameter();
    const pkId = formShowParameter.getPkId();

    // 新建模式不拦截
    if (pkId == null || pkId === "0" || pkId === "") {
      return;
    }

    // 查询该付款单的金额
    const billInfo = QueryServiceHelper.queryOne(
      "ap_paybill",
      "totalamount,currency.name as currencyname",
      [new QFilter("id", "=", pkId)]
    );

    if (billInfo == null) {
      return;
    }

    const totalAmount = billInfo.get("totalamount") as BigDecimal;
    const amountThreshold = new BigDecimal("1000000");

    // 金额未超过阈值，正常打开
    if (totalAmount == null || totalAmount.compareTo(amountThreshold) < 0) {
      return;
    }

    // 金额超过阈值，检查用户是否具有大额付款查看权限
    const hasPermission = QueryServiceHelper.queryOne(
      "sys_user_permission",
      "id",
      [
        new QFilter("user", "=", currentUserId),
        new QFilter("permissioncode", "=", "LARGE_PAYMENT_VIEW"),
        new QFilter("usestatus", "=", 1)
      ]
    );

    if (hasPermission == null) {
      // 无权限，拦截打开
      const currencyName = (billInfo.get("currencyname") as string) || "人民币";

      e.setCancel(true);
      e.setCancelMessage(
        "该付款单金额为" + currencyName + " " + totalAmount.toString() +
        "，超过大额付款阈值（100万），您没有查看大额付款单的权限。请联系管理员授权。"
      );
    }

    // 有权限的用户，在标题中标注大额标记
    const customParams = formShowParameter.getCustomParams();
    if (customParams != null) {
      customParams.put("isLargePayment", "true");
    }
  }

  afterBindData(e: $.java.util.EventObject): void {
    super.afterBindData(e);

    // 如果是大额付款单，界面加载后显示醒目提示
    const formShowParameter = this.getView().getFormShowParameter();
    const customParams = formShowParameter.getCustomParams();

    if (customParams != null && customParams.get("isLargePayment") === "true") {
      this.getView().showTipNotification(
        "注意：当前为大额付款单，请仔细核对付款信息"
      );
    }
  }
}

let plugin = new ApPaybillAccessPlugin();
export { plugin };
```

## 注意事项

- `preOpenForm` 是表单打开前最早触发的事件，此时界面控件尚未创建
- 通过 `e.setCancel(true)` 可以拦截表单打开，`e.setCancelMessage()` 设置拦截提示信息
- 必须先调用 `super.preOpenForm(e)`
- 不能在此事件中操作界面控件（如 `this.getView().setVisible()`），因为控件尚未初始化
- 适用于权限校验、参数预处理、表单标题修改等场景
- 如果需要在界面加载后做进一步处理，应在 `afterBindData` 中实现

# initialize - 初始化事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 插件初始化时触发，是插件生命周期中最早执行的方法之一 |
| 方法签名 | `initialize(): void` |

## 说明

插件的初始化方法，在表单生命周期的最早阶段执行。用于进行非UI相关的初始化操作，如注册工具栏按钮、初始化配置参数等。此时数据模型和界面控件可能尚未完全就绪，因此不建议在此方法中操作数据或控件。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| 无 | - | 该方法不接收参数 |

## 业务场景

表单初始化时，注册自定义工具栏按钮并绑定点击事件。在采购订单表单中，根据当前用户角色动态添加工具栏按钮，如"批量询价"、"价格对比"、"供应商评估"等自定义功能按钮，并在后续的 `registerListener` 中绑定点击事件。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";
import { FormShowParameter } from "@cosmic/bos-core/kd/bos/form";
import { ItemClickEvent } from "@cosmic/bos-core/kd/bos/form/control/events";

/**
 * 采购订单表单插件 - 初始化时注册自定义工具栏按钮
 */
class PmPurorderToolbarPlugin extends AbstractBillPlugIn {

  initialize(): void {
    super.initialize();

    // 在初始化阶段添加自定义工具栏按钮
    const toolBar = this.getView().getControl("tbmain");

    if (toolBar != null) {
      // 添加"批量询价"按钮
      toolBar.addButton(
        "btn_batch_inquiry",
        "批量询价",
        "icon-inquiry"
      );

      // 添加"价格对比"按钮
      toolBar.addButton(
        "btn_price_compare",
        "价格对比",
        "icon-compare"
      );

      // 根据用户角色决定是否显示"供应商评估"按钮
      const currentUserId = RequestContext.get().getCurrUserId();
      const isManager = QueryServiceHelper.queryOne(
        "sys_user_role",
        "id",
        [
          new QFilter("user", "=", currentUserId),
          new QFilter("role.number", "=", "purchase_manager")
        ]
      );

      if (isManager != null) {
        // 仅采购经理可看到供应商评估按钮
        toolBar.addButton(
          "btn_supplier_evaluate",
          "供应商评估",
          "icon-evaluate"
        );
      }
    }
  }

  registerListener(e: $.java.util.EventObject): void {
    super.registerListener(e);

    // 注册工具栏按钮点击监听
    this.addItemClickService("btn_batch_inquiry");
    this.addItemClickService("btn_price_compare");
    this.addItemClickService("btn_supplier_evaluate");
  }

  itemClick(e: ItemClickEvent): void {
    super.itemClick(e);

    const itemKey = e.getItemKey();

    if (itemKey === "btn_batch_inquiry") {
      // 批量询价：收集分录中所有物料，打开询价界面
      const rowCount = this.getModel().getEntryRowCount("billentry");
      if (rowCount === 0) {
        this.getView().showTipNotification("请先添加物料分录");
        return;
      }

      const materialIds: any[] = [];
      for (let i = 0; i < rowCount; i++) {
        const materialId = this.getModel().getValue("material", i);
        if (materialId != null) {
          materialIds.push(materialId);
        }
      }

      // 打开询价界面并传入物料列表
      const showParameter = new FormShowParameter();
      showParameter.setFormId("pm_inquiry");
      showParameter.getCustomParams().put("materialIds", JSON.stringify(materialIds));

      this.getView().showForm(showParameter);
    }

    if (itemKey === "btn_price_compare") {
      // 价格对比：查询各供应商报价
      const rowIndex = this.getModel().getEntryCurrentRowIndex("billentry");
      if (rowIndex < 0) {
        this.getView().showTipNotification("请先选择一行物料");
        return;
      }

      const materialId = this.getModel().getValue("material", rowIndex);
      if (materialId == null) {
        this.getView().showTipNotification("当前行未选择物料");
        return;
      }

      // 查询该物料的各供应商报价
      const priceList = QueryServiceHelper.query(
        "pm_pricelist",
        "supplier.name as suppliername,price,currency.name as currencyname,updatetime",
        [
          new QFilter("material", "=", materialId),
          new QFilter("usestatus", "=", 1)
        ]
      );

      if (priceList == null || priceList.size() === 0) {
        this.getView().showTipNotification("未找到该物料的供应商报价记录");
        return;
      }

      // 构建价格对比信息
      let priceInfo = "物料报价对比：\n";
      while (priceList.next()) {
        const supplierName = priceList.getString("suppliername");
        const price = priceList.getBigDecimal("price");
        const currencyName = priceList.getString("currencyname");
        priceInfo += supplierName + ": " + currencyName + " " + price.toString() + "\n";
      }

      this.getView().showTipNotification(priceInfo);
    }

    if (itemKey === "btn_supplier_evaluate") {
      // 供应商评估：打开供应商评估表单
      const supplierId = this.getModel().getValue("supplier");
      if (supplierId == null) {
        this.getView().showTipNotification("请先选择供应商");
        return;
      }

      const showParameter = new FormShowParameter();
      showParameter.setFormId("pm_supplier_evaluate");
      showParameter.getCustomParams().put("supplierId", supplierId.toString());

      this.getView().showForm(showParameter);
    }
  }
}

let plugin = new PmPurorderToolbarPlugin();
export { plugin };
```

## 注意事项

- `initialize` 是插件生命周期中最早执行的方法之一，必须先调用 `super.initialize()`
- 此阶段适合进行非UI相关的初始化操作，如注册工具栏按钮、初始化配置
- 不建议在此方法中操作数据模型（`this.getModel()`），因为数据可能尚未加载
- 界面控件的事件监听应在 `registerListener` 中注册，而非 `initialize`
- 动态添加的按钮需要在 `registerListener` 中注册点击监听，否则点击事件不会触发
- 插件类不能有类属性（如 `private xxx = ...`），只使用局部变量

# registerListener - 注册控件监听器事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 表单初始化阶段，控件创建完成后触发，用于注册各类控件事件监听器 |
| 方法签名 | `registerListener(e: $.java.util.EventObject): void` |

## 说明

注册控件监听器的回调事件，是表单插件中最核心的方法之一。在表单初始化阶段，控件创建完成后触发此方法，插件可在此注册各种控件的事件监听器，包括基础资料F7过滤监听、工具栏按钮点击监听、分录行点击监听等。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | $.java.util.EventObject | 事件参数对象（EventObject） |

## 业务场景

注册多个基础资料F7过滤监听、工具栏按钮点击监听、分录行点击监听。在采购订单中，物料字段需要根据当前组织过滤，自定义工具栏按钮需要响应点击事件，分录行点击时需要显示物料详情。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { BasedataEdit } from "@cosmic/bos-core/kd/bos/form/control";
import { ItemClickEvent, RowClickEvent } from "@cosmic/bos-core/kd/bos/form/control/events";
import { BeforeF7SelectEvent } from "@cosmic/bos-core/kd/bos/form/field/events";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 采购订单插件 - 注册F7过滤、按钮点击、分录行点击等多种监听器
 */
class PmPurorderListenerPlugin extends AbstractBillPlugIn {

  registerListener(e: $.java.util.EventObject): void {
    super.registerListener(e);

    // 1. 注册物料字段的F7过滤监听
    const materialEdit = this.getControl<BasedataEdit>("material");
    if (materialEdit != null) {
      materialEdit.addBeforeF7SelectListener(this);
    }

    // 2. 注册供应商字段的F7过滤监听
    const supplierEdit = this.getControl<BasedataEdit>("supplier");
    if (supplierEdit != null) {
      supplierEdit.addBeforeF7SelectListener(this);
    }

    // 3. 注册工具栏按钮点击监听
    this.addItemClickService("adv_toolbarap");

    // 4. 注册分录行点击监听
    this.addClickService("billentry");
  }

  // F7过滤事件处理
  beforeF7Select(e: BeforeF7SelectEvent): void {
    super.beforeF7Select(e);

    const fieldKey = e.getProperty().getName();
    const orgId = this.getModel().getValue("org_id");

    if (fieldKey === "material") {
      // 物料按当前采购组织过滤，且只显示启用状态的物料
      const orgFilter = new QFilter("useorgid", "=", orgId);
      const statusFilter = new QFilter("status", "=", "C");
      e.getQFilters().add(orgFilter);
      e.getQFilters().add(statusFilter);
    }

    if (fieldKey === "supplier") {
      // 供应商按当前组织过滤，且只显示已审核的供应商
      const orgFilter = new QFilter("entry_org", "=", orgId);
      const statusFilter = new QFilter("status", "=", "C");
      e.getQFilters().add(orgFilter);
      e.getQFilters().add(statusFilter);
    }
  }

  // 工具栏按钮点击事件处理
  itemClick(e: ItemClickEvent): void {
    super.itemClick(e);

    const itemKey = e.getItemKey();

    if (itemKey === "btn_batch_import") {
      // 批量导入按钮：打开导入界面
      const showParam = {
        formId: "pm_purorder_import",
        openStyle: "currentWindow"
      };
      this.getView().showForm(showParam);
    }

    if (itemKey === "btn_price_query") {
      // 价格查询按钮：打开价格查询弹窗
      const supplierId = this.getModel().getValue("supplier_id");
      const showParam = {
        formId: "pm_price_query",
        openStyle: "dialog",
        customParams: { supplierId: String(supplierId) }
      };
      this.getView().showForm(showParam);
    }
  }

  // 分录行点击事件处理
  entryRowClick(e: RowClickEvent): void {
    super.entryRowClick(e);

    const rowIndex = e.getRow();
    const materialId = this.getModel().getValue("material_id", rowIndex);

    if (materialId != null) {
      // 在页面底部状态栏显示物料信息
      const materialName = this.getModel().getValue("materialname", rowIndex) as string;
      const materialSpec = this.getModel().getValue("materialspec", rowIndex) as string;
      const info = "物料：" + materialName + "  规格：" + (materialSpec || "无");
      this.getView().showTipNotification(info);
    }
  }
}

let plugin = new PmPurorderListenerPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.registerListener(e)` 以确保父类的监听器注册逻辑正常执行
- `registerListener` 是注册所有控件监听器的统一入口，应将所有监听注册集中在此方法中
- 注册F7过滤监听前需先通过 `getControl()` 获取控件实例，并判断是否为 null
- 按钮点击监听通过 `addItemClickService` 注册，参数为工具栏控件标识
- 分录行点击监听通过 `addClickService` 注册，参数为分录标识
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量
- 监听注册应在此方法中完成，不要在其他生命周期方法中重复注册

# beforeBindData - 绑定数据前操作

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 数据加载完成但尚未绑定到界面控件之前触发，可在绑定前对数据进行最后的调整 |
| 方法签名 | `beforeBindData(e: BeforeBindDataEvent): void` |

## 说明

在数据模型加载完毕、界面控件绑定数据之前触发的事件回调。此时数据模型中已有值，但尚未渲染到界面上，适合在绑定前对数据进行补充或修改，例如从外部系统获取数据并写入单据字段。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | EventObject | 事件参数对象，包含事件源信息 |

## 业务场景

在采购订单绑定数据前，根据单据头的币别从外部汇率系统查询最新汇率，并自动设置到单据头的汇率字段，同时根据汇率重新计算各分录行的本位币金额。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 采购订单表单插件 - 绑定数据前查询汇率并设置
 */
class PmPurorderExchangeRatePlugin extends AbstractBillPlugIn {

  beforeBindData(e: BeforeBindDataEvent): void {
    super.beforeBindData(e);

    // 获取单据头币别
    const currencyId = this.getModel().getValue("currency");
    if (currencyId == null) {
      return;
    }

    // 获取组织本位币
    const orgId = this.getModel().getValue("org");
    if (orgId == null) {
      return;
    }

    // 查询最新汇率：从汇率表中获取该币别对本位币的汇率
    const rateInfo = QueryServiceHelper.queryOne(
      "bd_exchangerate",
      "exchangerate",
      [
        new QFilter("sourcecurrency", "=", currencyId),
        new QFilter("targetcurrency.isbasecurrency", "=", true),
        new QFilter("org", "=", orgId),
        new QFilter("effectdate", "<=", new Date())
      ],
      "effectdate desc"
    );

    if (rateInfo != null) {
      const exchangeRate = rateInfo.get("exchangerate") as BigDecimal;

      // 设置汇率到单据头
      this.getModel().setValue("exchangerate", exchangeRate);

      // 根据汇率重新计算各分录行的本位币金额
      const rowCount = this.getModel().getEntryRowCount("billentry");
      for (let i = 0; i < rowCount; i++) {
        const originalAmount = this.getModel().getValue("amount", i) as BigDecimal;
        if (originalAmount != null) {
          // 本位币金额 = 原币金额 * 汇率
          const localAmount = originalAmount.multiply(exchangeRate)
            .setScale(2, BigDecimal.ROUND_HALF_UP);
          this.getModel().setValue("localamount", localAmount, i);
        }
      }

      // 计算本位币合计金额
      let totalLocalAmount = BigDecimal.ZERO;
      for (let i = 0; i < rowCount; i++) {
        const localAmt = this.getModel().getValue("localamount", i) as BigDecimal;
        if (localAmt != null) {
          totalLocalAmount = totalLocalAmount.add(localAmt);
        }
      }
      this.getModel().setValue("totallocalamount", totalLocalAmount);
    }
  }
}

let plugin = new PmPurorderExchangeRatePlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.beforeBindData(e)` 以确保父类逻辑正常执行
- 此方法在数据绑定到界面之前触发，修改的值会直接反映到界面上
- 与 `afterBindData` 不同，此方法执行时界面控件尚未渲染，不适合做控件状态控制（如 setEnable、setVisible）
- 查询外部系统数据时注意性能，避免在此方法中执行耗时操作
- 新建单据和编辑单据都会触发此方法
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量
- BigDecimal 的乘除运算需注意精度设置，使用 `setScale` 指定小数位数和舍入模式

# afterBindData - 绑定数据后操作

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 数据完全绑定到界面控件之后触发，此时界面已完成渲染，可安全操作控件状态 |
| 方法签名 | `afterBindData(e: AfterBindDataEvent): void` |

## 说明

在数据加载并绑定到界面控件完成后触发的事件回调。适用于根据已加载的数据动态控制界面控件的可见性、可编辑性、样式等。此时数据模型中的值已全部就绪，可以安全读取字段值来决定界面行为。

| 参数 | 类型 | 说明 |
|------|------|------|
| e | EventObject | 事件参数对象，包含事件源信息 |

## 业务场景

单据加载完成后，根据单据状态动态控制各字段的可编辑性和可见性。例如：暂存状态时所有字段可编辑；已提交状态时锁定关键字段但允许修改备注；已审核或已关闭状态时整单只读，并隐藏保存和提交按钮。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { IBillView, BillOperationStatus } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 采购订单表单插件 - 根据单据状态控制字段可编辑性和可见性
 */
class PmPurorderStatusControlPlugin extends AbstractBillPlugIn {

  afterBindData(e: AfterBindDataEvent): void {
    super.afterBindData(e);

    // 获取当前单据状态：A-暂存 B-已提交 C-已审核 D-已关闭
    const billStatus = this.getModel().getValue("billstatus") as string;
    const view = this.getView();

    switch (billStatus) {
      case "A":
        // 暂存状态：所有字段可编辑，显示保存和提交按钮
        view.setEnable(true, "supplier");
        view.setEnable(true, "billdate");
        view.setEnable(true, "currency");
        view.setEnable(true, "remark");
        view.setVisible(true, "bar_save");
        view.setVisible(true, "bar_submit");
        view.setVisible(false, "bar_audit");
        break;

      case "B":
        // 已提交状态：锁定关键字段，仅允许修改备注
        view.setEnable(false, "supplier");
        view.setEnable(false, "billdate");
        view.setEnable(false, "currency");
        view.setEnable(true, "remark");
        view.setVisible(false, "bar_save");
        view.setVisible(false, "bar_submit");
        view.setVisible(true, "bar_audit");
        break;

      case "C":
        // 已审核状态：整单只读，隐藏操作按钮
        const auditView = view as IBillView;
        auditView.setBillStatus(BillOperationStatus.VIEW);
        view.setVisible(false, "bar_save");
        view.setVisible(false, "bar_submit");
        view.setVisible(false, "bar_audit");
        break;

      case "D":
        // 已关闭状态：整单只读，隐藏所有操作按钮
        const closedView = view as IBillView;
        closedView.setBillStatus(BillOperationStatus.VIEW);
        view.setVisible(false, "bar_save");
        view.setVisible(false, "bar_submit");
        view.setVisible(false, "bar_audit");
        break;
    }

    // 根据是否有关联源单控制源单信息区域的可见性
    const sourceBillNo = this.getModel().getValue("sourcebillno") as string;
    if (sourceBillNo != null && sourceBillNo !== "") {
      view.setVisible(true, "panel_source");
    } else {
      view.setVisible(false, "panel_source");
    }

    // 分录行级别控制：锁定已关联下游单据的分录行数量字段
    const rowCount = this.getModel().getEntryRowCount("billentry");
    for (let i = 0; i < rowCount; i++) {
      const linkedQty = this.getModel().getValue("linkedqty", i) as BigDecimal;
      if (linkedQty != null && linkedQty.compareTo(BigDecimal.ZERO) > 0) {
        view.setEnable(false, i, "qty");
        view.setEnable(false, i, "material");
      }
    }
  }
}

let plugin = new PmPurorderStatusControlPlugin();
export { plugin };
```

- 必须调用 `super.afterBindData(e)` 以确保父类逻辑正常执行
- 此方法在数据绑定完成后触发，可安全读取字段值并控制界面状态
- 与 `beforeBindData` 不同，此方法执行时界面控件已完成渲染，适合做控件状态控制

- 使用 `setBillStatus(BillOperationStatus.VIEW)` 可将整个单据设为只读模式

# beforeDoOperation - 操作前事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 操作（如保存、提交、审核等）执行之前触发，可在此进行界面级校验或二次确认 |
| 方法签名 | `beforeDoOperation(e: BeforeDoOperationEventArgs): void` |

## 说明

在用户触发操作（保存、提交、审核、删除等）但操作尚未真正执行之前触发的事件回调。通过该方法可以进行界面级别的业务校验，若校验不通过可调用 `e.setCancel(true)` 阻止操作继续执行。也可用于弹出二次确认对话框。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | BeforeDoOperationEventArgs | 事件参数对象 |
| e.getSource() | Object | 获取事件源；如需操作 key，需继续确认源对象类型 |
| `operate.getOperateKey()` | string | 在源对象可确认为 `FormOperate` 时读取当前操作标识 |
| e.getSource() | FormOperate | 获取操作源对象，可进一步获取操作选项 |
| e.setCancel(boolean) | void | 设置为 true 可取消本次操作 |
| e.setCancelMessage(string) | void | 设置取消操作时的提示消息 |

## 业务场景

提交采购订单前，校验分录合计金额是否与单据头金额一致。如果不一致则阻止提交并提示用户修正，确保数据准确性。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { BeforeDoOperationEventArgs } from "@cosmic/bos-core/kd/bos/form/events";
import { FormOperate } from "@cosmic/bos-core/kd/bos/form/operate";

/**
 * 采购订单表单插件 - 提交前校验分录合计金额与单据头金额一致
 */
class PmPurorderAmountCheckPlugin extends AbstractBillPlugIn {

  beforeDoOperation(e: BeforeDoOperationEventArgs): void {
    super.beforeDoOperation(e);

    const operate = e.getSource() as FormOperate;
    const opKey = operate.getOperateKey();

    // 仅在提交操作时进行校验
    if (opKey === "submit") {
      // 校验分录是否为空
      const rowCount = this.getModel().getEntryRowCount("billentry");
      if (rowCount === 0) {
        e.setCancel(true);
        this.getView().showTipNotification("请至少添加一行分录后再提交");
        return;
      }

      // 计算分录合计金额
      let entryTotalAmount = BigDecimal.ZERO;
      for (let i = 0; i < rowCount; i++) {
        const amount = this.getModel().getValue("amount", i) as BigDecimal;
        if (amount != null) {
          entryTotalAmount = entryTotalAmount.add(amount);
        }
      }

      // 获取单据头合计金额
      const headTotalAmount = this.getModel().getValue("totalamount") as BigDecimal;

      if (headTotalAmount == null) {
        e.setCancel(true);
        this.getView().showTipNotification("单据头合计金额不能为空");
        return;
      }

      // 比较分录合计与单据头金额是否一致（允许0.01的精度误差）
      const diff = entryTotalAmount.subtract(headTotalAmount).abs();
      const tolerance = new BigDecimal("0.01");

      if (diff.compareTo(tolerance) > 0) {
        e.setCancel(true);
        this.getView().showTipNotification(
          "分录合计金额（" + entryTotalAmount.toString() +
          "）与单据头金额（" + headTotalAmount.toString() +
          "）不一致，差额：" + diff.toString() + "，请检查后再提交"
        );
        return;
      }

      // 校验每行分录的数量和单价是否为正数
      for (let i = 0; i < rowCount; i++) {
        const qty = this.getModel().getValue("qty", i) as BigDecimal;
        const price = this.getModel().getValue("price", i) as BigDecimal;
        const rowNum = i + 1;

        if (qty == null || qty.compareTo(BigDecimal.ZERO) <= 0) {
          e.setCancel(true);
          this.getView().showTipNotification("第" + rowNum + "行数量必须大于0");
          return;
        }

        if (price == null || price.compareTo(BigDecimal.ZERO) < 0) {
          e.setCancel(true);
          this.getView().showTipNotification("第" + rowNum + "行单价不能为负数");
          return;
        }
      }
    }
  }
}

let plugin = new PmPurorderAmountCheckPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.beforeDoOperation(e)` 以确保父类逻辑正常执行
- 需要区分操作类型时，先确认操作 key 是挂在当前事件参数上，还是挂在 `e.getSource()` 返回的操作对象上
- `e.setCancel(true)` 会阻止操作继续执行，用户界面不会有任何操作动作
- 此方法属于界面级校验，适合做数据完整性检查；更严格的业务规则校验应放在操作插件中
- 金额比较时建议使用 BigDecimal 的 `compareTo` 方法，不要直接用 `==` 比较
- 校验不通过时应给出明确的错误提示，帮助用户定位问题
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# afterDoOperation - 操作后事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 操作（如保存、提交、审核等）执行完成之后触发，可根据操作结果执行后续逻辑 |
| 方法签名 | `afterDoOperation(e: AfterDoOperationEventArgs): void` |

## 说明

在操作执行完成后触发的事件回调。通过该方法可以获取操作执行结果，根据操作是否成功来执行后续业务逻辑，如刷新界面、关闭页面、触发下游操作等。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | AfterDoOperationEventArgs | 事件参数对象 |
| e.getOperateKey() | string | 获取当前操作标识（如 save、submit、audit） |
| e.getOperationResult() | OperationResult | 获取操作执行结果对象 |
| e.getOperationResult().isSuccess() | boolean | 判断操作是否执行成功 |
| e.getOperationResult().getMessage() | string | 获取操作结果消息 |

## 业务场景

采购订单审核通过后，自动触发下推操作生成采购入库单，实现审核即入库的业务流程自动化。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { AfterDoOperationEventArgs } from "@cosmic/bos-core/kd/bos/form/events";
import { OperationServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper/operation";
import { ConvertServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { PushArgs } from "@cosmic/bos-core/kd/bos/entity/botp/runtime";

/**
 * 采购订单表单插件 - 审核通过后自动下推生成采购入库单
 */
class PmPurorderAutoPushPlugin extends AbstractBillPlugIn {

  afterDoOperation(e: AfterDoOperationEventArgs): void {
    super.afterDoOperation(e);

    const opKey = e.getOperateKey();

    // 仅在审核操作成功后触发
    if (opKey === "audit" &&
        e.getOperationResult() != null &&
        e.getOperationResult().isSuccess()) {

      // 获取当前单据ID
      const billId = this.getModel().getDataEntity().getPkValue();

      // 执行下推操作
      this.autoPushToInboundBill(billId);
    }

    // 提交成功后提示
    if (opKey === "submit" &&
        e.getOperationResult() != null &&
        e.getOperationResult().isSuccess()) {
      this.getView().showSuccessNotification("单据提交成功，请等待审核");
    }
  }

  /**
   * 自动下推生成采购入库单
   */
  private autoPushToInboundBill(sourceBillId: any): void {
    try {
      // 构建下推参数
      const pushArgs = new PushArgs();

      // 设置源单FormId
      pushArgs.setSourceFormId("pm_purorder");

      // 设置目标单FormId
      pushArgs.setTargetFormId("im_purinbill");

      // 设置源单ID列表
      const sourceIds: any[] = [sourceBillId];
      pushArgs.setBillIds(sourceIds);

      // 设置转换规则（可选，不设置则使用默认规则）
      pushArgs.setConvertRuleKey("purorder_to_purinbill");

      // 执行下推
      const pushResult = ConvertServiceHelper.push(pushArgs);

      if (pushResult != null && pushResult.isSuccess()) {
        // 获取下推生成的目标单据
        const targetBills = pushResult.getTargetDataEntities();

        if (targetBills != null && targetBills.length > 0) {
          // 保存目标单据
          const saveResult = OperationServiceHelper.executeOperate(
            "save",
            "im_purinbill",
            targetBills,
            null
          );

          if (saveResult.isSuccess()) {
            this.getView().showSuccessNotification("已自动生成采购入库单");
          } else {
            this.getView().showTipNotification("采购入库单保存失败，请手动创建");
          }
        }
      } else {
        this.getView().showTipNotification("下推生成采购入库单失败，请手动操作");
      }
    } catch (error: any) {
      this.getView().showErrorNotification("自动下推异常：" + error.message);
    }
  }
}

let plugin = new PmPurorderAutoPushPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.afterDoOperation(e)` 以确保父类逻辑正常执行
- 务必通过 `e.getOperationResult().isSuccess()` 判断操作是否成功，只有成功时才执行后续逻辑
- 下推操作可能因为转换规则、数据校验等原因失败，需做好异常处理
- 自动下推生成的目标单据需要手动调用保存操作才能持久化
- 此方法属于界面级事件，如果需要在无界面场景（如批量操作）中实现类似逻辑，应使用操作插件的 `afterExecuteOperationTransaction`
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# beforeClosed - 关闭前操作

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 页面关闭之前触发，可在此向父页面回传数据或提示用户保存未提交的修改 |
| 方法签名 | `beforeClosed(e: BeforeClosedEvent): void` |

## 说明

在页面即将关闭时触发的事件回调。常用于弹窗页面关闭前将选中的数据回传给父页面，或在用户关闭编辑页面时提示未保存的修改。通过 `this.getView().returnDataToParent(data)` 将数据回传给父页面。

## 业务场景

自定义弹窗选择页面，用户在弹窗中勾选多条物料数据后点击确认关闭，将选中的物料信息回传给父页面（采购订单），父页面在 `closedCallBack` 中接收数据并填充到分录中。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { BeforeClosedEvent } from "@cosmic/bos-core/kd/bos/form/events";
import { ClosedCallBackEvent } from "@cosmic/bos-core/kd/bos/form/events";

/**
 * 物料选择弹窗插件 - 关闭前回传选中的物料数据给父页面
 */
class MaterialSelectDialogPlugin extends AbstractBillPlugIn {

  beforeClosed(e: BeforeClosedEvent): void {
    super.beforeClosed(e);

    // 获取分录中用户勾选的行
    const entryKey = "billentry";
    const rowCount = this.getModel().getEntryRowCount(entryKey);

    // 收集选中的物料数据
    const selectedMaterials: any[] = [];

    for (let i = 0; i < rowCount; i++) {
      const isSelected = this.getModel().getValue("isselected", i) as boolean;

      if (isSelected) {
        const materialData: any = {};
        materialData.materialId = this.getModel().getValue("material", i);
        materialData.materialName = this.getModel().getValue("materialname", i);
        materialData.materialNumber = this.getModel().getValue("materialnumber", i);
        materialData.specification = this.getModel().getValue("specification", i);
        materialData.unitId = this.getModel().getValue("unit", i);
        materialData.price = this.getModel().getValue("price", i);
        materialData.qty = this.getModel().getValue("qty", i);

        selectedMaterials.push(materialData);
      }
    }

    // 将选中的数据回传给父页面
    if (selectedMaterials.length > 0) {
      this.getView().returnDataToParent(selectedMaterials);
    }
  }
}
```

```typescript
  }
```

## 注意事项

- 必须调用 `super.beforeClosed(e)` 以确保父类逻辑正常执行
- `this.getView().returnDataToParent(data)` 用于向父页面回传数据，父页面在 `closedCallBack` 中接收
- 回传的数据类型可以是任意可序列化对象，包括数组、对象等
- 父页面需要在打开弹窗时通过 `CloseCallBack` 设置回调标识，在 `closedCallBack` 中通过 `getActionId()` 匹配
- 如果弹窗中没有选中任何数据，建议不调用 `returnDataToParent` 或传回空标记，父页面做好空值判断
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# beforeCheckDataPermission - 数据权限验权前事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 系统进行数据权限校验之前触发，可在此跳过或自定义数据权限检查逻辑 |
| 方法签名 | `beforeDoCheckDataPermission(e: BeforeDoCheckDataPermissionArgs): void` |

## 说明

在系统执行数据权限校验之前触发的事件回调。通过该方法可以根据业务需要跳过数据权限检查，或自定义权限校验逻辑。例如，某些特殊角色（如总经理、审计人员）需要跨组织查看所有数据时，可以在此方法中跳过权限校验。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | BeforeDoCheckDataPermissionArgs | 事件参数对象 |
| e.setCancel(boolean) | void | 设置为 true 可跳过本次数据权限校验 |
| e.getFormId() | string | 获取当前表单标识 |
| e.getPermissionItemId() | string | 获取权限项ID |

## 业务场景

特殊角色（如总经理、审计部门）在查看采购订单时需要跳过数据权限校验，可查看所有组织的数据，不受数据权限规则限制。其他普通角色仍按正常权限进行校验。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 采购订单表单插件 - 特殊角色跳过数据权限校验
 */
class PmPurorderPermissionPlugin extends AbstractBillPlugIn {

  beforeDoCheckDataPermission(e: BeforeDoCheckDataPermissionArgs): void {
    super.beforeDoCheckDataPermission(e);

    // 获取当前登录用户ID
    const currentUserId = RequestContext.get().getCurrUserId();

    // 判断当前用户是否属于特殊角色（总经理、审计部门）
    const isSpecialRole = this.checkSpecialRole(currentUserId);

    if (isSpecialRole) {
      // 跳过数据权限校验，允许查看所有组织数据
      e.setCancel(true);
    }
  }

  /**
   * 检查用户是否属于特殊角色
   */
  private checkSpecialRole(userId: any): boolean {
    // 查询用户所属角色，判断是否包含总经理或审计角色
    const roleInfo = QueryServiceHelper.query(
      "sec_userrole",
      "role.number",
      [
        new QFilter("user", "=", userId),
        new QFilter("role.number", "in", ["GM", "AUDIT_MANAGER"])
      ]
    );

    if (roleInfo != null) {
      while (roleInfo.next()) {
        const roleNumber = roleInfo.get("role.number") as string;
        if (roleNumber === "GM" || roleNumber === "AUDIT_MANAGER") {
          return true;
        }
      }
    }

    return false;
  }
}

let plugin = new PmPurorderPermissionPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.beforeCheckDataPermission(e)` 以确保父类逻辑正常执行
- `e.setCancel(true)` 会跳过数据权限校验，使用时务必谨慎，确保符合安全策略
- 跳过权限校验应仅针对明确的特殊角色，避免无差别跳过导致数据泄露风险
- 角色判断逻辑建议使用角色编码而非角色名称，避免因名称变更导致逻辑失效
- 此方法每次打开或操作单据时都可能触发，注意查询性能，可考虑缓存角色判断结果
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量
- 在生产环境中，建议将特殊角色编码配置化，而非硬编码在插件中

# beforeFieldPostBack - 客户端字段值变动回传事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 客户端字段值发生变动准备回传服务器之前触发，可在此拦截不必要的回传请求 |
| 方法签名 | `beforeFieldPostBack(e: BeforeFieldPostBackEvent): void` |

## 说明

当客户端（浏览器端）字段值发生变动，准备将变更回传到服务器端之前触发的事件回调。通过该方法可以控制哪些字段的值变更需要回传服务器处理，哪些不需要。对于不需要服务端处理的字段值变更，可以取消回传以减少网络请求，提升界面操作性能。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | any | 事件参数对象 |
| e.getFieldKey() | string | 获取发生变动的字段标识 |
| e.setCancel(boolean) | void | 设置为 true 可取消本次回传，字段值变更不会发送到服务器 |

## 业务场景

采购订单中有多个文本类备注字段（如行备注、内部备注、物流备注），这些字段的值变更不需要触发服务端联动计算。通过拦截这些字段的回传请求，减少不必要的网络交互，提高用户录单效率。同时保留关键字段（如数量、单价、物料）的回传，确保联动计算正常触发。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 采购订单表单插件 - 控制字段值变更回传策略以提升性能
 */
class PmPurorderPostBackControlPlugin extends AbstractBillPlugIn {

  beforeFieldPostBack(e: BeforeFieldPostBackEvent): void {
    super.beforeFieldPostBack(e);

    const fieldKey = e.getFieldKey();

    // 定义不需要回传服务器的字段列表
    // 这些字段的值变更不涉及服务端联动计算
    const skipPostBackFields = [
      "entryremark",       // 行备注
      "internalremark",    // 内部备注
      "logisticsremark",   // 物流备注
      "description",       // 描述信息
      "headremark",        // 单据头备注
      "deliveryaddress",   // 收货地址（纯文本）
      "contactperson",     // 联系人
      "contactphone"       // 联系电话
    ];

    // 判断当前变动字段是否在跳过列表中
    for (let i = 0; i < skipPostBackFields.length; i++) {
      if (fieldKey === skipPostBackFields[i]) {
        // 取消回传，减少不必要的服务端请求
        e.setCancel(true);
        return;
      }
    }

    // 以下字段值变更需要正常回传服务器（默认行为，无需额外处理）：
    // - qty（数量）：触发金额联动计算
    // - price（单价）：触发金额联动计算
    // - material（物料）：触发带出物料属性
    // - supplier（供应商）：触发带出供应商信息
    // - currency（币别）：触发汇率联动
  }
}

let plugin = new PmPurorderPostBackControlPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.beforeFieldPostBack(e)` 以确保父类逻辑正常执行
- `e.setCancel(true)` 取消回传后，服务端的 `propertyChanged` 等事件将不会针对该字段触发
- 只应对纯展示或纯录入字段（如备注、描述）取消回传，不要对参与联动计算的字段取消
- 取消回传不影响字段值的最终保存，保存操作时所有字段值仍会完整提交到服务器
- 在分录行较多、字段较多的单据中，合理使用此方法可以显著提升录单效率
- 如果某个字段的 `propertyChanged` 事件中有重要业务逻辑，不应取消该字段的回传
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# clientCallBack - 客户端回调事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 服务端调用客户端JS执行完毕后，客户端将结果回传到服务端时触发 |
| 方法签名 | `clientCallBack(e: ClientCallBackEvent): void` |

## 说明

用于服务端与客户端JS之间的异步交互。服务端通过 `this.getView().addClientCallBack(jsCode)` 向客户端发送JS代码执行请求，客户端执行完毕后将结果回传到服务端，触发此事件。常用于获取客户端浏览器信息、调用客户端特定API等场景。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | ClientCallBackEvent | 事件参数对象 |
| e.getCallBackId() | string | 获取回调标识，与发起调用时设置的标识一致 |
| e.getReturnData() | string | 获取客户端JS执行后回传的数据（字符串格式） |

## 业务场景

用户登录系统打开采购订单页面时，通过客户端JS获取浏览器类型、操作系统、屏幕分辨率等设备信息，然后在服务端记录用户的登录设备信息，用于安全审计和运维分析。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";
import { SaveServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { BusinessDataServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";

/**
 * 采购订单表单插件 - 获取客户端设备信息并记录
 */
class PmPurorderDeviceInfoPlugin extends AbstractBillPlugIn {

  afterBindData(e: $.java.util.EventObject): void {
    super.afterBindData(e);

    // 向客户端发送JS代码，获取浏览器和设备信息
    const jsCode = `
      (function() {
        var info = {};
        info.userAgent = navigator.userAgent;
        info.platform = navigator.platform;
        info.language = navigator.language;
        info.screenWidth = screen.width;
        info.screenHeight = screen.height;
        info.windowWidth = window.innerWidth;
        info.windowHeight = window.innerHeight;
        info.cookieEnabled = navigator.cookieEnabled;
        info.timestamp = new Date().toISOString();
        return JSON.stringify(info);
      })()
    `;

    // 发起客户端回调，标识为 "getDeviceInfo"
    this.getView().addClientCallBack(jsCode, "getDeviceInfo");
  }

  clientCallBack(e: ClientCallBackEvent): void {
    super.clientCallBack(e);

    const callBackId = e.getCallBackId();

    if (callBackId === "getDeviceInfo") {
      const returnData = e.getReturnData() as string;

      if (returnData == null || returnData === "") {
        return;
      }

      try {
        // 解析客户端回传的设备信息JSON
        const deviceInfo = JSON.parse(returnData);

        // 获取当前用户信息
        const userId = RequestContext.get().getCurrUserId();
        const userName = RequestContext.get().getUserName();

        // 解析浏览器类型
        const browserType = this.parseBrowserType(deviceInfo.userAgent);

        // 构建日志记录内容
        const logContent = "用户：" + userName +
          "，浏览器：" + browserType +
          "，系统：" + deviceInfo.platform +
          "，分辨率：" + deviceInfo.screenWidth + "x" + deviceInfo.screenHeight +
          "，登录时间：" + deviceInfo.timestamp;

        // 创建设备登录日志记录
        const logEntity = BusinessDataServiceHelper.newDynamicObject("custom_device_log");
        logEntity.set("user", userId);
        logEntity.set("browsertype", browserType);
        logEntity.set("useragent", deviceInfo.userAgent);
        logEntity.set("platform", deviceInfo.platform);
        logEntity.set("screenresolution", deviceInfo.screenWidth + "x" + deviceInfo.screenHeight);
        logEntity.set("logintime", new Date());
        logEntity.set("logcontent", logContent);

        // 保存日志记录
        SaveServiceHelper.save("custom_device_log", [logEntity]);

      } catch (error: any) {
        // 设备信息记录失败不影响正常业务
        this.getView().showTipNotification("设备信息记录失败：" + error.message);
      }
    }
  }

  /**
   * 解析浏览器类型
   */
  private parseBrowserType(userAgent: string): string {
    if (userAgent == null) {
      return "Unknown";
    }
    if (userAgent.indexOf("Chrome") > -1 && userAgent.indexOf("Edg") === -1) {
      return "Chrome";
    } else if (userAgent.indexOf("Firefox") > -1) {
      return "Firefox";
    } else if (userAgent.indexOf("Edg") > -1) {
      return "Edge";
    } else if (userAgent.indexOf("Safari") > -1) {
      return "Safari";
    }
    return "Other";
  }
}

let plugin = new PmPurorderDeviceInfoPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.clientCallBack(e)` 以确保父类逻辑正常执行
- `addClientCallBack` 发送的JS代码在客户端浏览器中执行，需注意浏览器兼容性
- 客户端回调是异步的，服务端发起调用后不会阻塞等待结果
- 回传数据为字符串格式，复杂数据需使用 JSON 序列化/反序列化
- 通过 `callBackId` 区分不同的回调请求，确保对应正确的业务处理逻辑
- 客户端JS代码中不能访问服务端对象，两端数据交互仅通过字符串传递
- 客户端回调失败不应影响主业务流程，建议做好异常处理
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

# contextMenuClick - 菜单点击事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 用户点击右键菜单项或自定义菜单项时触发 |
| 方法签名 | `contextMenuClick(e: ContextMenuClickEvent): void` |

## 说明

当用户在单据体分录区域点击右键菜单中的菜单项时触发的事件回调。可通过此方法响应自定义的右键菜单操作，如复制行、插入行、批量修改等。系统默认的右键菜单包括新增行、删除行等操作，开发者可以添加自定义菜单项并在此方法中处理。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | ContextMenuClickEvent | 事件参数对象 |
| e.getItemKey() | string | 获取被点击的菜单项标识 |
| e.getEntryKey() | string | 获取菜单所在的单据体标识 |
| e.getRow() | number | 获取右键点击所在的分录行索引 |

## 业务场景

在采购订单的分录区域，用户右键点击"复制行"菜单项时，将当前选中行的物料、数量、单价等数据完整复制到一个新的分录行中，方便用户快速录入相似的采购明细。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 采购订单表单插件 - 右键菜单复制分录行
 */
class PmPurorderCopyRowPlugin extends AbstractBillPlugIn {

  contextMenuClick(e: ContextMenuClickEvent): void {
    super.contextMenuClick(e);

    const menuKey = e.getItemKey();

    if (menuKey === "copy_row") {
      // 复制行操作
      this.copyEntryRow(e);
    } else if (menuKey === "batch_set_delivery_date") {
      // 批量设置交货日期
      this.batchSetDeliveryDate();
    } else if (menuKey === "clear_row_data") {
      // 清空当前行数据（保留行）
      this.clearRowData(e);
    }
  }

  /**
   * 复制当前分录行数据到新行
   */
  private copyEntryRow(e: ContextMenuClickEvent): void {
    const sourceRowIndex = e.getRow();
    const entryKey = "billentry";

    // 读取源行数据
    const materialId = this.getModel().getValue("material", sourceRowIndex);
    const materialName = this.getModel().getValue("materialname", sourceRowIndex);
    const unitId = this.getModel().getValue("unit", sourceRowIndex);
    const qty = this.getModel().getValue("qty", sourceRowIndex) as BigDecimal;
    const price = this.getModel().getValue("price", sourceRowIndex) as BigDecimal;
    const taxRate = this.getModel().getValue("taxrate", sourceRowIndex) as BigDecimal;
    const warehouseId = this.getModel().getValue("entrywarehouse", sourceRowIndex);
    const remark = this.getModel().getValue("entryremark", sourceRowIndex) as string;
    const deliveryDate = this.getModel().getValue("deliverydate", sourceRowIndex);

    // 在源行下方新增一行
    const newRowIndex = this.getModel().createNewEntryRow(entryKey);

    // 将源行数据复制到新行
    if (materialId != null) {
      this.getModel().setValue("material", materialId, newRowIndex);
    }
    if (unitId != null) {
      this.getModel().setValue("unit", unitId, newRowIndex);
    }
    if (qty != null) {
      this.getModel().setValue("qty", qty, newRowIndex);
    }
    if (price != null) {
      this.getModel().setValue("price", price, newRowIndex);
    }
    if (taxRate != null) {
      this.getModel().setValue("taxrate", taxRate, newRowIndex);
    }
    if (warehouseId != null) {
      this.getModel().setValue("entrywarehouse", warehouseId, newRowIndex);
    }
    if (remark != null) {
      this.getModel().setValue("entryremark", remark, newRowIndex);
    }
    if (deliveryDate != null) {
      this.getModel().setValue("deliverydate", deliveryDate, newRowIndex);
    }

    // 重新计算新行金额
    if (qty != null && price != null) {
      const amount = qty.multiply(price).setScale(2, BigDecimal.ROUND_HALF_UP);
      this.getModel().setValue("amount", amount, newRowIndex);
    }

    this.getView().showSuccessNotification("已复制第" + (sourceRowIndex + 1) + "行到第" + (newRowIndex + 1) + "行");
  }

  /**
   * 批量设置所有分录行的交货日期为当前日期后7天
   */
  private batchSetDeliveryDate(): void {
    const rowCount = this.getModel().getEntryRowCount("billentry");
    const today = new Date();
    const deliveryDate = new Date(today.getTime() + 7 * 24 * 60 * 60 * 1000);

    for (let i = 0; i < rowCount; i++) {
      this.getModel().setValue("deliverydate", deliveryDate, i);
    }

    this.getView().showSuccessNotification("已批量设置交货日期为7天后");
  }

  /**
   * 清空当前行的数据但保留行
   */
  private clearRowData(e: ContextMenuClickEvent): void {
    const rowIndex = e.getRow();

    this.getModel().setValue("material", null, rowIndex);
    this.getModel().setValue("qty", BigDecimal.ZERO, rowIndex);
    this.getModel().setValue("price", BigDecimal.ZERO, rowIndex);
    this.getModel().setValue("amount", BigDecimal.ZERO, rowIndex);
    this.getModel().setValue("entryremark", "", rowIndex);

    this.getView().showSuccessNotification("已清空第" + (rowIndex + 1) + "行数据");
  }
}

let plugin = new PmPurorderCopyRowPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.contextMenuClick(e)` 以确保父类逻辑正常执行
- 自定义右键菜单项需要在表单设计器中配置，或通过 `registerListener` 动态添加
- `e.getRow()` 获取的是用户右键点击时所在的分录行索引，从0开始
- 复制行时注意不要复制系统自动生成的字段值（如行ID、序号等），这些由系统自动处理
- 复制基础资料字段时直接复制其ID即可，系统会自动带出对应的名称和编码
- 如果复制的字段存在联动逻辑，需注意联动计算的触发顺序
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

| e.getName() | string | 获取事件名称，由前端 fireEvent 时指定 |
| e.getEventArgs() | string | 获取事件携带的参数数据（字符串格式） |
| e.getKey() | string | 获取触发事件的控件标识 |

## 业务场景

在仓库入库单中，用户使用扫码枪扫描物料条码，前端自定义扫码控件接收到条码数据后通过 `fireEvent` 将条码信息发送到服务端，服务端解析条码查询物料信息，并自动新增分录行填入物料、数量等数据，实现快速扫码录入功能。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

/**
 * 仓库入库单表单插件 - 扫码录入物料
 */
class ImPurinbillScanPlugin extends AbstractBillPlugIn {

  customEvent(e: CustomEventArgs): void {
    super.customEvent(e);

    const eventName = e.getName();
    const controlKey = e.getKey();

    // 处理扫码控件触发的扫码完成事件
    if (controlKey === "custom_scanner" && eventName === "scan_complete") {
      const scanData = e.getEventArgs() as string;
      this.handleScanResult(scanData);
    }

    // 处理自定义地图控件的定位选择事件
    if (controlKey === "custom_map" && eventName === "location_selected") {
      const locationData = e.getEventArgs() as string;
      this.handleLocationSelect(locationData);
    }
  }

  /**
   * 处理扫码结果
   */
  private handleScanResult(scanData: string): void {
    if (scanData == null || scanData === "") {
      this.getView().showTipNotification("扫码数据为空，请重新扫描");
      return;
    }

    try {
      // 解析扫码数据（假设条码格式为：物料编码|批号|数量）
      const parts = scanData.split("|");
      const materialNumber = parts[0];
      const batchNo = parts.length > 1 ? parts[1] : "";
      const scanQty = parts.length > 2 ? new BigDecimal(parts[2]) : new BigDecimal("1");

      // 根据物料编码查询物料信息
      const materialInfo = QueryServiceHelper.queryOne(
        "bd_material",
        "id, name, number, baseunit, baseunit.name as unitname",
        [
          new QFilter("number", "=", materialNumber),
          new QFilter("status", "=", "C")
        ]
      );

      if (materialInfo == null) {
        this.getView().showTipNotification("未找到编码为 " + materialNumber + " 的物料，请检查条码");
        return;
      }

      // 检查分录中是否已存在相同物料和批号的行
      const existingRowIndex = this.findExistingRow(materialNumber, batchNo);

      if (existingRowIndex >= 0) {
        // 已存在相同物料行，累加数量
        const currentQty = this.getModel().getValue("qty", existingRowIndex) as BigDecimal;
        const newQty = currentQty != null ? currentQty.add(scanQty) : scanQty;
        this.getModel().setValue("qty", newQty, existingRowIndex);

        // 重新计算金额
        const price = this.getModel().getValue("price", existingRowIndex) as BigDecimal;
        if (price != null) {
          const amount = newQty.multiply(price).setScale(2, BigDecimal.ROUND_HALF_UP);
          this.getModel().setValue("amount", amount, existingRowIndex);
        }

        this.getView().showSuccessNotification(
          "物料 " + materialNumber + " 数量已累加至 " + newQty.toString()
        );
      } else {
        // 新增分录行
        const newRowIndex = this.getModel().createNewEntryRow("billentry");

        // 填充物料信息
        const materialId = materialInfo.get("id");
        this.getModel().setValue("material", materialId, newRowIndex);
        this.getModel().setValue("qty", scanQty, newRowIndex);

        // 设置批号
        if (batchNo !== "") {
          this.getModel().setValue("batchno", batchNo, newRowIndex);
        }

        // 从单据头带出默认仓库
        const warehouseId = this.getModel().getValue("warehouse");
        if (warehouseId != null) {
          this.getModel().setValue("entrywarehouse", warehouseId, newRowIndex);
        }

        // 设置扫码时间
        this.getModel().setValue("scantime", new Date(), newRowIndex);

        this.getView().showSuccessNotification(
          "已添加物料：" + materialInfo.get("name")
        );
      }
    } catch (error: any) {
      this.getView().showErrorNotification("扫码数据解析失败：" + error.message);
    }
  }

  /**
   * 查找分录中是否已存在相同物料编码和批号的行
   */
  private findExistingRow(materialNumber: string, batchNo: string): number {
    const rowCount = this.getModel().getEntryRowCount("billentry");

    for (let i = 0; i < rowCount; i++) {
      const existNumber = this.getModel().getValue("materialnumber", i) as string;
      const existBatch = this.getModel().getValue("batchno", i) as string;

      if (existNumber === materialNumber) {
        if (batchNo === "" || batchNo === existBatch) {
          return i;
        }
      }
    }
    return -1;
  }

  /**
   * 处理地图定位选择
   */
  private handleLocationSelect(locationData: string): void {
    if (locationData == null) {
      return;
    }

    try {
      const location = JSON.parse(locationData);
      this.getModel().setValue("deliveryaddress", location.address);
      this.getModel().setValue("longitude", location.lng);
      this.getModel().setValue("latitude", location.lat);
    } catch (error: any) {
      this.getView().showTipNotification("位置信息解析失败");
    }
  }
}

let plugin = new ImPurinbillScanPlugin();
export { plugin };
```

|------|------|------|

## 业务场景

打印销售订单时，需要在打印模板中展示二维码数据（包含单据编号和客户信息的编码字符串）和合计金额的大写中文表示。这些数据在单据中不存在，需要在打印时动态计算并添加到打印数据包中。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 销售订单表单插件 - 自定义打印数据包（添加二维码和大写金额）
 */
class SmSaleorderPrintPlugin extends AbstractBillPlugIn {

  customPrintDataObject(e: $.kd.bos.form.events.CustomPrintDataObjectArgs): void {
    super.customPrintDataObject(e);

    const dataObject = e.getDataObject();
    if (dataObject == null) {
      return;
    }

    // 获取单据基本信息
    const billNo = dataObject.get("billno") as string;
    const customerName = dataObject.get("customer_name") as string;
    const totalAmount = dataObject.get("totalamount") as BigDecimal;
    const billDate = dataObject.get("billdate");

    // 1. 生成二维码数据字符串
    const qrCodeData = this.buildQrCodeData(billNo, customerName, totalAmount);
    dataObject.set("qrcode_data", qrCodeData);

    // 2. 生成合计金额大写中文
    if (totalAmount != null) {
      const upperCaseAmount = this.convertToUpperCase(totalAmount);
      dataObject.set("uppercase_amount", upperCaseAmount);
    } else {
      dataObject.set("uppercase_amount", "零元整");
    }

    // 3. 添加打印日期和打印人
    dataObject.set("print_date", this.formatDate(new Date()));

    // 4. 添加分录合计行数
    const entryEntity = dataObject.get("billentry");
    if (entryEntity != null) {
      const rowCount = entryEntity.size();
      dataObject.set("entry_row_count", "共 " + rowCount + " 行");

      // 5. 计算分录汇总信息
      let totalQty = BigDecimal.ZERO;
      let totalTaxAmount = BigDecimal.ZERO;

      for (let i = 0; i < rowCount; i++) {
        const row = entryEntity.get(i);
        const qty = row.get("qty") as BigDecimal;
        const taxAmount = row.get("taxamount") as BigDecimal;

        if (qty != null) {
          totalQty = totalQty.add(qty);
        }
        if (taxAmount != null) {
          totalTaxAmount = totalTaxAmount.add(taxAmount);
        }
      }

      dataObject.set("total_qty", totalQty.toString());
      dataObject.set("total_tax_amount", totalTaxAmount.setScale(2, BigDecimal.ROUND_HALF_UP).toString());
      dataObject.set("uppercase_tax_amount", this.convertToUpperCase(totalTaxAmount));
    }
  }

  /**
   * 构建二维码数据字符串
   */
  private buildQrCodeData(billNo: string, customerName: string, amount: BigDecimal): string {
    const qrData: any = {};
    qrData.billNo = billNo != null ? billNo : "";
    qrData.customer = customerName != null ? customerName : "";
    qrData.amount = amount != null ? amount.toString() : "0";
    qrData.timestamp = new Date().getTime();

    return JSON.stringify(qrData);
  }

  /**
   * 将金额转换为大写中文
   */
  private convertToUpperCase(amount: BigDecimal): string {
    if (amount == null || amount.compareTo(BigDecimal.ZERO) === 0) {
      return "零元整";
    }

    const digitChars = ["零", "壹", "贰", "叁", "肆", "伍", "陆", "柒", "捌", "玖"];
    const unitChars = ["", "拾", "佰", "仟"];
    const sectionUnits = ["", "万", "亿", "兆"];

    // 处理负数
    let isNegative = false;
    let absAmount = amount;
    if (amount.compareTo(BigDecimal.ZERO) < 0) {
      isNegative = true;
      absAmount = amount.abs();
    }

    // 分离整数部分和小数部分
    const amountStr = absAmount.setScale(2, BigDecimal.ROUND_HALF_UP).toString();
    const dotIndex = amountStr.indexOf(".");
    const integerPart = dotIndex > 0 ? amountStr.substring(0, dotIndex) : amountStr;
    const decimalPart = dotIndex > 0 ? amountStr.substring(dotIndex + 1) : "00";

    let result = "";

    // 处理整数部分
    if (integerPart === "0") {
      result = "零";
    } else {
      const intLen = integerPart.length;
      let zeroFlag = false;

      for (let i = 0; i < intLen; i++) {
        const digit = parseInt(integerPart.charAt(i));
        const posFromRight = intLen - 1 - i;
        const section = Math.floor(posFromRight / 4);
        const unitIndex = posFromRight % 4;

        if (digit === 0) {
          zeroFlag = true;
          if (unitIndex === 0 && section > 0) {
            result += sectionUnits[section];
          }
        } else {
          if (zeroFlag) {
            result += "零";
            zeroFlag = false;
          }
          result += digitChars[digit] + unitChars[unitIndex];
          if (unitIndex === 0 && section > 0) {
            result += sectionUnits[section];
          }
        }
      }
    }

    result += "元";

    // 处理小数部分（角和分）
    const jiao = parseInt(decimalPart.charAt(0));
    const fen = decimalPart.length > 1 ? parseInt(decimalPart.charAt(1)) : 0;

    if (jiao === 0 && fen === 0) {
      result += "整";
    } else {
      if (jiao > 0) {
        result += digitChars[jiao] + "角";
      } else if (fen > 0) {
        result += "零";
      }
      if (fen > 0) {
        result += digitChars[fen] + "分";
      }
    }

    if (isNegative) {
      result = "负" + result;
    }

    return result;
  }

  /**
   * 格式化日期为 yyyy-MM-dd 格式
   */
  private formatDate(date: Date): string {
    const year = date.getFullYear();
    const month = (date.getMonth() + 1).toString().padStart(2, "0");
    const day = date.getDate().toString().padStart(2, "0");
    return year + "-" + month + "-" + day;
  }
}

let plugin = new SmSaleorderPrintPlugin();
export { plugin };
```

- 必须调用 `super.customPrintDataObject(e)` 以确保父类逻辑正常执行

- 二维码数据字段需在打印模板中配置为二维码控件并绑定对应的字段名

## 业务场景

表单关闭时释放插件中注册的定时器和外部资源连接。在一个实时监控看板中，插件启动了定时器用于轮询数据，当用户关闭该看板页面时，需要停止定时器并释放相关资源。

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { TimerElapsedArgs } from "@cosmic/bos-core/kd/bos/form/events";

/**
 * 实时监控看板插件 - 关闭时释放定时器和外部资源
 */
class MonitorDashboardPlugin extends AbstractBillPlugIn {

  afterBindData(e: $.java.util.EventObject): void {
    super.afterBindData(e);

    // 启动定时器，每30秒刷新一次看板数据
    this.getView().getPageCache().put("timerStarted", "true");
    this.getView().addTimer("refreshTimer", 30000);
  }

  timerElapsed(e: TimerElapsedArgs): void {
    super.timerElapsed(e);

    const timerKey = e.getTimerKey();
    if (timerKey === "refreshTimer") {
      // 刷新看板数据
      this.getView().updateView("dashboardpanel");
    }
  }

    super.destory();

    // 停止定时器
    const timerStarted = this.getView().getPageCache().get("timerStarted");
    if (timerStarted === "true") {
      this.getView().removeTimer("refreshTimer");
      this.getView().getPageCache().remove("timerStarted");
    }

    // 清理页面缓存中的临时数据
    this.getView().getPageCache().remove("monitorConfig");
    this.getView().getPageCache().remove("lastRefreshTime");
  }
}
```

# messageBoxClosed - 关闭消息提示窗口

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 用户在消息提示框中点击按钮（如确定、取消）并关闭后触发 |
| 方法签名 | `messageBoxClosed(e: MessageBoxClosedEvent): void` |

## 说明

消息提示窗口关闭后的回调事件。当通过 `showConfirmMessage` 或 `showYesNoMessage` 等方法弹出消息提示框后，用户点击按钮关闭消息框时触发此方法。通过该方法可获取用户的选择结果，并据此执行后续业务逻辑。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | MessageBoxClosedEvent | 消息框关闭事件参数对象 |
| e.getCallBackId() | string | 获取消息框的回调标识，用于区分不同的消息提示 |
| e.getResult() | string | 获取用户的选择结果，如 "yes"、"no"、"cancel" |

## 业务场景

消息提示框关闭后，根据用户选择决定是否继续执行保存操作。在采购订单中，当单据金额超过审批限额时弹出确认提示，用户确认后继续保存，取消则中止操作。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { BeforeFieldPostBackEvent } from "@cosmic/bos-core/kd/bos/form/events";
import { BeforeDoOperationEventArgs, MessageBoxClosedEvent } from "@cosmic/bos-core/kd/bos/form/events";
import { FormOperate } from "@cosmic/bos-core/kd/bos/form/operate";

/**
 * 采购订单插件 - 金额超限确认后继续保存
 */
class PmPurorderConfirmPlugin extends AbstractBillPlugIn {

  beforeDoOperation(e: BeforeDoOperationEventArgs): void {
    super.beforeDoOperation(e);

    const operate = e.getSource() as FormOperate;
    const opKey = operate.getOperateKey();

    if (opKey === "save") {
      // 获取单据合计金额
      const totalAmount = this.getModel().getValue("totalamount") as BigDecimal;
      const limitAmount = new BigDecimal("100000");

      // 金额超过10万时弹出确认提示
      if (totalAmount != null && totalAmount.compareTo(limitAmount) > 0) {
        // 检查是否已经确认过
        const confirmed = this.getView().getPageCache().get("amountConfirmed");
        if (confirmed !== "true") {
          e.setCancel(true);
          this.getView().showConfirmMessage(
            "采购金额超过10万元（当前金额：" + totalAmount.toString() + "），是否确认保存？",
            "amountOverLimitConfirm"
          );
        }
      }
    }
  }

  messageBoxClosed(e: MessageBoxClosedEvent): void {
    super.messageBoxClosed(e);

    const callBackId = e.getCallBackId();
    const result = e.getResult();

    if (callBackId === "amountOverLimitConfirm") {
      if (result === "yes") {
        // 用户确认，标记已确认并重新触发保存
        this.getView().getPageCache().put("amountConfirmed", "true");
        this.getView().invokeOperation("save");

        // 保存完成后清除确认标记
        this.getView().getPageCache().remove("amountConfirmed");
      } else {
        this.getView().showTipNotification("已取消保存操作");
      }
    }
  }
}

let plugin = new PmPurorderConfirmPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.messageBoxClosed(e)` 以确保父类逻辑正常执行
- `e.getCallBackId()` 是区分不同消息框的关键，弹出消息时需要指定唯一的回调标识
- `e.getResult()` 的返回值取决于消息框类型："yes"/"no" 对应确认/取消
- 在 `beforeDoOperation` 中取消操作后弹出确认框，用户确认后需通过 `invokeOperation` 重新触发操作
- 使用 `getPageCache()` 做确认标记，避免重新触发操作时再次弹出确认框导致死循环
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量

## 说明

获取指定控件的回调事件。当系统根据控件标识查找控件实例时触发此方法。插件可以在此方法中拦截控件获取过程，动态创建并返回自定义的控件实例，实现程序化构建界面的能力。

```typescript
 * 动态报表插件 - 动态创建自定义展示控件
 */
class DynamicReportControlPlugin extends AbstractBillPlugIn {

  onGetControl(e: OnGetControlArgs): void {
    super.onGetControl(e);

    const controlKey = e.getControlKey();

    // 拦截统计卡片控件的获取请求
    if (controlKey === "stat_card_total") {
      const totalAmount = this.getModel().getValue("totalamount") as BigDecimal;
      const displayValue = totalAmount != null ? totalAmount.toString() : "0.00";

      // 创建自定义统计卡片控件

      e.setControl(cardControl);
      return;
    }

    // 拦截动态标签控件的获取请求
    if (controlKey === "dynamic_status_label") {
      const billStatus = this.getModel().getValue("billstatus") as string;

      const labelControl = this.getView().createCustomControl("label");

      if (billStatus === "A") {
        labelControl.setProperty("text", "暂存");
        labelControl.setProperty("style", "color: #999999;");
      } else if (billStatus === "B") {
        labelControl.setProperty("text", "已提交");
        labelControl.setProperty("style", "color: #faad14;");
      } else if (billStatus === "C") {
        labelControl.setProperty("text", "已审核");
        labelControl.setProperty("style", "color: #52c41a;");
      } else {
        labelControl.setProperty("text", "未知状态");
        labelControl.setProperty("style", "color: #ff4d4f;");
      }

      e.setControl(labelControl);
      return;
    }
  }
}

let plugin = new DynamicReportControlPlugin();
export { plugin };
```

# pageRelease - 页面释放

| 属性 | 说明 |

## 说明

页面释放时的回调方法。此方法在页面被回收时触发，通常在 `destory` 之后执行。适用于清理页面级别的缓存数据、临时文件、会话级别的临时存储等，确保页面关闭后不会残留无用数据。

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";

/**
 * 单据打印预览插件 - 页面释放时清理缓存和临时数据
 */
class PrintPreviewReleasePlugin extends AbstractBillPlugIn {

  afterBindData(e: $.java.util.EventObject): void {
    super.afterBindData(e);

    // 生成打印预览数据并缓存
    const billNo = this.getModel().getValue("billno") as string;
    const billId = this.getModel().getDataEntity().getPkValue();

    // 将预览相关数据存入页面缓存
    this.getView().getPageCache().put("previewBillId", String(billId));
    this.getView().getPageCache().put("previewBillNo", billNo);
    this.getView().getPageCache().put("previewTimestamp", String(new Date().getTime()));

    // 缓存打印配置信息
    const printConfig = {
      paperSize: "A4",
      orientation: "portrait",
      copies: 1
    };
    this.getView().getPageCache().put("printConfig", JSON.stringify(printConfig));
  }

  pageRelease(): void {
    super.pageRelease();

    // 清理页面缓存中的预览数据
    this.getView().getPageCache().remove("previewBillId");
    this.getView().getPageCache().remove("previewBillNo");
    this.getView().getPageCache().remove("previewTimestamp");
    this.getView().getPageCache().remove("printConfig");

    // 清理临时生成的预览文件标记
    this.getView().getPageCache().remove("tempPreviewFile");

    // 清理分页缓存
    this.getView().getPageCache().remove("currentPage");
    this.getView().getPageCache().remove("totalPages");
  }
}

let plugin = new PrintPreviewReleasePlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.pageRelease()` 以确保父类的清理逻辑正常执行

## 注意事项

- 必须调用 `super.setWaterMarkInfo(e)` 以确保父类逻辑正常执行
- 水印文本应简洁明了，通常使用2-4个字的状态描述
- 透明度建议设置在 0.1-0.25 之间，过高会影响正常内容的阅读
- 不同状态使用不同颜色可以增强辨识度（如绿色表示通过，红色表示作废）
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量
- 水印显示为只读信息，不影响单据的实际数据和操作

# timerElapsed - 定时器事件

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 通过 `addTimer` 注册的定时器到达间隔时间时触发 |
| 方法签名 | `timerElapsed(e: TimerElapsedArgs): void` |

## 说明

定时器触发的回调事件。当通过 `this.getView().addTimer()` 注册定时器后，每当定时器达到设定的时间间隔时触发此方法。适用于需要定时刷新数据、轮询状态、自动保存等场景。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| e | TimerElapsedArgs | 定时器事件参数对象 |
| e.getTimerKey() | string | 获取触发的定时器标识，用于区分不同的定时器 |

## 业务场景

定时刷新看板数据，每30秒自动查询最新的待办任务数量。在工作台看板页面中，需要实时展示当前用户的待办任务数量、待审批单据数量等信息，通过定时器定期刷新确保数据的实时性。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { TimerElapsedArgs } from "@cosmic/bos-core/kd/bos/form/events";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";

/**
 * 工作台看板插件 - 定时刷新待办任务数量
 */
class WorkbenchTimerPlugin extends AbstractBillPlugIn {

  afterBindData(e: $.java.util.EventObject): void {
    super.afterBindData(e);

    // 首次加载时立即查询待办数据
    this.refreshTodoCount();

    // 注册定时器：每30秒刷新一次待办数据
    this.getView().addTimer("refreshTodoTimer", 30000);

    // 注册定时器：每5分钟刷新一次统计汇总数据
    this.getView().addTimer("refreshStatTimer", 300000);
  }

  timerElapsed(e: TimerElapsedArgs): void {
    super.timerElapsed(e);

    const timerKey = e.getTimerKey();

    if (timerKey === "refreshTodoTimer") {
      // 刷新待办任务数量
      this.refreshTodoCount();
    }

    if (timerKey === "refreshStatTimer") {
      // 刷新统计汇总数据
      this.refreshStatistics();
    }
  }

  // 查询并更新待办任务数量
  private refreshTodoCount(): void {
    const userId = RequestContext.get().getUserId();

    // 查询待办任务数量
    const todoFilter = new QFilter("assignee", "=", userId);
    const statusFilter = new QFilter("taskstatus", "=", "pending");
    const todoCount = QueryServiceHelper.queryCount(
      "bos_task",
      [todoFilter, statusFilter]
    );

    // 查询待审批单据数量
    const approveFilter = new QFilter("approver", "=", userId);
    const approveStatusFilter = new QFilter("approvestatus", "=", "waiting");
    const approveCount = QueryServiceHelper.queryCount(
      "bos_approval",
      [approveFilter, approveStatusFilter]
    );

    // 更新界面显示
    this.getModel().setValue("todocount", new BigDecimal(String(todoCount)));
    this.getModel().setValue("approvecount", new BigDecimal(String(approveCount)));

    // 更新最后刷新时间
    this.getModel().setValue("lastrefreshtime", new Date());

    // 刷新看板面板
    this.getView().updateView("todopanel");
    this.getView().updateView("approvepanel");
  }

  // 刷新统计汇总数据
  private refreshStatistics(): void {
    const userId = RequestContext.get().getUserId();

    // 查询本月已完成任务数
    const now = new Date();
    const monthStart = new Date(now.getFullYear(), now.getMonth(), 1);

    const completedFilter = new QFilter("assignee", "=", userId);
    const statusFilter = new QFilter("taskstatus", "=", "completed");
    const dateFilter = new QFilter("completetime", ">=", monthStart);

    const completedCount = QueryServiceHelper.queryCount(
      "bos_task",
      [completedFilter, statusFilter, dateFilter]
    );

    this.getModel().setValue("monthcompletedcount", new BigDecimal(String(completedCount)));
    this.getView().updateView("statpanel");
  }

  destory(): void {
    super.destory();

    // 页面关闭时移除定时器
    this.getView().removeTimer("refreshTodoTimer");
    this.getView().removeTimer("refreshStatTimer");
  }
}

let plugin = new WorkbenchTimerPlugin();
export { plugin };
```

## 注意事项

- 必须调用 `super.timerElapsed(e)` 以确保父类逻辑正常执行
- 通过 `this.getView().addTimer(key, interval)` 注册定时器，key 为定时器标识，interval 为间隔时间（毫秒）
- 通过 `e.getTimerKey()` 区分不同的定时器，一个插件中可以注册多个定时器
- 定时器间隔不宜过短，避免频繁查询给服务器造成压力，建议最短不低于10秒
- 页面关闭时应在 `destory` 方法中移除定时器，防止资源泄漏
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量
- 定时器回调中的数据库查询应尽量简单高效，避免复杂查询影响性能

# trace - 记录脚本执行结果

## 基本信息

| 属性 | 说明 |
|------|------|
| 所属接口 | AbstractFormPlugin |
| 触发时机 | 由插件主动调用，用于记录脚本执行过程中的关键信息到日志系统 |
| 方法签名 | `trace(msg: string): void` |

## 说明

记录脚本执行结果的日志方法。通过调用 `this.trace()` 可将关键业务操作的执行信息记录到苍穹的脚本日志系统中，便于后续进行问题排查、执行审计和性能分析。日志可在苍穹平台的脚本日志查看界面中检索。

## 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| msg | string | 要记录的日志消息内容 |

## 业务场景

记录关键业务操作的执行过程，用于问题排查和审计追踪。在采购订单审核流程中，记录审核操作的关键节点信息，包括操作人、操作时间、金额校验结果等，便于事后追溯审核过程。

## 完整示例代码

```typescript
import { AbstractBillPlugIn } from "@cosmic/bos-core/kd/bos/bill";
import { QueryServiceHelper } from "@cosmic/bos-core/kd/bos/servicehelper";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";
import { RequestContext } from "@cosmic/bos-core/kd/bos/context";

/**
 * 采购订单插件 - 记录关键业务操作的执行过程
 */
class PmPurorderAuditTracePlugin extends AbstractBillPlugIn {

  beforeDoOperation(e: BeforeDoOperationEventArgs): void {
    super.beforeDoOperation(e);

    const operate = e.getSource() as FormOperate;
    const opKey = operate.getOperateKey();
    const billNo = this.getModel().getValue("billno") as string;
    const userName = RequestContext.get().getUserName();

    this.trace("[操作前] 用户 " + userName + " 对单据 " + billNo + " 执行操作：" + opKey);

    if (opKey === "audit") {
      // 记录审核前的关键数据
      const totalAmount = this.getModel().getValue("totalamount") as BigDecimal;
      this.trace("[审核校验] 单据 " + billNo + " 合计金额：" + totalAmount.toString());

      // 校验审批权限和金额限制
      const userId = RequestContext.get().getUserId();
      const approvalLimit = this.queryApprovalLimit(userId);

      this.trace("[审核校验] 用户 " + userName + " 审批限额：" + approvalLimit.toString());

      if (totalAmount.compareTo(approvalLimit) > 0) {
        e.setCancel(true);
        this.trace("[审核拦截] 单据 " + billNo + " 金额 " + totalAmount.toString()
          + " 超过审批限额 " + approvalLimit.toString() + "，审核被拦截");
        this.getView().showErrorNotification(
          "单据金额超过您的审批限额（" + approvalLimit.toString() + "），请提交上级审批"
        );
        return;
      }

      this.trace("[审核校验] 单据 " + billNo + " 金额校验通过");
    }

    if (opKey === "save") {
      // 记录保存操作的分录行数
      const rowCount = this.getModel().getEntryRowCount("billentry");
      this.trace("[保存] 单据 " + billNo + " 分录行数：" + rowCount);
    }
  }

  afterDoOperation(e: AfterDoOperationEventArgs): void {
    super.afterDoOperation(e);

    const opKey = e.getOperateKey();
    const billNo = this.getModel().getValue("billno") as string;
    const isSuccess = e.getOperationResult().isSuccess();
    const userName = RequestContext.get().getUserName();

    if (isSuccess) {
      this.trace("[操作完成] 用户 " + userName + " 对单据 " + billNo
        + " 执行 " + opKey + " 操作成功");
    } else {
      const message = e.getOperationResult().getMessage();
      this.trace("[操作失败] 用户 " + userName + " 对单据 " + billNo
        + " 执行 " + opKey + " 操作失败，原因：" + message);
    }
  }

  // 查询用户审批限额
  private queryApprovalLimit(userId: any): BigDecimal {
    const filter = new QFilter("user_id", "=", userId);
    const result = QueryServiceHelper.queryOne(
      "bos_approval_limit",
      "limitamount",
      [filter]
    );

    if (result != null) {
      const limit = result.get("limitamount") as BigDecimal;
      this.trace("[查询] 用户审批限额查询结果：" + limit.toString());
      return limit;
    }

    // 默认审批限额
    this.trace("[查询] 未查到用户审批限额配置，使用默认限额 50000");
    return new BigDecimal("50000");
  }
}

let plugin = new PmPurorderAuditTracePlugin();
export { plugin };
```

## 注意事项

- `this.trace()` 是 AbstractFormPlugin 提供的方法，无需额外导入
- 日志内容会记录到苍穹平台的脚本日志系统中，可在"脚本日志"界面检索查看
- 建议在日志消息中加入分类标签（如 `[操作前]`、`[校验]`），便于日志检索和过滤
- 日志中应包含关键业务标识（如单据编号、操作人），便于定位具体单据
- 避免在循环中频繁调用 `trace()`，大量日志会影响性能
- 不要在日志中记录敏感信息（如密码、银行账号等）
- 插件类不能定义类属性，所有变量应在方法内部声明为局部变量
- 必须调用 `super.beforeDoOperation(e)` 和 `super.afterDoOperation(e)` 等父类方法
