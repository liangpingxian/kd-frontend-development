# 列表插件和列表控件业务示例

## 1. CloseQueryByOrListener (FilterContainer)

**场景**：在复杂的财务单据查询中，用户可能会不小心开启“或者(OR)”查询模式导致查询结果过多，严重影响性能。开发插件强制关闭特定单据的“或者”查询模式。

**方法**：`setQueryByOr(CloseQueryByOrEvent setQueryByOrEvent)`

```typescript
import { AbstractListPlugin } from "@cosmic/bos-core/kd/bos/list/plugin";
import { CloseQueryByOrEvent } from "@cosmic/bos-core/kd/bos/filter/events";

export class CloseQueryExample extends AbstractListPlugin {
    // 虽然通常在 FilterContainer 中注册，但列表插件基类常用于处理此类逻辑
    // 注意：实际开发中更多是在 filterContainerInit 中设置属性，此处演示事件响应
    public setQueryByOr(e: CloseQueryByOrEvent): void {
        // 场景：对于数据量巨大的“总账凭证”列表，禁止使用 OR 查询，强制为 AND
        if (this.getView().getEntityId() === "gl_voucher") {
            e.setCancel(true); // 阻止切换到 OR 模式
            this.getView().showTipNotification("为保证查询性能，凭证列表仅支持精确组合查询(AND)，不支持或者(OR)模式。");
        }
    }
}
```

```typescript
            // 逻辑：检查是否已加载，未加载则调用后台服务填充下拉数据...

    }
}
```

## 5. ActiveViewListener (BillList)

**场景**：在多视图列表（如“我的待办”、“全部单据”）切换时，根据当前激活的视图动态调整工具栏按钮的可用性。

**方法**：`activeView(ActiveViewEvent evt)`

```typescript
import { AbstractListPlugin } from "@cosmic/bos-core/kd/bos/list/plugin";
import { ActiveViewEvent } from "@cosmic/bos-core/kd/bos/list/events";

export class ActiveViewExample extends AbstractListPlugin {
    public activeView(evt: ActiveViewEvent): void {
        const viewKey = evt.getViewKey();
        // 场景：切换到“已审批”视图时，禁用“审核”按钮，启用“反审核”按钮
        if (viewKey === "approved_view") {
            this.getView().setEnable(false, "btn_audit");
            this.getView().setEnable(true, "btn_unaudit");
        } else if (viewKey === "pending_view") {
            this.getView().setEnable(true, "btn_audit");
            this.getView().setEnable(false, "btn_unaudit");
        }
    }
}
```

## 15. SetFilterListener (BillList)

**场景**：**最核心方法**。在列表查询数据前，强制追加数据权限过滤条件，或根据URL参数过滤列表。

**方法**：`setFilter(SetFilterEvent evt)`

```typescript
import { AbstractListPlugin } from "@cosmic/bos-core/kd/bos/list/plugin";
import { SetFilterEvent } from "@cosmic/bos-core/kd/bos/list/events";
import { QFilter } from "@cosmic/bos-core/kd/bos/orm/query";

export class SetFilterExample extends AbstractListPlugin {
    public setFilter(e: SetFilterEvent): void {
        // 场景：只允许用户看到自己创建的单据（除了管理员）
        const userId = this.getContext().getUserId();
        if (userId !== "admin") {
            e.getQFilter().add(new QFilter("creator", "=", userId));
        }

        // 场景：根据 URL 参数过滤，例如从报表穿透到列表
        const billNo = this.getView().getFormShowParameter().getCustomParam("billno");
        if (billNo) {
            e.getQFilter().add(new QFilter("billno", "=", billNo));
        }
    }
}
```

## 一、核心列表插件方法 (AbstractListPlugin)

### 1. initialize
**定义**: `initialize(InitListEventArgs e)`
**场景**: 插件加载时获取列表的标识（FormId）或上下文参数，用于区分不同业务场景。
```typescript
public initialize(e: InitListEventArgs): void {
    super.initialize(e);
    // 场景：根据不同的调用入口，标记当前插件模式
    const customParam = this.getView().getFormShowParameter().getCustomParam("mode");
    if (customParam === "simple") {
        this._isSimpleMode = true;
    }
}
```

### 2. listRowFilter
**定义**: `listRowFilter(ListRowFilterEvent args)`
**场景**: 复杂的行过滤逻辑，根据特定算法排除不符合条件的行。
```typescript
public listRowFilter(args: ListRowFilterEvent): void {
    // 场景：仅展示当前用户所属部门的单据（比 setFilter 更底层的控制）
    // 通常推荐优先使用 setFilter
}
```

### 3. setFilter (TOP 1)
**定义**: `setFilter(SetFilterEvent e)`
**场景**: **最常用**。在列表加载数据前，强制添加过滤条件和排序规则。
```typescript
public setFilter(e: SetFilterEvent): void {
    // 场景：默认只显示未关闭的单据，并按日期倒序排列
    e.getQFilter().add(new QFilter("billstatus", "!=", "C"));
    e.setOrderBy("billdate desc");
}
```

### 6. beforePackageData (TOP 5)
**定义**: `beforePackageData(BeforePackageDataEvent e)`
**场景**: 批量数据处理。查询出 100 行数据后，一次性查询这 100 个物料的实时库存，放入缓存，供 `packageData` 使用。
```typescript

```

### 7. packageData (TOP 8)
**定义**: `packageData(PackageDataEvent e)`
**场景**: 格式化单元格显示。例如将状态码 `A` 显示为 `保存`，或给金额加上千分位。

```typescript
        if (val) {

        }
    }
}
```
