# SDK Deprecated Index

这个索引用来回答两类问题:

- 某个类、方法或扩展点是否已经废弃
- 老版本写法迁移到当前 Cosmic V8.0.1 时应该看哪里

## 当前状态

仓库里还没有成体系的废弃 API 清单，所以这里先作为可读入口，而不是空占位。

## 推荐检索顺序

1. 先看 [method-index.md](method-index.md) 或 [class-index.md](class-index.md) 确认目标名称
2. 再回到对应知识卡、package 卡或 example，查看是否已经标记替代入口
3. 如果本地卡片没有结论，再回退到本地 `@cosmic/bos-core` 声明和在线 Javadoc 核对 `deprecated` 标记

## 首批优先补充方向

- 操作链路里已经不推荐继续直接使用的旧回调
- 列表、表单、F7 相关旧事件名与新事件名映射
- 数据访问和序列化相关老写法的替代建议
- 版本迁移时高频踩坑的旧 API

## 写条目时建议包含

- 废弃对象
- 废弃原因
- 推荐替代方案
- 影响范围
- 版本信息

在真正补出条目之前，不建议删除这个文件，因为 `strategy.md` 和 `README` 已把它当成固定检索入口。
