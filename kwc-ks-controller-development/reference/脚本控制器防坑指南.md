# 脚本控制器防坑指南

> 生成 KingScript Controller 后端脚本时需提前了解的运行时陷阱。
> 适用 Skill：`kwc-ks-controller-development`

---

## 一、KingScript 运行时 API

### 1.1 Unknown identifier: getParameter

**报错**：`Unknown identifier: getParameter`

**根因**：KS 运行时的 `request`（`ScriptRequestImpl`）不暴露 Java Servlet 风格方法。

```typescript
// ✅ 正确
const status = request.getStringQueryParam('status');
const page = request.getIntQueryParam('page') || 1;

// ❌ 错误 — Servlet 风格
const status = request.getParameter('status');
// ❌ 错误 — Express 风格
const status = request.query.status;
```

> **规则**：字符串参数用 `getStringQueryParam()`，整数参数用 `getIntQueryParam()`，完整 API 参见《脚本控制器开发指南》第五章。

### 1.2 Java is not defined

**报错**：`Java is not defined`

**根因**：KingScript 运行时中 `Java` 全局对象不存在，不能使用 GraalJS / Nashorn 风格的 `Java.type()`。

```typescript
// ✅ 正确 — 通过 import 引入
import { ArrayList, HashMap } from '@cosmic/bos-script/java/util';
import { QFilter } from '@cosmic/bos-core/kd/bos/orm/query';

// ❌ 错误 — GraalJS 风格
const ArrayList = Java.type('java.util.ArrayList');
```

> **规则**：所有 Java 类型必须通过 `import` 语句从对应包路径引入。

## 二、数据处理

### 2.1 ArrayList 序列化陷阱

**现象**：`ArrayList` 放入 `response.ok()` 后，前端收到空对象或字段丢失。

**根因**：KS 运行时的 JSON 序列化器不能自动处理 `ArrayList<HashMap>` 等 Java 集合类型。

```typescript
// ✅ 正确 — 调用 toArray() 转为原生数组
response.ok({
  orders: orders.toArray(),
  total: orders.size(),
});

// ❌ 错误 — 直接传 ArrayList
response.ok({
  orders: orders,
  total: orders.size(),
});
```

**前端兜底**：
```typescript
function safeParseArray(raw: any): any[] {
  if (Array.isArray(raw)) return raw;
  if (typeof raw === 'string') {
    try { const p = JSON.parse(raw); return Array.isArray(p) ? p : []; } catch { return []; }
  }
  return [];
}
```

### 2.2 QueryServiceHelper.query 结果不是 JS 数组

**现象**：对 `query()` 返回值调用 `.length`、`.map()` 报错。

**根因**：返回的是 Java `DynamicObjectCollection`，不支持 JS 数组方法。

```typescript
// ✅ 正确 — 用 for-of 遍历，构造 ArrayList + HashMap
const result = QueryServiceHelper.query('entity', 'f1, f2', [], '', 0);
const list = new ArrayList();
for (let row of result) {
  let item = new HashMap();
  item.put('f1', row.getString('f1'));
  item.put('f2', row.getString('f2'));
  list.add(item);
}

// ❌ 错误 — 当作 JS 数组
const count = result.length;       // undefined
const mapped = result.map(r => r); // TypeError
```

### 2.3 QFilter 字段名是实体标识不是数据库列名

**现象**：查询结果为空或报字段不存在。

```typescript
// ✅ 正确 — 苍穹实体标识
filters.push(new QFilter('billstatus', '=', status));

// ❌ 错误 — 数据库列名
filters.push(new QFilter('FBILLSTATUS', '=', status));
```

### 2.4 response.throwException 参数与 catch 声明

**规范**：`response.throwException(message, httpStatusCode, businessErrorCode)`

```typescript
// ✅ 正确 — e 声明为 any
} catch (e: any) {
  response.throwException(e.message || '查询失败', 500, 'QUERY_FAILED');
  return;
}

// ❌ 错误 — 未声明 any，TS 编译报错
} catch (e) {
  response.throwException(e.message, 500, 'QUERY_FAILED');
}
```

> `throwException` 调用后必须 `return`，否则后续代码仍会执行。

## 三、.kws 权限配置

### 3.1 Controller 401 — 权限配置矛盾

**根因**：`<permitAll>true</permitAll>` 与 `<anonymousUser>false</anonymousUser>` 同时存在，形成逻辑矛盾。

```xml
<!-- ✅ 公开接口只保留 permitAll -->
<permission><permission>
    <permitAll>true</permitAll>
</permission></permission>

<!-- ❌ permitAll 与 anonymousUser=false 矛盾 -->
<permission><permission>
    <permitAll>true</permitAll>
    <anonymousUser>false</anonymousUser>
    <entityNumber>sal_salorder</entityNumber>
</permission></permission>
```

## 快速检查清单

- [ ] request API：使用 `getStringQueryParam()` / `getIntQueryParam()`，非 `getParameter()` 或 `request.query`
- [ ] Java 类型：通过 `import` 引入，非 `Java.type()`
- [ ] import 路径：`QueryServiceHelper` → `@cosmic/bos-core/kd/bos/servicehelper`
- [ ] 查询方法：`QueryServiceHelper.query()`，非 `queryDataSet()`
- [ ] QFilter：`@cosmic/bos-core/kd/bos/orm/query`，字段名用实体标识
- [ ] ArrayList/HashMap：`@cosmic/bos-script/java/util`，响应前调用 `toArray()`
- [ ] catch 块：`e` 声明为 `e: any`
- [ ] throwException 后必须 `return`
- [ ] 公开接口 permission 只保留 `<permitAll>true</permitAll>`
